# 쿠폰 발급 시간 제한 TOCTOU 문제 분석

## 문제 요약

`MemberCouponController.issueCoupon()` 메서드에서 발급 제한 시간이 지난 후에도 쿠폰이 발급될 수 있는 **Time-of-Check to Time-of-Use (TOCTOU)** 취약점이 존재합니다.

## 취약한 코드 위치

### Coupon.java:59-61
```java
if (this.issueStartedAt.isAfter(LocalDateTime.now()) ||
    this.issueEndedAt.isBefore(LocalDateTime.now())) {
    throw new IllegalArgumentException("쿠폰을 발급할 수 없는 시간입니다.");
}
```

### MemberCoupon.java:48-59
```java
public static MemberCoupon issue(Long memberId, Coupon coupon) {
    coupon.issue();  // 여기서 시간 검증

    MemberCoupon memberCoupon = new MemberCoupon();
    memberCoupon.memberId = memberId;
    memberCoupon.coupon = coupon;
    memberCoupon.issuedAt = LocalDateTime.now();  // 다시 현재 시각 획득
    memberCoupon.useEndedAt = memberCoupon.issuedAt.plusDays(COUPON_USABLE_DAYS);
    memberCoupon.modifiedAt = LocalDateTime.now();
    memberCoupon.used = false;

    return memberCoupon;
}
```

## 문제의 핵심 원인

### 1. LocalDateTime.now()의 다중 호출
- `coupon.issue()` 내부에서 `LocalDateTime.now()` 호출 (검증용)
- `MemberCoupon.issue()` 내부에서 `LocalDateTime.now()` 다시 호출 (기록용)
- 두 호출 사이의 시간 차이로 인해 검증과 실제 발급 시점이 달라질 수 있음

### 2. 트랜잭션 실행 시간
- `@Transactional` 어노테이션으로 인한 트랜잭션 범위
- 검증부터 DB 커밋까지의 시간 차이

### 3. 동시성 환경에서의 타이밍 이슈
- GC, 스케줄링, 네트워크 지연 등으로 인한 처리 지연

## 영향 분석

### 비즈니스 영향
- ❌ 발급 종료된 쿠폰이 추가로 발급될 수 있음
- ❌ 쿠폰 발급 수량 제한 우회 가능
- ❌ 이벤트 종료 후 불공정한 혜택 제공
- ❌ 운영 정책 위반

### 기술적 영향
- ❌ 데이터 무결성 훼손
- ❌ 시간 기반 제약 조건 신뢰성 저하
- ❌ 감사 추적(audit trail) 정확성 문제

## 해결 방안

### 방안 1: 트랜잭션 시작 시점에 시간 캡처 (권장)

**장점:**
- 검증과 발급이 동일한 시간 기준 사용
- 코드 변경 최소화
- 트랜잭션 일관성 보장

**구현:**

#### CouponIssuer.java 수정
```java
@Transactional
public MemberCoupon issueCoupon(Long couponId, Long memberId) {
    log.info("쿠폰 발급 요청: couponId={}, memberId={}", couponId, memberId);

    LocalDateTime now = LocalDateTime.now();  // 트랜잭션 시작 시점 캡처

    Member member = memberService.getMember(memberId);
    Coupon coupon = couponService.getCouponWithLock(couponId);
    MemberCoupon memberCoupon = issueMemberCoupon(member, coupon, now);  // 시간 전달

    observers.forEach(observer -> observer.onIssue(memberCoupon.getId()));

    log.info("쿠폰 발급 완료: couponId={}, memberId={}, memberCouponId={}",
             couponId, memberId, memberCoupon.getId());

    return memberCoupon;
}

@Transactional
public MemberCoupon issueMemberCoupon(Member member, Coupon coupon, LocalDateTime now) {
    MemberCoupon memberCoupon = MemberCoupon.issue(member.getId(), coupon, now);
    memberCouponRepository.save(memberCoupon);
    return memberCoupon;
}
```

#### Coupon.java 수정
```java
public void issue(LocalDateTime now) {
    if (this.issueStartedAt.isAfter(now) || this.issueEndedAt.isBefore(now)) {
        throw new IllegalArgumentException("쿠폰을 발급할 수 없는 시간입니다.");
    }
    if (couponStatus.isNotIssuable() || !this.issuable) {
        throw new IllegalArgumentException("쿠폰을 발급할 수 없는 상태입니다.");
    }
    if (this.limitType.isNotIssueCountLimit()) {
        return;
    }
    if (this.issueLimit <= this.issueCount) {
        throw new IllegalArgumentException("쿠폰을 더 이상 발급할 수 없습니다.");
    }
    this.issueCount++;
}
```

### 방안 2: DB 제약 조건 추가 (방어적 프로그래밍)

**장점:**
- 데이터베이스 레벨에서 강제적으로 보장
- 애플리케이션 버그가 있어도 방어 가능
- 데이터 무결성 최대 보장

**구현:**

```sql
-- member_coupon 테이블에 CHECK 제약 조건 추가
ALTER TABLE member_coupon
ADD CONSTRAINT chk_issue_time_valid
CHECK (
    issued_at >= (
        SELECT issue_started_at
        FROM coupon
        WHERE coupon.id = member_coupon.coupon_id
    ) AND
    issued_at <= (
        SELECT issue_ended_at
        FROM coupon
        WHERE coupon.id = member_coupon.coupon_id
    )
);
```

### 방안 3: 발급 시간 먼저 기록 후 검증

**장점:**
- 검증과 기록 사이의 시간 차이 제거
- 실제 발급 시간으로 검증

**구현:**

## 테스트 케이스

```java
@Test
void 발급_종료_시간_직전_발급_성공() {
    // given
    LocalDateTime now = LocalDateTime.of(2025, 10, 31, 23, 59, 59);
    Coupon coupon = createCoupon(now.minusDays(1), now);

    // when
    MemberCoupon memberCoupon = MemberCoupon.issue(1L, coupon, now);

    // then
    assertThat(memberCoupon.getIssuedAt()).isEqualTo(now);
}

@Test
void 발급_종료_시간_직후_발급_실패() {
    // given
    LocalDateTime endTime = LocalDateTime.of(2025, 10, 31, 23, 59, 59);
    LocalDateTime now = endTime.plusNanos(1);
    Coupon coupon = createCoupon(endTime.minusDays(1), endTime);

    // when & then
    assertThatThrownBy(() -> MemberCoupon.issue(1L, coupon, now))
        .isInstanceOf(IllegalArgumentException.class)
        .hasMessage("쿠폰을 발급할 수 없는 시간입니다.");
}

@Test
void 발급_검증과_기록_시간_일치_확인() {
    // given
    LocalDateTime now = LocalDateTime.of(2025, 10, 31, 12, 0, 0);
    Coupon coupon = createCoupon(now.minusDays(1), now.plusDays(1));

    // when
    MemberCoupon memberCoupon = MemberCoupon.issue(1L, coupon, now);

    // then
    assertThat(memberCoupon.getIssuedAt()).isEqualTo(now);
    assertThat(memberCoupon.getModifiedAt()).isEqualTo(now);
    assertThat(memberCoupon.getUseEndedAt()).isEqualTo(now.plusDays(7));
}
```
