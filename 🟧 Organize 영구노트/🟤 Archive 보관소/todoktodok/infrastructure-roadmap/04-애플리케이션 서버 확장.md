# 애플리케이션 서버 확장

## 1만 명 단계
### 단일 인스턴스 최적화

#### JVM 튜닝
```bash
java -Xms2G -Xmx2G \
     -XX:+UseG1GC \
     -XX:MaxGCPauseMillis=200 \
     -XX:+HeapDumpOnOutOfMemoryError \
     -jar app.jar
```

#### 모니터링 설정
```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,metrics,prometheus
  metrics:
    export:
      prometheus:
        enabled: true
```

#### 주의사항
- Heap 메모리 모니터링 필수
- CPU/Memory 사용률 80% 이하 유지
- 로그 관리 (logback async appender)

## 10만 명 단계
### 스케일 아웃 (Scale Out)
**3-5개 인스턴스로 확장**

#### Auto Scaling 설정
```yaml
# AWS Auto Scaling Group
MinSize: 3
MaxSize: 10
DesiredCapacity: 3

# Scaling Policy
TargetValue: 70  # CPU 사용률
ScaleOutCooldown: 300
ScaleInCooldown: 300
```

#### Session 클러스터링
```yaml
spring:
  session:
    store-type: redis
    redis:
      namespace: spring:session
      flush-mode: on_save
```

```java
@Configuration
@EnableRedisHttpSession(maxInactiveIntervalInSeconds = 3600)
public class SessionConfig {
    // Redis Session 자동 설정
}
```

#### Stateless 설계 원칙
- JWT 기반 인증
- 서버에 상태 저장 금지
- 파일 업로드 → S3 직접 업로드

## 1000만 명 단계
### 마이크로서비스 아키텍처

#### 서비스 분리
```
API Gateway
  ├── Room Service (토론방 관리)
  ├── User Service (사용자 관리)
  ├── Comment Service (댓글 관리)
  ├── Notification Service (알림)
  └── Analytics Service (통계)
```

#### Spring Cloud 구성
```yaml
# application.yml (Room Service)
spring:
  application:
    name: room-service
  cloud:
    gateway:
      routes:
        - id: room-service
          uri: lb://room-service
          predicates:
            - Path=/api/rooms/**
```

#### 서비스 간 통신
```java
// OpenFeign 사용
@FeignClient(name = "user-service")
public interface UserServiceClient {
    @GetMapping("/api/users/{id}")
    UserDto getUser(@PathVariable Long id);
}

// 또는 Kafka 이벤트 기반
@KafkaListener(topics = "room-created")
public void handleRoomCreated(RoomCreatedEvent event) {
    // 알림 발송
}
```

#### Circuit Breaker
```java
@CircuitBreaker(name = "userService", fallbackMethod = "fallbackGetUser")
@Retry(name = "userService", fallbackMethod = "fallbackGetUser")
public UserDto getUser(Long userId) {
    return userServiceClient.getUser(userId);
}

public UserDto fallbackGetUser(Long userId, Exception e) {
    return UserDto.createDefault(userId);
}
```

#### Kubernetes 배포
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: room-service
spec:
  replicas: 5
  selector:
    matchLabels:
      app: room-service
  template:
    metadata:
      labels:
        app: room-service
    spec:
      containers:
      - name: room-service
        image: room-service:latest
        resources:
          requests:
            memory: "2Gi"
            cpu: "1000m"
          limits:
            memory: "4Gi"
            cpu: "2000m"
```
