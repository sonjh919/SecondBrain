# 모니터링과 관찰성

## 1만 명 단계
### 기본 모니터링

#### Spring Boot Actuator
```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,metrics,info
  endpoint:
    health:
      show-details: always
```

#### 기본 메트릭 수집
- CPU, Memory 사용률
- Heap 메모리
- GC 통계
- HTTP 요청 카운트
- Response Time

#### 로깅
```xml
<!-- logback-spring.xml -->
<appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
    <file>logs/application.log</file>
    <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
        <fileNamePattern>logs/application-%d{yyyy-MM-dd}.log</fileNamePattern>
        <maxHistory>30</maxHistory>
    </rollingPolicy>
</appender>
```

```java
// Custom Metrics
@Component
public class RoomMetrics {
    private final Counter roomCreatedCounter;
    private final Gauge activeRoomsGauge;
    
    public RoomMetrics(MeterRegistry registry) {
        this.roomCreatedCounter = Counter.builder("room.created")
            .description("Number of rooms created")
            .register(registry);
            
        this.activeRoomsGauge = Gauge.builder("room.active", 
                                               roomRepository, 
                                               RoomRepository::countActiveRooms)
            .register(registry);
    }
    
    public void recordRoomCreated() {
        roomCreatedCounter.increment();
    }
}
```

#### Slow Query 모니터링
```yaml
# application.yml
spring:
  jpa:
    properties:
      hibernate:
        use_sql_comments: true
        format_sql: true
logging:
  level:
    org.hibernate.SQL: DEBUG
    org.hibernate.type.descriptor.sql.BasicBinder: TRACE

# MySQL 설정
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow-query.log
long_query_time = 1
```

#### Alert 설정
```yaml
# Grafana Alert
- alert: HighErrorRate
  expr: rate(http_requests_total{status=~"5.."}[5m]) > 0.05
  for: 5m
  annotations:
    summary: "High error rate detected"
    
- alert: HighResponseTime
  expr: http_request_duration_seconds{quantile="0.99"} > 2
  for: 5m
  annotations:
    summary: "P99 response time > 2s"
```

## 1000만 명 단계
### 분산 추적 (Distributed Tracing)

#### Spring Cloud Sleuth + Zipkin
```yaml
spring:
  sleuth:
    sampler:
      probability: 0.1  # 10% 샘플링
  zipkin:
    base-url: http://zipkin-server:9411
```


#### ELK Stack (Elasticsearch, Logstash, Kibana)
```yaml
# logback-spring.xml with Logstash
<appender name="LOGSTASH" class="net.logstash.logback.appender.LogstashTcpSocketAppender">
    <destination>logstash:5000</destination>
    <encoder class="net.logstash.logback.encoder.LogstashEncoder">
        <includeMdc>true</includeMdc>
        <customFields>{"service":"room-service"}</customFields>
    </encoder>
</appender>
```

#### 실시간 대시보드
- **메트릭**: Grafana + Prometheus
- **로그**: Kibana (ELK)
- **추적**: Zipkin 또는 Jaeger
- **비즈니스 메트릭**: Custom Dashboard

#### SLO/SLI 정의
```yaml
# Service Level Objectives
- Availability: 99.9% (월 43분 다운타임)
- P99 Latency: < 500ms
- Error Rate: < 0.1%

# Service Level Indicators
SLI:
  - http_requests_total{status="200"} / http_requests_total
  - histogram_quantile(0.99, http_request_duration_seconds)
```

#### Chaos Engineering

```java
// 장애 주입 테스트
```
