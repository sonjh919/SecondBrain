# 데이터베이스 확장

## 1만 명 단계
### 현재 상황
- 읽기 쿼리가 쓰기보다 많고 느림
- 읽기 병목이 쓰기 성능에도 영향

### 우선 조치
#### 1. 인덱스 최적화
```sql
-- 최신순 조회를 위한 커버링 인덱스
CREATE INDEX idx_room_created_status 
ON room(created_at DESC, status) 
INCLUDE (id, title, view_count);

-- 슬로우 쿼리 분석
EXPLAIN ANALYZE 
SELECT * FROM room 
WHERE status = 'ACTIVE' 
ORDER BY created_at DESC 
LIMIT 20;
```

#### 2. Connection Pool 최적화
```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 10
      minimum-idle: 5
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000
```

## 10만 명 단계
### Read Replica 도입
**가장 중요한 변화**

#### 아키텍처
```
Master (Write)
  ├── Replica 1 (Read)
  ├── Replica 2 (Read)
  └── Replica 3 (Read)
```

#### Spring 설정
```java
@Transactional(readOnly = true)
public List<Room> getRooms() {
    // Replica로 자동 라우팅
    return roomRepository.findAll();
}

@Transactional
public Room createRoom(RoomDto dto) {
    // Master로 라우팅
    return roomRepository.save(room);
}
```

#### application.yml
```yaml
spring:
  datasource:
    master:
      jdbc-url: jdbc:mysql://master-db:3306/mydb
      username: admin
      password: ${DB_PASSWORD}
    slave:
      jdbc-url: jdbc:mysql://replica-db:3306/mydb
      username: readonly
      password: ${DB_PASSWORD}
```

### 쿼리 최적화
- N+1 문제 해결 (Fetch Join, @EntityGraph)
- Batch Insert/Update
- 페이징 성능 개선 (No Offset)

```java
// No Offset 페이징
@Query("SELECT r FROM Room r WHERE r.id < :lastId ORDER BY r.id DESC")
List<Room> findRoomsNoOffset(@Param("lastId") Long lastId, Pageable pageable);
```

## 1000만 명 단계
### 샤딩 (Sharding)
**데이터를 수평 분할**

#### 샤딩 전략
```java
// Hash 기반 샤딩
int shardId = roomId % 4;

// Shard 0: room_id % 4 == 0 → DB0
// Shard 1: room_id % 4 == 1 → DB1
// Shard 2: room_id % 4 == 2 → DB2
// Shard 3: room_id % 4 == 3 → DB3
```

#### 샤딩 라우터 구현
```java
@Component
public class ShardingRouter {
    private final Map<Integer, DataSource> shardMap;
    
    public DataSource getDataSource(Long roomId) {
        int shardId = (int) (roomId % 4);
        return shardMap.get(shardId);
    }
}
```

### CQRS 패턴
- **Command (Write)**: MySQL Master
- **Query (Read)**: Elasticsearch or MongoDB

```java
// Write Model
@Service
public class RoomCommandService {
    public Room createRoom(RoomDto dto) {
        Room room = roomRepository.save(dto.toEntity());
        eventPublisher.publish(new RoomCreatedEvent(room));
        return room;
    }
}

// Read Model
@Service
public class RoomQueryService {
    public List<Room> searchRooms(String keyword) {
        return elasticsearchRepository.search(keyword);
    }
}
```

### 데이터 아카이빙
```sql
-- 6개월 이상 된 토론방 → Cold Storage
INSERT INTO room_archive 
SELECT * FROM room 
WHERE created_at < DATE_SUB(NOW(), INTERVAL 6 MONTH);

DELETE FROM room 
WHERE created_at < DATE_SUB(NOW(), INTERVAL 6 MONTH);
```
