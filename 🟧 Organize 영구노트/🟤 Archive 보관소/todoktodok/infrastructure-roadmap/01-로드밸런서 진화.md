# 로드밸런서 진화

## 1만 명 단계
### 문제점
- Nginx SPOF (Single Point of Failure)
- 외부 장애 시 트래픽 진입 불가
- 단일 서버 장애가 전체 서비스 중단으로 이어짐

### 해결방안
- **AWS ALB (Application Load Balancer) 도입**
  - 고가용성 보장 (Multi-AZ)
  - Auto Scaling과 통합
  - Health Check 자동화

### 설정 예시
```yaml
# ALB Target Group 설정
HealthCheck:
  Path: /actuator/health
  Interval: 30s
  Timeout: 5s
  HealthyThreshold: 2
  UnhealthyThreshold: 3
```

## 10만 명 단계
### 변화
- ALB + 다중 AZ 배포
- Connection Draining 설정
- 트래픽 기반 Auto Scaling

### 고려사항
- ALB Idle Timeout: 60초 (WebSocket 사용 시 조정 필요)
- Target Group별 라우팅 규칙 세분화
- CloudWatch로 ALB 메트릭 모니터링

---

## 1000만 명 단계
### 아키텍처
- **Multi-Region 배포**
  - Route 53 Geo-routing
  - 지역별 ALB 배치
  
- **Global Accelerator 고려**
  - 글로벌 정적 IP
  - AWS 백본 네트워크 활용
  - DDoS 보호 강화

### CDN 통합
- CloudFront + ALB
- API Gateway + Lambda@Edge (동적 콘텐츠)
- 정적 리소스는 CDN에서 직접 서빙

---

## 핵심 포인트
- 1만: Nginx → ALB (SPOF 제거)
- 10만: Multi-AZ + Auto Scaling
- 1000만: Multi-Region + Global Accelerator
