# 캐싱 전략

## 개요
1억 계정 규모에서 비밀번호 검증 성능을 최적화하기 위한 다층 캐싱 전략입니다.

## 캐싱 계층 구조

```
요청 → Bloom Filter → Redis Cache → Database
        (수 μs)      (수 ms)        (수십 ms)
```

## 1. Redis 캐싱

### 설정

```java
@Configuration
@EnableCaching
public class RedisConfig {
    
    @Bean
    public RedisConnectionFactory redisConnectionFactory() {
        RedisStandaloneConfiguration config = new RedisStandaloneConfiguration();
        config.setHostName("localhost");
        config.setPort(6379);
        
        LettuceConnectionFactory factory = new LettuceConnectionFactory(config);
        factory.setShareNativeConnection(false); // 동시성 처리
        
        return factory;
    }
    
    @Bean
    public RedisTemplate<String, Integer> redisTemplate(
            RedisConnectionFactory connectionFactory) {
        
        RedisTemplate<String, Integer> template = new RedisTemplate<>();
        template.setConnectionFactory(connectionFactory);
        
        // Key: String Serializer
        template.setKeySerializer(new StringRedisSerializer());
        
        // Value: Integer Serializer
        template.setValueSerializer(new GenericToStringSerializer<>(Integer.class));
        
        template.afterPropertiesSet();
        return template;
    }
    
    @Bean
    public CacheManager cacheManager(RedisConnectionFactory factory) {
        RedisCacheConfiguration config = RedisCacheConfiguration.defaultCacheConfig()
            .entryTtl(Duration.ofHours(24)) // 24시간 TTL
            .disableCachingNullValues()
            .serializeKeysWith(
                RedisSerializationContext.SerializationPair.fromSerializer(
                    new StringRedisSerializer()))
            .serializeValuesWith(
                RedisSerializationContext.SerializationPair.fromSerializer(
                    new GenericJackson2JsonRedisSerializer()));
        
        return RedisCacheManager.builder(factory)
            .cacheDefaults(config)
            .build();
    }
}
```

## 2. 캐시 키 전략

### 키 네이밍 규칙

```java
public class CacheKeyGenerator {
    
    private static final String PREFIX_PASSWORD_COUNT = "pwd:count:";
    private static final String PREFIX_USER_HISTORY = "user:history:";
    private static final String PREFIX_COMMON_PASSWORDS = "common:passwords";
    
    /**
     * 비밀번호 사용 카운트 캐시 키
     * 예: pwd:count:5f4dcc3b5aa765d61d8327deb882cf99
     */
    public static String passwordCountKey(String passwordHash) {
        return PREFIX_PASSWORD_COUNT + passwordHash;
    }
    
    /**
     * 사용자 비밀번호 이력 캐시 키
     * 예: user:history:123:recent5
     */
    public static String userRecentHistoryKey(Long userId) {
        return PREFIX_USER_HISTORY + userId + ":recent5";
    }
    
    /**
     * 사용자 3개월 이력 캐시 키
     * 예: user:history:123:3months
     */
    public static String userThreeMonthHistoryKey(Long userId) {
        return PREFIX_USER_HISTORY + userId + ":3months";
    }
    
    /**
     * 흔한 비밀번호 집합 캐시 키
     */
    public static String commonPasswordsKey() {
        return PREFIX_COMMON_PASSWORDS;
    }
}
```

## 3. 캐시 적용 전략

### 비밀번호 사용 카운트 캐싱

```java
@Service
public class PasswordUsageCountCacheService {
    
    private final RedisTemplate<String, Integer> redisTemplate;
    private final PasswordUsageCountRepository repository;
    
    private static final Duration CACHE_TTL = Duration.ofHours(24);
    
    @Autowired
    public PasswordUsageCountCacheService(
            RedisTemplate<String, Integer> redisTemplate,
            PasswordUsageCountRepository repository) {
        this.redisTemplate = redisTemplate;
        this.repository = repository;
    }
    
    /**
     * 비밀번호 사용 카운트 조회 (캐시 우선)
     */
    public Integer getUsageCount(String passwordHash) {
        String cacheKey = CacheKeyGenerator.passwordCountKey(passwordHash);
        
        // 1. Redis 캐시 확인
        Integer cachedCount = redisTemplate.opsForValue().get(cacheKey);
        if (cachedCount != null) {
            return cachedCount;
        }
        
        // 2. DB 조회
        Integer dbCount = repository.findById(passwordHash)
            .map(PasswordUsageCount::getUsageCount)
            .orElse(0);
        
        // 3. 캐시 저장 (0은 캐싱하지 않음)
        if (dbCount > 0) {
            redisTemplate.opsForValue().set(cacheKey, dbCount, CACHE_TTL);
        }
        
        return dbCount;
    }
    
    /**
     * 캐시 무효화
     */
    public void invalidateCache(String passwordHash) {
        String cacheKey = CacheKeyGenerator.passwordCountKey(passwordHash);
        redisTemplate.delete(cacheKey);
    }
    
    /**
     * 여러 비밀번호의 캐시 일괄 무효화
     */
    public void invalidateCacheBatch(List<String> passwordHashes) {
        List<String> keys = passwordHashes.stream()
            .map(CacheKeyGenerator::passwordCountKey)
            .collect(Collectors.toList());
        
        redisTemplate.delete(keys);
    }
}
```

### 사용자 이력 캐싱

```java
@Service
public class PasswordHistoryCacheService {
    
    private final RedisTemplate<String, List<String>> redisTemplate;
    private final PasswordHistoryRepository repository;
    
    /**
     * 최근 5개 비밀번호 이력 조회 (캐시 활용)
     */
    @Cacheable(value = "userRecentHistory", key = "#userId")
    public List<String> getRecentFivePasswords(Long userId) {
        return repository.findTop5ByUserIdOrderByUsedAtDesc(userId)
            .stream()
            .map(PasswordHistory::getPasswordHash)
            .collect(Collectors.toList());
    }
    
    /**
     * 3개월 이력 조회 (캐시 활용)
     */
    @Cacheable(value = "userThreeMonthHistory", key = "#userId")
    public List<String> getThreeMonthPasswords(Long userId) {
        LocalDateTime threeMonthsAgo = LocalDateTime.now().minusMonths(3);
        
        return repository.findByUserIdAndUsedAtAfter(userId, threeMonthsAgo)
            .stream()
            .map(PasswordHistory::getPasswordHash)
            .collect(Collectors.toList());
    }
    
    /**
     * 사용자 이력 캐시 무효화
     */
    @CacheEvict(value = {"userRecentHistory", "userThreeMonthHistory"}, key = "#userId")
    public void invalidateUserCache(Long userId) {
        // 어노테이션이 처리
    }
}
```

## 4. 캐시 워밍 전략

### 애플리케이션 시작 시 캐시 예열

```java
@Component
public class CacheWarmer implements ApplicationRunner {
    
    private final PasswordUsageCountRepository usageCountRepository;
    private final RedisTemplate<String, Integer> redisTemplate;
    
    @Override
    public void run(ApplicationArguments args) {
        warmUpCommonPasswords();
    }
    
    /**
     * 100명 이상 사용하는 비밀번호를 미리 캐싱
     */
    private void warmUpCommonPasswords() {
        log.info("Starting cache warm-up for common passwords");
        
        List<PasswordUsageCount> commonPasswords = 
            usageCountRepository.findByUsageCountGreaterThanEqual(100);
        
        for (PasswordUsageCount puc : commonPasswords) {
            String cacheKey = CacheKeyGenerator.passwordCountKey(puc.getPasswordHash());
            redisTemplate.opsForValue().set(
                cacheKey, 
                puc.getUsageCount(),
                Duration.ofHours(24)
            );
        }
        
        log.info("Cache warm-up completed: {} passwords cached", 
                 commonPasswords.size());
    }
}
```

### 주기적 캐시 갱신

```java
@Component
public class CacheRefreshScheduler {
    
    private final PasswordUsageCountRepository repository;
    private final RedisTemplate<String, Integer> redisTemplate;
    
    /**
     * 매 시간마다 인기 비밀번호 캐시 갱신
     */
    @Scheduled(cron = "0 0 * * * *") // 매 시 정각
    public void refreshPopularPasswordsCache() {
        log.info("Refreshing popular passwords cache");
        
        // 상위 1000개 비밀번호 갱신
        List<PasswordUsageCount> topPasswords = 
            repository.findTop1000ByOrderByUsageCountDesc();
        
        for (PasswordUsageCount puc : topPasswords) {
            String cacheKey = CacheKeyGenerator.passwordCountKey(puc.getPasswordHash());
            redisTemplate.opsForValue().set(
                cacheKey,
                puc.getUsageCount(),
                Duration.ofHours(24)
            );
        }
        
        log.info("Refreshed {} password caches", topPasswords.size());
    }
}
```

## 5. 캐시 무효화 전략

### Write-Through 패턴

```java
@Service
public class PasswordUpdateService {
    
    private final UserRepository userRepository;
    private final PasswordUsageCountRepository usageCountRepository;
    private final PasswordUsageCountCacheService cacheService;
    
    @Transactional
    public void updatePassword(Long userId, String oldHash, String newHash) {
        // 1. DB 업데이트
        userRepository.updatePassword(userId, newHash);
        usageCountRepository.incrementOrCreate(newHash);
        usageCountRepository.decrement(oldHash);
        
        // 2. 캐시 무효화 (Write-Through)
        cacheService.invalidateCache(newHash);
        cacheService.invalidateCache(oldHash);
        
        // 3. Bloom Filter 업데이트 이벤트 발행
        eventPublisher.publishEvent(new PasswordUsageChangedEvent(newHash, oldHash));
    }
}
```

### 이벤트 기반 캐시 무효화

```java
@Component
public class CacheInvalidationListener {
    
    private final PasswordUsageCountCacheService cacheService;
    
    @EventListener
    @Async
    public void onPasswordChanged(PasswordChangedEvent event) {
        // 비동기로 캐시 무효화
        cacheService.invalidateCache(event.getNewPasswordHash());
        cacheService.invalidateCache(event.getOldPasswordHash());
        
        log.debug("Cache invalidated for password change");
    }
}
```

## 6. 캐시 히트율 모니터링

### 메트릭 수집

```java
@Aspect
@Component
public class CacheMetricsAspect {
    
    private final MeterRegistry meterRegistry;
    private final Counter cacheHits;
    private final Counter cacheMisses;
    
    @Autowired
    public CacheMetricsAspect(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
        this.cacheHits = Counter.builder("cache.hits")
            .description("Number of cache hits")
            .tag("type", "password_count")
            .register(meterRegistry);
        this.cacheMisses = Counter.builder("cache.misses")
            .description("Number of cache misses")
            .tag("type", "password_count")
            .register(meterRegistry);
    }
    
    @Around("execution(* PasswordUsageCountCacheService.getUsageCount(..))")
    public Object monitorCacheAccess(ProceedingJoinPoint joinPoint) throws Throwable {
        String passwordHash = (String) joinPoint.getArgs()[0];
        String cacheKey = CacheKeyGenerator.passwordCountKey(passwordHash);
        
        // 캐시 존재 여부 확인
        Boolean exists = redisTemplate.hasKey(cacheKey);
        
        if (Boolean.TRUE.equals(exists)) {
            cacheHits.increment();
        } else {
            cacheMisses.increment();
        }
        
        return joinPoint.proceed();
    }
}
```

### 캐시 성능 리포트

```java
@Service
public class CacheMetricsService {
    
    private final MeterRegistry meterRegistry;
    
    /**
     * 캐시 히트율 계산
     */
    public double getCacheHitRate() {
        double hits = meterRegistry.counter("cache.hits").count();
        double misses = meterRegistry.counter("cache.misses").count();
        double total = hits + misses;
        
        return total > 0 ? (hits / total) * 100 : 0;
    }
    
    /**
     * 캐시 통계 조회
     */
    public CacheStats getCacheStats() {
        return CacheStats.builder()
            .hitRate(getCacheHitRate())
            .hits(meterRegistry.counter("cache.hits").count())
            .misses(meterRegistry.counter("cache.misses").count())
            .build();
    }
    
    /**
     * 매일 캐시 통계 로깅
     */
    @Scheduled(cron = "0 0 0 * * *")
    public void logDailyCacheStats() {
        CacheStats stats = getCacheStats();
        log.info("Daily Cache Stats - Hit Rate: {:.2f}%, Hits: {}, Misses: {}",
                 stats.getHitRate(), stats.getHits(), stats.getMisses());
    }
}
```

```java
@Service
public class RedisCacheMonitor {
    
    private final RedisTemplate<String, ?> redisTemplate;
    
    /**
     * Redis 메모리 사용량 조회
     */
    public Map<String, Object> getMemoryInfo() {
        return redisTemplate.execute((RedisCallback<Map<String, Object>>) connection -> {
            Properties info = connection.info("memory");
            
            Map<String, Object> memInfo = new HashMap<>();
            memInfo.put("used_memory", info.getProperty("used_memory"));
            memInfo.put("used_memory_human", info.getProperty("used_memory_human"));
            memInfo.put("used_memory_peak", info.getProperty("used_memory_peak"));
            memInfo.put("used_memory_peak_human", info.getProperty("used_memory_peak_human"));
            
            return memInfo;
        });
    }
    
    /**
     * 특정 패턴의 키 개수 조회
     */
    public long countKeysByPattern(String pattern) {
        Set<String> keys = redisTemplate.keys(pattern);
        return keys != null ? keys.size() : 0;
    }
    
    /**
     * 캐시 크기 모니터링
     */
    @Scheduled(cron = "0 */10 * * * *") // 10분마다
    public void monitorCacheSize() {
        long passwordCountKeys = countKeysByPattern("pwd:count:*");
        long userHistoryKeys = countKeysByPattern("user:history:*");
        
        log.info("Cache Size - Password Count: {}, User History: {}",
                 passwordCountKeys, userHistoryKeys);
    }
}
```

## 8. 장애 대응 전략

### Circuit Breaker 패턴

```java
@Service
public class ResilientPasswordService {
    
    private final PasswordUsageCountCacheService cacheService;
    private final PasswordUsageCountRepository repository;
    
    @CircuitBreaker(name = "redisCache", fallbackMethod = "fallbackGetUsageCount")
    @Retry(name = "redisCache", fallbackMethod = "fallbackGetUsageCount")
    public Integer getUsageCount(String passwordHash) {
        return cacheService.getUsageCount(passwordHash);
    }
    
    /**
     * Redis 장애 시 DB 직접 조회
     */
    private Integer fallbackGetUsageCount(String passwordHash, Exception e) {
        log.warn("Redis unavailable, falling back to database: {}", e.getMessage());
        
        return repository.findById(passwordHash)
            .map(PasswordUsageCount::getUsageCount)
            .orElse(0);
    }
}
```

### 캐시 실패 시 처리

```java
@Service
public class SafePasswordValidator {
    
    /**
     * 캐시 실패해도 검증은 계속 진행
     */
    public void validatePassword(String passwordHash) {
        try {
            Integer count = cacheService.getUsageCount(passwordHash);
            
            if (count != null && count >= 100) {
                throw new InvalidPasswordException("Common password");
            }
        } catch (RedisConnectionException e) {
            log.error("Redis connection failed, skipping cache check", e);
            // 캐시 실패 시 DB 직접 조회
            Integer count = repository.findById(passwordHash)
                .map(PasswordUsageCount::getUsageCount)
                .orElse(0);
            
            if (count >= 100) {
                throw new InvalidPasswordException("Common password");
            }
        }
    }
}
```

## 9. 성능 벤치마크

### 캐시 적용 전후 비교

| 시나리오 | 캐시 미적용 | Redis 캐시 | Bloom Filter + Redis |
|---------|-----------|-----------|---------------------|
| 일반 비밀번호 검증 | 50ms | 5ms | 0.5ms |
| 흔한 비밀번호 검증 | 50ms | 5ms | 5ms |
| 캐시 히트율 | - | 85% | 95% |
| DB 쿼리 감소율 | - | 85% | 95% |

### 예상 성능 개선

```
1억 계정 기준:
- 일일 비밀번호 변경 요청: 10만 건
- 캐시 미적용: 50ms × 100,000 = 5,000초 (83분)
- 캐시 적용: 5ms × 100,000 = 500초 (8분)
- Bloom Filter + 캐시: 0.5ms × 95,000 + 5ms × 5,000 = 72초

약 70배 성능 향상
```
