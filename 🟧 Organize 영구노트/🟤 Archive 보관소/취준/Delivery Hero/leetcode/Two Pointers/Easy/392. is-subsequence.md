#String #TwoPointers #DynamicProgramming

## 풀이 코드
```java
class Solution {
    public boolean isSubsequence(String s, String t) {
        // 1. s가 빈 문자열이면 true 반환
        // why: 빈 문자열은 모든 문자열의 subsequence
        if (s.isEmpty()) return true;
        
        // 2. s와 t를 순회할 포인터 초기화
        // why: s의 각 문자를 t에서 순서대로 찾기 위함
        int sIndex = 0;
        int tIndex = 0;
        
        // 3. t를 순회하며 s의 문자를 순서대로 찾음
        // why: t에서 s의 모든 문자를 순서대로 찾으면 subsequence
        while (tIndex < t.length() && sIndex < s.length()) {
            // 4. 현재 위치의 문자가 일치하면 s의 포인터 이동
            // why: s의 다음 문자를 찾기 위해 진행
            if (s.charAt(sIndex) == t.charAt(tIndex)) {
                sIndex++;
            }
            // 5. t의 포인터는 항상 이동
            // why: t를 계속 탐색하며 s의 다음 문자를 찾음
            tIndex++;
        }
        
        // 6. s의 모든 문자를 찾았는지 확인
        // why: sIndex가 s.length()와 같으면 모든 문자를 순서대로 찾은 것
        return sIndex == s.length();
    }
}
```

**시간 복잡도**: O(n) - t의 길이만큼 순회 (n은 t의 길이)  
**공간 복잡도**: O(1) - 두 개의 포인터 변수만 사용

## 면접 대본 (영어)

"Let me explain my approach to checking if one string is a subsequence of another.

The key insight is that a subsequence maintains the relative order of characters, but doesn't require them to be consecutive. So I need to traverse through the longer string t and match characters with s in order, but I can skip characters in t that don't match.

I'll use a two-pointer technique. One pointer, sIndex, tracks my position in string s - the string I'm trying to match. The other pointer, tIndex, tracks my position in string t - the string I'm searching through.

First, I handle an edge case: if s is empty, it's automatically a subsequence of any string, so I return true.

Here's how the algorithm works: I iterate through string t using tIndex. At each position, I check if the current character in t matches the character I'm looking for in s, which is at position sIndex. If they match, it means I've found the next character of my subsequence, so I increment sIndex to look for the next character. Regardless of whether there's a match or not, I always increment tIndex to move forward in t and continue searching.

The process continues until either I've matched all characters in s, or I've exhausted string t. At the end, if sIndex equals the length of s, it means I successfully found all characters of s in t in the correct order, so s is a subsequence of t.

Let me walk through an example: if we have s = 'abc' and t = 'ahbgdc', initially both pointers are at 0. At t[0]='a', it matches s[0]='a', so sIndex becomes 1. At t[1]='h', it doesn't match s[1]='b', so only tIndex moves. At t[2]='b', it matches s[1]='b', so sIndex becomes 2. At t[3]='g', no match, tIndex moves. At t[4]='d', no match. At t[5]='c', it matches s[2]='c', so sIndex becomes 3. We've reached the end of s, so we return true.

Another example: if we have s = 'axc' and t = 'ahbgdc', we match 'a' at position 0. But after that, we search through the entire remaining string and never find 'x', so sIndex only reaches 1, not 3, and we return false.

The time complexity is O(n) where n is the length of t, since we traverse t at most once. The space complexity is O(1) because we only use two integer pointers regardless of input size. This solution is optimal for a single query.

For the follow-up question about checking many strings against the same t, I would preprocess t by creating a map where each character maps to a list of all its indices in t. Then for each query string, I could use binary search on these index lists to find the next occurrence of each character, which would be more efficient for multiple queries."
