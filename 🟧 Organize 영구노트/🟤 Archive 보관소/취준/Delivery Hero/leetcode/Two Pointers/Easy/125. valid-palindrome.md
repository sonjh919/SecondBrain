#String #TwoPointers

## 풀이 코드
```java
class Solution {
    public boolean isPalindrome(String s) {
        // 1. 양 끝에서 시작하는 두 포인터 초기화
        // why: 문자열의 양 끝에서 중앙으로 이동하며 비교하는 Two Pointer 기법 사용
        int left = 0;
        int right = s.length() - 1;
        
        // 2. 두 포인터가 교차할 때까지 반복
        // why: 양쪽에서 중앙으로 이동하며 모든 문자 쌍을 비교
        while (left < right) {
            // 3. 왼쪽 포인터가 알파벳/숫자가 아니면 오른쪽으로 이동
            // why: 알파벳과 숫자만 비교 대상이므로 다른 문자는 건너뜀
            while (left < right && !Character.isLetterOrDigit(s.charAt(left))) {
                left++;
            }
            
            // 4. 오른쪽 포인터가 알파벳/숫자가 아니면 왼쪽으로 이동
            // why: 알파벳과 숫자만 비교 대상이므로 다른 문자는 건너뜀
            while (left < right && !Character.isLetterOrDigit(s.charAt(right))) {
                right--;
            }
            
            // 5. 양쪽 문자를 소문자로 변환하여 비교
            // why: 대소문자 구분 없이 비교해야 하므로 소문자로 통일
            if (Character.toLowerCase(s.charAt(left)) != Character.toLowerCase(s.charAt(right))) {
                return false;
            }
            
            // 6. 일치하면 양쪽 포인터를 중앙으로 이동
            // why: 다음 문자 쌍을 비교하기 위함
            left++;
            right--;
        }
        
        // 7. 모든 비교를 통과하면 팰린드롬
        // why: 양쪽에서 비교한 모든 문자가 일치하면 팰린드롬
        return true;
    }
}
```

**시간 복잡도**: O(n) - 문자열을 한 번 순회 (n은 문자열 길이)  
**공간 복잡도**: O(1) - 두 개의 포인터 변수만 사용

## 면접 대본 (영어)

"Let me explain my approach to validating whether a string is a palindrome.

The key insight is to use a two-pointer technique where we compare characters from both ends of the string moving towards the center, while ignoring non-alphanumeric characters and treating uppercase and lowercase letters as the same.

I'll use two pointers: left starting at the beginning of the string and right starting at the end. The algorithm works by moving these pointers towards each other.

Here's how it works: At each step, I first skip any non-alphanumeric characters on the left side by incrementing the left pointer until I find a letter or digit. Similarly, I skip non-alphanumeric characters on the right side by decrementing the right pointer. Once both pointers are at alphanumeric characters, I compare them after converting both to lowercase. If they don't match, the string is not a palindrome, so I return false immediately. If they match, I move both pointers - left increments and right decrements - and continue the process.

I keep doing this until the pointers meet or cross each other, which means I've successfully compared all valid character pairs, so the string is a palindrome.

Let me walk through an example: if we have s = 'A man, a plan, a canal: Panama', left starts at 0 and right at the last index. Left is at 'A', right is at 'a'. Converting to lowercase, both are 'a', they match, so we move on. We skip the space and comma, then compare 'm' with 'm' - they match. We continue this process, skipping all punctuation and spaces, comparing only letters. Eventually all pairs match, so we return true.

Another example: if we have s = 'race a car', we start comparing: 'r' with 'r' matches, 'a' with 'a' matches, 'c' with 'c' matches, 'e' with 'a' - they don't match, so we immediately return false.

The time complexity is O(n) where n is the length of the string, because we visit each character at most once. The space complexity is O(1) because we only use two pointer variables regardless of input size. This solution is efficient because it doesn't create any new strings - we work directly on the original string, which is important for memory efficiency with large inputs."

## 핵심 아이디어

Two Pointer로 양 끝에서 중앙으로 이동하며 알파벳/숫자만 소문자로 변환하여 비교하면 O(n) 시간에 팰린드롬 판별 가능
