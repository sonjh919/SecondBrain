#LinkedList #HashTable #TwoPointers

## 풀이 코드
```java
/**
 * Definition for singly-linked list.
 * class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) {
 *         val = x;
 *         next = null;
 *     }
 * }
 */
public class Solution {
    public boolean hasCycle(ListNode head) {
        // 1. 빈 리스트나 노드가 하나면 사이클 불가능
        // why: 사이클을 만들 수 없는 경우 조기 반환
        if (head == null || head.next == null) {
            return false;
        }
        
        // 2. Floyd's Cycle Detection - 느린 포인터와 빠른 포인터 초기화
        // why: 토끼와 거북이 알고리즘으로 O(1) 공간에 사이클 감지
        ListNode slow = head;
        ListNode fast = head;
        
        // 3. 빠른 포인터가 끝에 도달할 때까지 이동
        // why: 사이클이 없으면 fast가 null에 도달, 있으면 slow와 만남
        while (fast != null && fast.next != null) {
            slow = slow.next;           // 한 칸 이동
            fast = fast.next.next;      // 두 칸 이동
            
            // 4. 두 포인터가 만나면 사이클 존재
            // why: 사이클이 있으면 빠른 포인터가 결국 느린 포인터를 따라잡음
            if (slow == fast) {
                return true;
            }
        }
        
        // 5. fast가 끝에 도달하면 사이클 없음
        // why: 정상적으로 리스트의 끝에 도달한 경우
        return false;
    }
}
```

**시간 복잡도**: O(n) - 최악의 경우 리스트의 모든 노드를 방문 (n은 노드 개수)  
**공간 복잡도**: O(1) - 두 개의 포인터만 사용

## 면접 대본 (영어)

"Let me explain my approach to detecting a cycle in a linked list using Floyd's Cycle Detection Algorithm, also known as the tortoise and hare algorithm.

The key insight is that if we have two pointers moving at different speeds through the list, they will eventually meet if there's a cycle. Think of it like a race track - if two runners are on a circular track and one runs twice as fast as the other, the faster runner will eventually lap the slower one.

I'll use two pointers: a slow pointer that moves one step at a time, and a fast pointer that moves two steps at a time. If there's a cycle, these pointers will eventually point to the same node. If there's no cycle, the fast pointer will reach the end of the list.

First, I handle edge cases: if the list is empty or has only one node, there can't be a cycle, so I return false immediately.

Here's how the algorithm works: I initialize both pointers at the head. Then I enter a loop that continues as long as the fast pointer and fast.next are not null - I need to check both because fast takes two steps at a time. In each iteration, slow moves one node forward, and fast moves two nodes forward. After each move, I check if slow equals fast. If they're pointing to the same node, I've detected a cycle and return true.

If the loop terminates naturally - meaning fast reached null or fast.next reached null - it means there's no cycle, so I return false.

Let me walk through an example: suppose we have a list [3,2,0,-4] where -4 points back to 2, forming a cycle. Initially, slow and fast both point to 3. First iteration: slow moves to 2, fast moves to 0. Second iteration: slow moves to 0, fast moves to 2. Third iteration: slow moves to -4, fast moves to -4. Now slow equals fast, so we return true.

Another example: a list [1,2] with no cycle. Initially, both pointers are at 1. First iteration: slow moves to 2, fast moves to null (trying to go beyond 2). The loop condition fails because fast is null, so we return false.

An interesting case to consider: what if there's a cycle but the pointers start at different positions relative to it? The algorithm still works. Once both pointers enter the cycle, fast is catching up to slow at a rate of one node per iteration. Eventually they must meet because the cycle has finite length.

The time complexity is O(n) where n is the number of nodes. In the worst case without a cycle, fast traverses the entire list. With a cycle, we can prove that slow will be caught within n steps. The space complexity is O(1) because we only use two pointer variables regardless of the list size.

This solution elegantly satisfies the follow-up requirement of using constant memory. An alternative approach would use a HashSet to track visited nodes, which would also work but use O(n) space. The two-pointer approach is preferred because it achieves the same result with O(1) space."
