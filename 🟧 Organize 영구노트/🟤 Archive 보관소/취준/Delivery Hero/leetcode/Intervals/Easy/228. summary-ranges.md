## 풀이 코드
```java
class Solution {
    public List<String> summaryRanges(int[] nums) {
        // 1. 결과를 저장할 리스트 생성
        // why: 각 범위를 문자열로 변환하여 저장
        List<String> result = new ArrayList<>();
        
        // 2. 빈 배열이면 빈 리스트 반환
        // why: 범위를 만들 숫자가 없으면 조기 반환
        if (nums.length == 0) return result;
        
        // 3. 배열을 순회하며 연속된 범위 찾기
        // why: 현재 범위의 시작점을 추적하며 끊어지는 지점에서 범위 추가
        int start = nums[0];
        
        for (int i = 1; i < nums.length; i++) {
            // 4. 현재 숫자가 이전 숫자 + 1이 아니면 범위 끝
            // why: 연속되지 않으면 이전 범위를 완성하고 새 범위 시작
            if (nums[i] != nums[i - 1] + 1) {
                // 5. 범위를 문자열로 변환하여 결과에 추가
                // why: start와 이전 숫자로 범위를 표현
                result.add(formatRange(start, nums[i - 1]));
                start = nums[i];
            }
        }
        
        // 6. 마지막 범위를 결과에 추가
        // why: 루프가 끝나면 마지막으로 진행 중이던 범위를 추가해야 함
        result.add(formatRange(start, nums[nums.length - 1]));
        
        return result;
    }
    
    // 7. 범위를 문자열로 포맷하는 헬퍼 함수
    // why: start와 end가 같으면 단일 숫자, 다르면 "start->end" 형식
    private String formatRange(int start, int end) {
        return start == end ? String.valueOf(start) : start + "->" + end;
    }
}
```

**시간 복잡도**: O(n) - 배열을 한 번 순회 (n은 배열 길이)  
**공간 복잡도**: O(1) - 결과 리스트 제외 시 추가 공간 사용 없음

## 면접 대본 (영어)

"Let me explain my approach to summarizing ranges in a sorted array.

The key insight is that since the array is sorted and contains unique integers, I can identify continuous ranges by checking if each number is exactly one more than the previous number. When this continuity breaks, I know a range has ended and a new one begins.

I'll track the start of the current range and check for breaks in continuity as I traverse the array.

First, I handle the edge case of an empty array by returning an empty list immediately.

Here's how the algorithm works: I initialize start with the first element, which marks the beginning of the first range. Then I iterate through the array starting from index 1. At each position, I check if the current number equals the previous number plus one. If it does, the range continues, so I just move to the next element. If it doesn't, it means the range has ended at the previous number.

When a range ends, I format it as a string. If start equals the end value, the range contains only one number, so I output just that number. If start is different from the end, I output the range as 'start->end'. After adding the range to my result list, I update start to the current number, which begins a new range.

After the loop completes, I need to handle the last range that was being built, since the loop ends without adding it. I add this final range using the start value and the last element of the array.

Let me walk through an example: if we have nums = [0,1,2,4,5,7], start begins at 0. At i=1, we have 1 which is 0+1, so we continue. At i=2, we have 2 which is 1+1, so we continue. At i=3, we have 4 which is not 2+1, so the range breaks. We add '0->2' to our result and set start to 4. At i=4, we have 5 which is 4+1, so we continue. At i=5, we have 7 which is not 5+1, so we add '4->5' and set start to 7. After the loop, we add the final range '7' (since 7==7, it's a single number). Our result is ['0->2', '4->5', '7'].

Another example: nums = [0,2,3,4,6,8,9]. Start is 0. At i=1, 2 is not 0+1, so we add '0' and start becomes 2. At i=2, 3 is 2+1, continue. At i=3, 4 is 3+1, continue. At i=4, 6 is not 4+1, so we add '2->4' and start becomes 6. At i=5, 8 is not 6+1, so we add '6' and start becomes 8. At i=6, 9 is 8+1, continue. After the loop, we add '8->9'. Result is ['0', '2->4', '6', '8->9'].

The time complexity is O(n) where n is the length of the array, since we make a single pass through it. The space complexity is O(1) if we don't count the output list, as we only use a constant amount of extra space for tracking the start variable. This solution efficiently handles all cases including single-element ranges and multi-element ranges."

## 핵심 아이디어
