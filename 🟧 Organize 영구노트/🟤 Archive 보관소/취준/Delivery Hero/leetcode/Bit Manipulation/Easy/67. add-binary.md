## 풀이 코드
```java
class Solution {
    public String addBinary(String a, String b) {
        // 1. 결과를 저장할 StringBuilder와 carry 초기화
        // why: 뒤에서부터 더하므로 StringBuilder로 효율적 문자열 조작
        StringBuilder result = new StringBuilder();
        int carry = 0;
        
        // 2. 두 문자열의 끝에서부터 시작하는 포인터 초기화
        // why: 이진수 덧셈은 가장 낮은 자리부터 시작
        int i = a.length() - 1;
        int j = b.length() - 1;
        
        // 3. 두 문자열을 모두 처리하거나 carry가 남아있을 때까지 반복
        // why: 모든 자리를 처리하고 마지막 carry도 처리해야 함
        while (i >= 0 || j >= 0 || carry > 0) {
            int sum = carry;
            
            // 4. a의 현재 자리 값을 더함
            // why: i가 유효한 범위에 있으면 해당 자리값 추가
            if (i >= 0) {
                sum += a.charAt(i) - '0';
                i--;
            }
            
            // 5. b의 현재 자리 값을 더함
            // why: j가 유효한 범위에 있으면 해당 자리값 추가
            if (j >= 0) {
                sum += b.charAt(j) - '0';
                j--;
            }
            
            // 6. 현재 자리의 결과값(sum % 2)을 결과에 추가
            // why: 이진수에서 2로 나눈 나머지가 현재 자리의 값
            result.append(sum % 2);
            
            // 7. 다음 자리로 올림수(carry) 계산
            // why: sum이 2 이상이면 다음 자리로 1을 올림
            carry = sum / 2;
        }
        
        // 8. 역순으로 쌓인 결과를 뒤집어서 반환
        // why: 낮은 자리부터 append했으므로 reverse 필요
        return result.reverse().toString();
    }
}
```

**시간 복잡도**: O(max(n, m)) - 두 문자열 중 긴 쪽의 길이만큼 순회 (n, m은 각 문자열 길이)  
**공간 복잡도**: O(max(n, m)) - 결과 문자열의 길이 (최대 max(n,m) + 1)

## 면접 대본 (영어)

"Let me explain my approach to adding two binary strings.

The key insight is to simulate the process of binary addition by hand, just like we would add decimal numbers. We start from the rightmost digit, add the corresponding digits along with any carry from the previous position, and work our way left.

I'll use a StringBuilder to efficiently build the result string, since we'll be appending characters one by one. I also maintain a carry variable to track whether we need to carry over a 1 to the next position.

I initialize two pointers, i and j, pointing to the last characters of strings a and b respectively. These will move leftward as we process each digit.

The main loop continues while there are still digits to process in either string, or while there's a carry remaining. In each iteration, I start with sum equal to the current carry. Then, if pointer i is still valid, I add the digit from string a (converting the character to an integer by subtracting '0'), and decrement i. Similarly for string b with pointer j.

After summing the digits and carry, I append the result digit to the StringBuilder using sum modulo 2 - this gives us 0 or 1, which is the current position's value in binary. Then I calculate the new carry as sum divided by 2 - this will be 1 if we need to carry, or 0 otherwise.

Finally, after processing all digits, I reverse the StringBuilder since I've been appending from right to left, and convert it to a string.

Let me walk through an example: a = '11', b = '1'. I start with i=1, j=0, carry=0. First iteration: sum = 0 + 1 + 1 = 2. Append 2%2=0, carry becomes 2/2=1. Now i=0, j=-1. Second iteration: sum = 1 + 1 = 2. Append 0, carry becomes 1. Now i=-1, j=-1, but carry is 1. Third iteration: sum = 1. Append 1, carry becomes 0. Result is '001' backwards, so we reverse to get '100'.

Another example: a = '1010', b = '1011'. Starting from the right: 0+1=1, append 1, carry 0. Then 1+1=2, append 0, carry 1. Then 0+0+1=1, append 1, carry 0. Then 1+1=2, append 0, carry 1. Finally, carry 1 remains, append 1. Result is '10101' after reversing.

An important edge case: what if one string is much longer than the other? The algorithm handles this naturally - when one pointer goes negative, we simply stop adding from that string and continue with the other string plus any carry. For example, a='1111', b='1' would work correctly because we continue processing a's remaining digits even after b is exhausted.

Another edge case: both strings are '1'. Sum is 2, append 0, carry 1. Then carry 1 remains, append 1. Result is '10' after reversing, which is correct.

The time complexity is O(max(n, m)) where n and m are the lengths of the two strings, because we process each digit once. The space complexity is also O(max(n, m)) for the StringBuilder that stores the result, which can be at most one digit longer than the longer input string.

This solution elegantly simulates manual binary addition and handles all edge cases including different string lengths and carries. It's similar to how we'd solve decimal addition, just with base 2 instead of base 10."
