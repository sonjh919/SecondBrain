#Array #BitManipulation

## 풀이 코드
```java
class Solution {
    public int singleNumber(int[] nums) {
        // 1. XOR 연산의 결과를 저장할 변수 초기화
        // why: XOR의 항등원은 0이므로 0으로 시작
        int result = 0;
        
        // 2. 배열의 모든 원소를 XOR 연산
        // why: 같은 숫자끼리 XOR하면 0이 되고, 0과 XOR하면 원래 값이 됨
        for (int num : nums) {
            result ^= num;
        }
        
        // 3. 한 번만 나타나는 숫자 반환
        // why: 모든 쌍은 상쇄되고 단독으로 나타나는 숫자만 남음
        return result;
    }
}
```

**시간 복잡도**: O(n) - 배열을 한 번 순회 (n은 배열 길이)  
**공간 복잡도**: O(1) - result 변수만 사용

## 면접 대본 (영어)

"Let me explain my approach to finding the single number that appears only once while all others appear twice.

The key insight is to use the XOR (exclusive OR) bitwise operation, which has several useful properties that make this problem elegant. First, XOR is commutative and associative, meaning order doesn't matter. Second, any number XORed with itself equals zero: a ⊕ a = 0. Third, any number XORed with zero equals itself: a ⊕ 0 = a. These properties combined mean that if we XOR all numbers together, the pairs will cancel out, leaving only the single number.

The algorithm is remarkably simple: I initialize a result variable to 0. Then I iterate through the entire array, XORing each number with result. After processing all numbers, result contains the answer.

Let me explain why this works mathematically. Suppose we have numbers [a, b, a, c, b]. When we XOR them all together: a ⊕ b ⊕ a ⊕ c ⊕ b. Because XOR is commutative, we can rearrange: (a ⊕ a) ⊕ (b ⊕ b) ⊕ c = 0 ⊕ 0 ⊕ c = c. All the pairs cancel out to zero, and we're left with the single number.

Let me walk through an example: nums = [4,1,2,1,2]. Starting with result = 0. XOR with 4: result = 4. XOR with 1: result = 4 ⊕ 1 = 5. XOR with 2: result = 5 ⊕ 2 = 7. XOR with 1: result = 7 ⊕ 1 = 6. XOR with 2: result = 6 ⊕ 2 = 4. The final result is 4, which is correct.

Let me verify the bit manipulation: 4 is 100, 1 is 001, 2 is 010. Starting: 000. After 4: 100. After first 1: 101. After first 2: 111. After second 1: 110. After second 2: 100. We end with 100, which is 4.

Another example: nums = [2,2,1]. Result starts at 0. XOR with first 2: result = 2. XOR with second 2: result = 2 ⊕ 2 = 0. XOR with 1: result = 0 ⊕ 1 = 1. Final result is 1.

The beauty of this solution is its simplicity and efficiency. We don't need any extra data structures like HashSets or HashMaps, which would use O(n) space. We don't need to sort the array, which would take O(n log n) time. We simply make one pass through the array with a single XOR operation per element.

The time complexity is O(n) where n is the length of the array, because we visit each element exactly once. The space complexity is O(1) because we only use one integer variable for the result, regardless of input size.

This problem beautifully demonstrates how understanding bitwise operations can lead to elegant solutions. While you could solve this with a HashMap to count occurrences, that would use O(n) space. The XOR approach achieves the same result with constant space by leveraging mathematical properties of the XOR operation.

An interesting extension: what if three elements appear twice and one appears once? The XOR approach still works. What if every element appears three times except one? Then we'd need a different approach, typically using bit counting at each position. But for the two-appearance case, XOR is perfect."
