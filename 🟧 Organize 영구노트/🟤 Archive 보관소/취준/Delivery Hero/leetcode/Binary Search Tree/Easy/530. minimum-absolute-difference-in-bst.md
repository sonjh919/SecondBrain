#Tree #DFS #BinarySearchTree #BinaryTree

## 풀이 코드
```java
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
    // 1. 이전 노드의 값과 최소 차이를 추적할 전역 변수
    // why: inorder 순회 중 이전 값과 비교하기 위해 상태 유지 필요
    private Integer prev = null;
    private int minDiff = Integer.MAX_VALUE;
    
    public int getMinimumDifference(TreeNode root) {
        // 2. inorder 순회로 BST를 정렬된 순서로 방문
        // why: BST의 inorder 순회는 오름차순으로 노드를 방문
        inorder(root);
        return minDiff;
    }
    
    private void inorder(TreeNode node) {
        // 3. 빈 노드면 종료
        // why: 재귀의 base case
        if (node == null) {
            return;
        }
        
        // 4. 왼쪽 서브트리 방문
        // why: inorder 순회는 왼쪽 → 현재 → 오른쪽 순서
        inorder(node.left);
        
        // 5. 이전 값이 있으면 현재 값과의 차이 계산
        // why: 정렬된 순서로 방문하므로 인접한 두 값의 차이가 최소 후보
        if (prev != null) {
            minDiff = Math.min(minDiff, node.val - prev);
        }
        
        // 6. 현재 노드 값을 이전 값으로 업데이트
        // why: 다음 노드와 비교하기 위해 현재 값 저장
        prev = node.val;
        
        // 7. 오른쪽 서브트리 방문
        // why: inorder 순회 완성
        inorder(node.right);
    }
}
```

**시간 복잡도**: O(n) - 모든 노드를 한 번씩 방문 (n은 노드 개수)  
**공간 복잡도**: O(h) - 재귀 호출 스택의 깊이 (h는 트리 높이)

## 면접 대본 (영어)

"Let me explain my approach to finding the minimum absolute difference in a Binary Search Tree.

The key insight is to leverage the BST property. In a BST, an inorder traversal visits nodes in sorted ascending order. This means that the minimum difference between any two nodes must occur between two consecutive nodes in the inorder traversal. We don't need to compare all pairs of nodes - just adjacent ones in sorted order.

I'll use inorder traversal with two instance variables to track state. The prev variable stores the value of the previously visited node, and minDiff tracks the minimum difference found so far.

Here's how the algorithm works: I perform a standard inorder traversal - left subtree first, then current node, then right subtree. The crucial part happens when visiting the current node. If prev is not null, meaning we've visited at least one node before, I calculate the difference between the current node's value and prev. Since we're traversing in sorted order, this difference is always positive, and I can simply do node.val minus prev. I update minDiff if this difference is smaller.

After calculating the difference, I update prev to the current node's value before moving to the right subtree. This ensures that when we visit the next node, prev will hold the value of the current node.

Let me walk through an example: root = [4,2,6,1,3]. The inorder traversal visits nodes in order: 1, 2, 3, 4, 6. Starting with node 1, prev is null, so we just set prev to 1. At node 2, prev is 1, difference is 2-1=1, minDiff becomes 1, prev becomes 2. At node 3, prev is 2, difference is 3-2=1, minDiff stays 1, prev becomes 3. At node 4, prev is 3, difference is 4-3=1, minDiff stays 1, prev becomes 4. At node 6, prev is 4, difference is 6-4=2, minDiff stays 1. Final answer is 1.

Another example: root = [1,0,48,null,null,12,49]. Inorder gives us: 0, 1, 12, 48, 49. At 1, difference from 0 is 1. At 12, difference from 1 is 11. At 48, difference from 12 is 36. At 49, difference from 48 is 1. The minimum is 1.

An important detail: I use Integer instead of int for prev because I need to distinguish between 'no previous value' and 'previous value is 0'. Using null as the initial value makes this distinction clear.

Why does this work? In a BST, for any two nodes, if they're not adjacent in the inorder traversal, there must be at least one node between them. The difference between non-adjacent nodes will always be larger than at least one pair of adjacent nodes. Therefore, we only need to check adjacent pairs.

The time complexity is O(n) where n is the number of nodes, because we visit each node exactly once during the inorder traversal. The space complexity is O(h) where h is the height of the tree, due to the recursion stack. In a balanced BST, this is O(log n), but in a skewed tree, it could be O(n).

This solution elegantly combines the BST property with inorder traversal to achieve an optimal solution. We avoid the O(n²) approach of comparing all pairs, and we don't need to store all values in an array first."
