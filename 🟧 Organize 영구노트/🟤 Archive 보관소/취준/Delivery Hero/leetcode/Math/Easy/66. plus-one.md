## 풀이 코드
```java
class Solution {
    public int[] plusOne(int[] digits) {
        // 1. 배열을 뒤에서부터 순회
        // why: 덧셈은 가장 낮은 자리부터 시작하여 올림수를 처리
        for (int i = digits.length - 1; i >= 0; i--) {
            // 2. 현재 자리가 9가 아니면 1을 더하고 바로 반환
            // why: 올림이 없으므로 더 이상 처리할 필요 없음
            if (digits[i] < 9) {
                digits[i]++;
                return digits;
            }
            
            // 3. 현재 자리가 9이면 0으로 만들고 계속 진행
            // why: 9+1=10이므로 현재 자리는 0이 되고 다음 자리로 올림
            digits[i] = 0;
        }
        
        // 4. 모든 자리가 9였던 경우 새로운 배열 생성
        // why: 999...9 + 1 = 1000...0이므로 자릿수가 하나 증가
        int[] result = new int[digits.length + 1];
        result[0] = 1;
        // 나머지는 자동으로 0으로 초기화됨
        
        return result;
    }
}
```

**시간 복잡도**: O(n) - 최악의 경우 배열 전체를 순회 (n은 배열 길이)  
**공간 복잡도**: O(1) - 새 배열 생성이 필요한 경우만 O(n), 입력 배열 수정 시 O(1)

## 면접 대본 (영어)

"Let me explain my approach to incrementing a large integer represented as an array of digits.

The key insight is to simulate manual addition starting from the rightmost digit. When we add 1 to the least significant digit, if it's less than 9, we simply increment it and we're done. If it's 9, it becomes 0 and we carry 1 to the next digit. This process continues until we either find a digit less than 9, or we've carried through all digits.

I'll iterate through the array from right to left, handling the carry as we go.

Here's how the algorithm works: I start from the last index and work backwards. At each position, I check if the current digit is less than 9. If it is, I can simply increment it and return the array immediately - there's no carry, so we're done.

If the digit is 9, it means adding 1 creates a carry. I set the current digit to 0 and continue to the next iteration, which will handle the carry by incrementing the next digit.

There's a special case: if we've gone through all digits and they were all 9s (like 999), we've exited the loop without returning. This means we need an extra digit at the front. I create a new array with length + 1, set the first element to 1, and return it. The remaining elements are automatically initialized to 0, which is exactly what we need.

Let me walk through an example: digits = [1,2,3]. Starting from index 2, digit is 3, which is less than 9. I increment it to 4 and return [1,2,4]. Simple and efficient.

Another example: digits = [1,9,9]. At index 2, digit is 9, so I set it to 0 and continue. At index 1, digit is 9, so I set it to 0 and continue. At index 0, digit is 1, less than 9, so I increment it to 2 and return [2,0,0].

The edge case: digits = [9,9,9]. At index 2, digit is 9, set to 0. At index 1, digit is 9, set to 0. At index 0, digit is 9, set to 0. Now we've exited the loop with digits = [0,0,0], but we haven't returned. This triggers our special case: create new array [1,0,0,0] and return it. This correctly represents 999 + 1 = 1000.

Another interesting case: digits = [9]. It's 9, set to 0, exit loop without returning, create [1,0]. Correct: 9 + 1 = 10.

Why does this work so elegantly? The key observation is that once we find a digit less than 9, all the carrying is done - we can immediately return. And if all digits are 9, we know the result must be 1 followed by all zeros, which is exactly what a new zero-initialized array with 1 at the front gives us.

The time complexity is O(n) where n is the length of the array. In the best case where the last digit isn't 9, it's O(1). In the worst case where all digits are 9, we visit every digit once and then create a new array. The space complexity is O(1) if we modify the input array in place and don't need the extra digit. It's O(n) only in the special case where we need to create a new array.

This solution demonstrates understanding of how carry propagation works in addition, and how to handle it efficiently without explicitly tracking a carry variable. The elegance lies in recognizing that we can stop as soon as we don't need to carry anymore, and in the clever handling of the all-9s case."
