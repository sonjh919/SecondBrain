## 풀이 코드
```java
class Solution {
    public int climbStairs(int n) {
        // 1. n이 1이나 2인 경우 직접 반환
        // why: base case로 n=1은 1가지, n=2는 2가지 방법
        if (n <= 2) {
            return n;
        }
        
        // 2. 이전 두 단계의 경우의 수를 저장할 변수 초기화
        // why: 피보나치 수열처럼 이전 두 값만 필요
        int prev2 = 1;  // n=1일 때
        int prev1 = 2;  // n=2일 때
        
        // 3. n=3부터 n까지 반복하며 경우의 수 계산
        // why: 현재 계단은 (n-1)계단이나 (n-2)계단에서 올 수 있음
        for (int i = 3; i <= n; i++) {
            int current = prev1 + prev2;
            prev2 = prev1;
            prev1 = current;
        }
        
        // 4. n번째 계단까지 가는 경우의 수 반환
        // why: prev1이 최종 답을 저장하고 있음
        return prev1;
    }
}
```

**시간 복잡도**: O(n) - n까지 반복하며 계산  
**공간 복잡도**: O(1) - 두 개의 변수만 사용

## 면접 대본 (영어)

"Let me explain my approach to solving the climbing stairs problem.

The key insight is to recognize this as a dynamic programming problem that follows the Fibonacci sequence. To reach step n, you can only come from step n-1 (by taking 1 step) or from step n-2 (by taking 2 steps). Therefore, the number of ways to reach step n is the sum of ways to reach n-1 and n-2.

This is essentially the Fibonacci sequence! If we denote f(n) as the number of ways to reach step n, then f(n) = f(n-1) + f(n-2), with base cases f(1) = 1 and f(2) = 2.

I'll use an iterative approach with constant space. Instead of storing all previous values in an array, I only need to track the last two values since each step only depends on the previous two.

First, I handle the base cases: if n is 1 or 2, I return n directly. For n=1, there's only one way (take 1 step). For n=2, there are two ways (1+1 or 2).

For n >= 3, I initialize prev2 to 1 (representing f(1)) and prev1 to 2 (representing f(2)). Then I iterate from 3 to n. In each iteration, I calculate the current value as prev1 + prev2, then update prev2 to prev1 and prev1 to current. This sliding window approach maintains only the two most recent values.

Let me walk through an example: n = 5. Base values: prev2 = 1, prev1 = 2. Iteration i=3: current = 2+1 = 3, prev2 = 2, prev1 = 3. Iteration i=4: current = 3+2 = 5, prev2 = 3, prev1 = 5. Iteration i=5: current = 5+3 = 8, prev2 = 5, prev1 = 8. Return 8.

Let's verify this is correct for n=5: We can reach step 5 from step 4 (5 ways) or step 3 (3 ways), so 5+3 = 8 ways total. The actual sequences are: 1+1+1+1+1, 1+1+1+2, 1+1+2+1, 1+2+1+1, 2+1+1+1, 1+2+2, 2+1+2, 2+2+1. That's indeed 8 ways.

Another example: n = 3. Prev2 = 1, prev1 = 2. Iteration i=3: current = 2+1 = 3. Return 3. The three ways are: 1+1+1, 1+2, 2+1.

Why does this work? At any step n, you must have arrived from either step n-1 or step n-2 - there's no other way. So the total ways to reach n is simply the sum of ways to reach those two previous steps. This recursive relationship defines a recurrence relation that we solve bottom-up.

The time complexity is O(n) because we iterate from 3 to n once, performing constant work in each iteration. The space complexity is O(1) because we only use three variables regardless of n.

An alternative approach would be to use recursion with memoization, which would also be O(n) time but O(n) space for the memoization table. Or we could use an array to store all values, which would be O(n) space. But the iterative approach with constant space is optimal.

Interestingly, since this is the Fibonacci sequence, we could also use the closed-form formula (Binet's formula) involving the golden ratio to compute it in O(1) time, but that involves floating-point arithmetic and can have precision issues for large n. The iterative approach is more reliable.

This problem beautifully demonstrates how recognizing a recurrence relation can transform what seems like a combinatorial problem into a simple dynamic programming solution. The insight that you can only arrive from the previous two steps is the key to unlocking the Fibonacci pattern."
