## 풀이 코드
```java
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
    public boolean isSymmetric(TreeNode root) {
        // 1. 루트가 null이면 대칭
        // why: 빈 트리는 자동으로 대칭
        if (root == null) {
            return true;
        }
        
        // 2. 왼쪽과 오른쪽 서브트리가 거울 이미지인지 확인
        // why: 대칭 트리는 왼쪽과 오른쪽이 서로 거울상
        return isMirror(root.left, root.right);
    }
    
    // 3. 두 트리가 거울 이미지인지 확인하는 헬퍼 함수
    // why: 재귀적으로 대칭성을 검사하기 위한 별도 함수
    private boolean isMirror(TreeNode left, TreeNode right) {
        // 4. 둘 다 null이면 대칭
        // why: 양쪽이 모두 비어있으면 거울상
        if (left == null && right == null) {
            return true;
        }
        
        // 5. 하나만 null이면 비대칭
        // why: 한쪽만 노드가 있으면 거울상이 아님
        if (left == null || right == null) {
            return false;
        }
        
        // 6. 현재 노드의 값이 같고, 교차된 서브트리들이 거울 이미지인지 확인
        // why: left.left와 right.right, left.right와 right.left가 거울상이어야 함
        return (left.val == right.val) 
            && isMirror(left.left, right.right) 
            && isMirror(left.right, right.left);
    }
}
```

**시간 복잡도**: O(n) - 모든 노드를 한 번씩 방문 (n은 노드 개수)  
**공간 복잡도**: O(h) - 재귀 호출 스택의 깊이 (h는 트리 높이, 최악의 경우 O(n))

## 면접 대본 (영어)

"Let me explain my approach to checking if a binary tree is symmetric.

The key insight is that a tree is symmetric if its left subtree is a mirror image of its right subtree. Two trees are mirror images if their root values are equal, and the left subtree of one is a mirror of the right subtree of the other, and vice versa.

I'll solve this recursively using a helper function that checks if two trees are mirrors of each other. The main function simply calls this helper with the left and right children of the root.

First, I handle the edge case where the root is null - an empty tree is considered symmetric by definition.

The helper function isMirror takes two nodes and checks if they form mirror images. The base cases are: if both nodes are null, they're mirrors of each other - two empty trees are symmetric. If only one is null, they're not mirrors - one side has a node where the other doesn't.

For the recursive case, I need to check three conditions: first, the values at both nodes must be equal. Second, the left subtree of the first node must be a mirror of the right subtree of the second node. Third, the right subtree of the first node must be a mirror of the left subtree of the second node. Notice the crossover pattern - we're comparing left with right and right with left, which is what creates the mirror effect.

Let me walk through an example: root = [1,2,2,3,4,4,3]. The root is 1. I call isMirror on the left child 2 and right child 2. Both are 2, so values match. Then I recursively check: left.left (3) with right.right (3) - they match. And left.right (4) with right.left (4) - they match. Continuing down, all leaf nodes match their mirror counterparts, so the tree is symmetric.

Another example: root = [1,2,2,null,3,null,3]. The root is 1. I compare the two 2s - values match. Then I check left.left (null) with right.right (3). One is null and one isn't, so they're not mirrors, and I return false immediately.

An important detail: the crossover comparison is crucial. If I compared left.left with right.left, I'd be checking if the two subtrees are identical, not if they're mirror images. The mirror property requires comparing outer edges with outer edges and inner edges with inner edges from opposite sides.

The time complexity is O(n) where n is the number of nodes, because in the worst case of a symmetric tree, we need to visit every node to verify symmetry. The space complexity is O(h) where h is the height, due to the recursion stack. For a balanced tree, this is O(log n), but for a skewed tree, it could be O(n).

For the follow-up about an iterative solution, I could use a queue and process nodes level by level, or use two simultaneous traversals - one going left-to-right and another going right-to-left, comparing nodes as I go. The iterative approach would have the same time complexity but use O(w) space where w is the maximum width of the tree, for the queue. However, the recursive solution is more intuitive and elegant for this problem."
