#Tree #DFS #BFS #BinaryTree

## 풀이 코드
```java
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
    public TreeNode invertTree(TreeNode root) {
        // 1. 빈 트리면 그대로 반환
        // why: null 노드는 뒤집을 것이 없으므로 base case
        if (root == null) {
            return null;
        }
        
        // 2. 왼쪽과 오른쪽 자식을 임시로 저장
        // why: swap하기 전에 원래 참조를 저장해야 함
        TreeNode temp = root.left;
        root.left = root.right;
        root.right = temp;
        
        // 3. 왼쪽 서브트리를 재귀적으로 뒤집기
        // why: 모든 서브트리도 뒤집어야 완전히 반전됨
        invertTree(root.left);
        
        // 4. 오른쪽 서브트리를 재귀적으로 뒤집기
        // why: 모든 서브트리도 뒤집어야 완전히 반전됨
        invertTree(root.right);
        
        // 5. 뒤집힌 트리의 루트 반환
        // why: 현재 노드를 루트로 하는 서브트리가 완전히 뒤집힘
        return root;
    }
}
```

**시간 복잡도**: O(n) - 모든 노드를 한 번씩 방문 (n은 노드 개수)  
**공간 복잡도**: O(h) - 재귀 호출 스택의 깊이 (h는 트리 높이, 최악의 경우 O(n))

## 면접 대본 (영어)

"Let me explain my approach to inverting a binary tree.

The key insight is that inverting a binary tree means swapping the left and right children at every node. This can be done elegantly with recursion - at each node, swap its children and then recursively invert both subtrees.

The base case is simple: if the root is null, there's nothing to invert, so I just return null. This handles both an initially empty tree and the recursive case when we reach beyond a leaf node.

For the recursive case, I first swap the left and right children of the current node. I use a temporary variable to hold one child while performing the swap - this is a classic swap operation. After swapping, the original left child is now on the right, and the original right child is now on the left.

But swapping at just the root isn't enough - I need to invert all the subtrees too. So I make two recursive calls: one on root.left (which is now the original right subtree) and one on root.right (which is now the original left subtree). These recursive calls ensure that every node in the tree gets its children swapped.

Finally, I return the root, which now points to the fully inverted tree.

Let me walk through an example: if we have root = [4,2,7,1,3,6,9], at the root 4, I swap its children, so 2 and 7 switch positions. Now I recursively call on the left child 7 (originally right). At node 7, I swap 6 and 9. Then I recursively call on 6 and 9, but they're leaves so nothing happens. Similarly, I process the right side: at node 2, I swap 1 and 3. After all recursive calls complete, the tree is [4,7,2,9,6,3,1].

A simpler example: root = [2,1,3]. At node 2, I swap 1 and 3. Then I recursively process nodes 3 and 1, but they're leaves with no children to swap. The result is [2,3,1].

An important detail: I could also invert the subtrees first and then swap, or I could do both operations in either order. The key is that every node must have its children swapped, and this must happen at all levels of the tree.

The time complexity is O(n) where n is the number of nodes, because we visit each node exactly once to perform the swap. The space complexity is O(h) where h is the height of the tree, due to the recursion call stack. In a balanced tree, h is log(n), but in a skewed tree, it could be n, making the space complexity O(n) in the worst case.

An alternative approach would be to use an iterative solution with a queue for breadth-first traversal, or a stack for depth-first traversal. These would have the same time complexity but different space characteristics. However, the recursive solution is more intuitive and elegant, making it easier to understand and implement correctly in an interview setting.

This problem is famously known because the creator of Homebrew was reportedly rejected from a Google interview for not being able to invert a binary tree on a whiteboard, which sparked a lot of discussion in the tech community about interview practices."
