#Tree #DFS #BinaryTree

## 풀이 코드
```java
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
    public boolean hasPathSum(TreeNode root, int targetSum) {
        // 1. 빈 트리면 경로가 없으므로 false
        // why: null 노드는 루트-리프 경로를 만들 수 없음
        if (root == null) {
            return false;
        }
        
        // 2. 리프 노드에 도달했을 때 남은 합이 현재 노드 값과 같은지 확인
        // why: 리프까지의 경로 합이 targetSum과 같으면 조건 만족
        if (root.left == null && root.right == null) {
            return targetSum == root.val;
        }
        
        // 3. 현재 노드 값을 빼고 왼쪽과 오른쪽 서브트리에서 재귀적으로 탐색
        // why: 남은 합(targetSum - root.val)을 자식 노드들에게 전달하여 확인
        int remainingSum = targetSum - root.val;
        
        // 4. 왼쪽 또는 오른쪽 서브트리 중 하나라도 조건을 만족하면 true
        // why: 어느 한 경로라도 목표 합을 만족하면 됨
        return hasPathSum(root.left, remainingSum) || hasPathSum(root.right, remainingSum);
    }
}
```

**시간 복잡도**: O(n) - 최악의 경우 모든 노드를 방문 (n은 노드 개수)  
**공간 복잡도**: O(h) - 재귀 호출 스택의 깊이 (h는 트리 높이, 최악의 경우 O(n))

## 면접 대본 (영어)

"Let me explain my approach to finding if there's a root-to-leaf path with a target sum.

The key insight is to use depth-first search with recursion. As we traverse down the tree, we subtract each node's value from the target sum. When we reach a leaf node, we check if the remaining sum equals the leaf's value. If so, we've found a valid path.

I'll handle this with two base cases and one recursive case. First base case: if the root is null, there's no path at all, so I return false. This handles an empty tree.

Second base case: if we've reached a leaf node - a node with no children - we check if the target sum equals this leaf's value. This is crucial because we're looking for root-to-leaf paths, not just any path. A leaf is identified by having both left and right children as null.

For the recursive case, I calculate the remaining sum by subtracting the current node's value from the target sum. Then I recursively check both the left and right subtrees with this remaining sum. I use the OR operator because we only need one valid path - if either the left or right subtree contains a path that sums to the remaining target, we return true.

Let me walk through an example: root = [5,4,8,11,null,13,4,7,2,null,null,null,1], targetSum = 22. At root 5, remaining sum is 22-5=17. I check the left child 4, remaining sum is 17-4=13. Then I check its left child 11, remaining sum is 13-11=2. Node 11 has two children: 7 and 2. When I check the right child 2, the remaining sum is 2, and 2 is a leaf node, and 2 equals the remaining sum, so I return true. This path 5→4→11→2 sums to 22.

Another example: root = [1,2,3], targetSum = 5. At root 1, remaining sum is 4. I check left child 2 (remaining sum 2) - it's a leaf but 2≠2... wait, actually the remaining is 4-2=2, and the leaf value is 2, so they match. But let me recalculate: the path 1→2 sums to 3, not 5. So when I reach leaf 2, the remaining sum should be 4, but the leaf value is 2, so they don't match. Similarly for the right path 1→3, the sum is 4, not 5. Both paths fail, so I return false.

An important edge case: what if we have a tree with a single node? If root = [1] and targetSum = 1, the root is also a leaf, so we check if 1 equals 1, which is true. If targetSum = 2, we return false.

Another edge case: nodes can have negative values. If root = [1,-2,3] and targetSum = 2, the left path 1→-2 sums to -1, and the right path 1→3 sums to 4. Neither equals 2, so we return false.

The time complexity is O(n) where n is the number of nodes, because in the worst case we might need to explore all paths in the tree. The space complexity is O(h) where h is the height of the tree, due to the recursion stack. In a balanced tree, h is log(n), but in a skewed tree, it could be n.

The beauty of this solution is its simplicity - by subtracting the current value from the target and passing the remainder down, we transform a path sum problem into a series of simpler subproblems, perfectly suited for recursion."
