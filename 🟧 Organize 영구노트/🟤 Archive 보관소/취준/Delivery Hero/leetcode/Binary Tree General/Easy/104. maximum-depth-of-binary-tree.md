## 풀이 코드
```java
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
    public int maxDepth(TreeNode root) {
        // 1. 빈 트리면 깊이 0 반환
        // why: null 노드는 트리가 아니므로 깊이가 0
        if (root == null) {
            return 0;
        }
        
        // 2. 왼쪽 서브트리의 최대 깊이를 재귀적으로 계산
        // why: 왼쪽 자식부터 리프까지의 최대 경로 길이 구함
        int leftDepth = maxDepth(root.left);
        
        // 3. 오른쪽 서브트리의 최대 깊이를 재귀적으로 계산
        // why: 오른쪽 자식부터 리프까지의 최대 경로 길이 구함
        int rightDepth = maxDepth(root.right);
        
        // 4. 두 서브트리 중 더 깊은 쪽을 선택하고 현재 노드 포함하여 1 추가
        // why: 현재 노드에서의 최대 깊이는 더 깊은 서브트리 + 1
        return Math.max(leftDepth, rightDepth) + 1;
    }
}
```

**시간 복잡도**: O(n) - 모든 노드를 한 번씩 방문 (n은 노드 개수)  
**공간 복잡도**: O(h) - 재귀 호출 스택의 깊이 (h는 트리 높이, 최악의 경우 O(n))

## 면접 대본 (영어)

"Let me explain my approach to finding the maximum depth of a binary tree.

The key insight is that the maximum depth of a tree can be defined recursively: it's the maximum depth of either the left or right subtree, plus one for the current node. This naturally leads to a depth-first search solution using recursion.

The base case is straightforward: if the root is null, there's no tree, so the depth is zero. This handles both the initial case of an empty tree and the recursive case when we reach beyond a leaf node.

For the recursive case, I need to find the maximum depth of both the left and right subtrees. I do this by recursively calling maxDepth on root.left and root.right. Each of these calls will explore their respective subtrees all the way down to the leaves, returning the maximum depth found.

Once I have both depths, I take the maximum of the two - because the overall depth is determined by whichever side goes deeper - and add one to account for the current node itself.

Let me walk through an example: if we have a tree [3,9,20,null,null,15,7], the root is 3. I recursively call maxDepth on the left child 9. Node 9 has no children, so both its recursive calls return 0, and it returns max(0,0) + 1 = 1. Then I call maxDepth on the right child 20. Node 20 has two children: 15 and 7. Both are leaves, so they return 1. Node 20 returns max(1,1) + 1 = 2. Back at the root, I have leftDepth = 1 and rightDepth = 2, so I return max(1,2) + 1 = 3.

Another example: tree [1,null,2] - root is 1. Left child is null, returns 0. Right child is 2, which is a leaf, returns 1. Root returns max(0,1) + 1 = 2.

The beauty of this solution is its simplicity and elegance. The recursive structure mirrors the tree structure perfectly. Each node only needs to know the depths of its immediate children - it doesn't need to know anything about the overall tree structure.

The time complexity is O(n) where n is the number of nodes, because we visit each node exactly once. The space complexity is O(h) where h is the height of the tree, due to the recursion call stack. In a balanced tree, h is log(n), but in a skewed tree where every node has only one child, h could be n, making the space complexity O(n) in the worst case.

An alternative approach would be to use an iterative level-order traversal with a queue, which would also be O(n) time and O(w) space where w is the maximum width of the tree. However, the recursive DFS approach is more intuitive and elegant for this problem."
