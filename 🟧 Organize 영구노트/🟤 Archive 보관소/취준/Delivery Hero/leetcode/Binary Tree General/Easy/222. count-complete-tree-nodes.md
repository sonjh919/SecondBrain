## 풀이 코드
```java
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
    public int countNodes(TreeNode root) {
        // 1. 빈 트리면 노드 개수 0
        // why: null 노드는 카운트하지 않음
        if (root == null) {
            return 0;
        }
        
        // 2. 왼쪽 끝까지 내려가며 높이 계산
        // why: 완전 이진 트리의 높이를 구하기 위해 가장 왼쪽 경로 탐색
        int leftHeight = getHeight(root.left);
        int rightHeight = getHeight(root.right);
        
        // 3. 왼쪽과 오른쪽 높이가 같으면 왼쪽 서브트리는 완전 이진 트리
        // why: 왼쪽 서브트리가 꽉 차있으므로 2^h - 1 공식으로 노드 수 계산
        if (leftHeight == rightHeight) {
            return (1 << leftHeight) + countNodes(root.right);
        }
        // 4. 높이가 다르면 오른쪽 서브트리는 완전 이진 트리
        // why: 마지막 레벨의 노드들이 왼쪽으로 채워지므로 오른쪽이 완전
        else {
            return (1 << rightHeight) + countNodes(root.left);
        }
    }
    
    // 5. 가장 왼쪽 경로를 따라 높이를 계산하는 헬퍼 함수
    // why: 완전 이진 트리의 높이는 가장 왼쪽 경로의 길이
    private int getHeight(TreeNode node) {
        int height = 0;
        while (node != null) {
            height++;
            node = node.left;
        }
        return height;
    }
}
```

```java
class Solution {
    public int countNodes(TreeNode root) {
        // 1. 베이스 케이스: null이면 0 반환
        if (root == null) {
            return 0;
        }
        
        // 2. 현재 노드(1) + 왼쪽 서브트리 + 오른쪽 서브트리
        return 1 + countNodes(root.left) + countNodes(root.right);
    }
}
```

**시간 복잡도**: O(log²n) - 높이 계산 O(log n)을 최대 log n번 수행  
**공간 복잡도**: O(log n) - 재귀 호출 스택의 깊이 (트리 높이만큼)

## 면접 대본 (영어)

"Let me explain my approach to counting nodes in a complete binary tree in less than O(n) time.

The key insight is to leverage the properties of a complete binary tree. In a complete binary tree, all levels are fully filled except possibly the last level, which is filled from left to right. This means we can determine if a subtree is a perfect binary tree by comparing heights, and for perfect trees, we can calculate the node count using the formula 2^h - 1 without traversing every node.

The algorithm works by comparing the left and right heights at each node. I calculate the height by going all the way left from the left child, and all the way left from the right child. In a complete binary tree, if these heights are equal, it means the left subtree is a perfect binary tree that's completely filled. If the heights differ by one, it means the right subtree is perfect but shorter.

Here's the detailed process: First, I handle the base case where root is null, returning 0. Then I calculate the left height by traversing left from root.left, and the right height by traversing left from root.right. Note that I always go left because in a complete tree, the leftmost path determines the height.

If leftHeight equals rightHeight, the left subtree is a perfect binary tree of height leftHeight. I can calculate its node count as 2^leftHeight - 1, add 1 for the root, which gives 2^leftHeight, then recursively count the right subtree. I use bit shifting (1 << leftHeight) for efficiency, which is equivalent to 2^leftHeight.

If leftHeight is greater than rightHeight, the right subtree is a perfect binary tree of height rightHeight. I calculate 2^rightHeight nodes for the right subtree plus root, then recursively count the left subtree.

Let me walk through an example: root = [1,2,3,4,5,6]. At root 1, left height going through 2→4 is 2, right height going through 3→6 is 2. They're equal, so the left subtree is perfect with 2^2 = 4 nodes including root 1. I recursively count root 3: left height 1, right height 1, equal, so left subtree is perfect with 2 nodes. Recursively count root 6: it's a leaf, return 1. Total: 4 + 1 + 1 = 6.

Another example: root = [1]. Left height is 0, right height is 0. They're equal, so 2^0 = 1 node, plus countNodes(null) = 0. Total: 1.

The brilliance of this algorithm is that we avoid counting every node. At each recursion level, we can calculate half the tree's nodes using the formula, and only recurse on the other half. This gives us a time complexity better than O(n).

The time complexity is O(log²n). At each level of recursion, we calculate the height which takes O(log n) time. We recurse at most O(log n) times because we're halving the problem each time. So total is O(log n × log n) = O(log²n). The space complexity is O(log n) for the recursion stack.

This is significantly better than the naive O(n) approach of visiting every node. For a tree with 10,000 nodes, instead of 10,000 operations, we'd do roughly 14² ≈ 196 operations - a huge improvement! This solution elegantly exploits the complete binary tree property to achieve logarithmic complexity."
