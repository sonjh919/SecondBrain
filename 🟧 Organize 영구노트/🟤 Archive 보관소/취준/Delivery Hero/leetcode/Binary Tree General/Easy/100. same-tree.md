## 풀이 코드
```java
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
    public boolean isSameTree(TreeNode p, TreeNode q) {
        // 1. 두 노드가 모두 null이면 같은 트리
        // why: 둘 다 비어있으면 구조적으로 동일
        if (p == null && q == null) {
            return true;
        }
        
        // 2. 하나만 null이면 다른 트리
        // why: 한쪽만 노드가 있으면 구조가 다름
        if (p == null || q == null) {
            return false;
        }
        
        // 3. 현재 노드의 값이 다르면 다른 트리
        // why: 같은 위치의 노드 값이 달라야 같은 트리
        if (p.val != q.val) {
            return false;
        }
        
        // 4. 왼쪽 서브트리와 오른쪽 서브트리가 모두 같은지 재귀적으로 확인
        // why: 현재 노드가 같아도 모든 자식 노드들이 같아야 동일한 트리
        return isSameTree(p.left, q.left) && isSameTree(p.right, q.right);
    }
}
```

**시간 복잡도**: O(min(n, m)) - 두 트리 중 작은 트리의 모든 노드를 방문 (n, m은 각 트리의 노드 개수)  
**공간 복잡도**: O(min(h1, h2)) - 재귀 호출 스택의 깊이 (h1, h2는 각 트리의 높이)

## 면접 대본 (영어)

"Let me explain my approach to checking if two binary trees are identical.

The key insight is that two trees are the same if and only if: first, their root values are equal; second, their left subtrees are the same; and third, their right subtrees are the same. This naturally leads to a recursive solution where we compare nodes pairwise.

I'll handle this with several base cases followed by a recursive case. The base cases cover all the scenarios where we can immediately determine the answer without further recursion.

First base case: if both nodes are null, they're the same - two empty trees are identical. This also handles the case when we've reached beyond the leaves of both trees simultaneously.

Second base case: if only one node is null, the trees are different. This means one tree has a node where the other doesn't, so they have different structures.

Third base case: if both nodes exist but have different values, the trees are different. Even if the structure matches, differing values make them non-identical.

If we pass all these checks, we know the current nodes are identical. But we still need to verify that all their descendants are identical too. So I make two recursive calls: one to check if the left subtrees are the same, and another to check if the right subtrees are the same. I return true only if both recursive calls return true.

Let me walk through an example: p = [1,2,3] and q = [1,2,3]. At the root, both are 1, so we recurse. For the left children, both are 2 and they have no children, so we return true. For the right children, both are 3 and they have no children, so we return true. Since both left and right subtrees are the same, we return true.

Another example: p = [1,2] and q = [1,null,2]. At the root, both are 1. For the left subtree, p has 2 but q has null. One is null and one isn't, so we return false immediately without checking further.

A more complex case: p = [1,2,1] and q = [1,1,2]. At the root, both are 1. For the left children, p has 2 but q has 1. The values differ, so we return false. We don't even need to check the right subtree because we've already found a mismatch.

The time complexity is O(min(n, m)) where n and m are the number of nodes in each tree. We visit each node once, but we stop as soon as we find a mismatch, so we might not visit all nodes if the trees differ early. In the best case where trees differ at the root, it's O(1). In the worst case where trees are identical, we visit all nodes.

The space complexity is O(min(h1, h2)) where h1 and h2 are the heights of the two trees, due to the recursion call stack. In a balanced tree, this is O(log n), but in a skewed tree, it could be O(n).

This recursive solution is elegant and mirrors the recursive structure of binary trees perfectly. Each node only needs to verify itself and delegate the subtree verification to recursive calls."
