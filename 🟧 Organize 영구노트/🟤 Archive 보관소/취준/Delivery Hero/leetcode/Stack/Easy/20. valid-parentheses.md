#String #Stack

## 풀이 코드
```java
class Solution {
    public boolean isValid(String s) {
        // 1. 여는 괄호를 저장할 스택 생성
        // why: LIFO 구조로 가장 최근의 여는 괄호와 매칭 확인
        Stack<Character> stack = new Stack<>();
        
        // 2. 문자열을 순회하며 각 괄호 처리
        // why: 왼쪽에서 오른쪽으로 읽으며 괄호 쌍을 검증
        for (char c : s.toCharArray()) {
            // 3. 여는 괄호면 스택에 추가
            // why: 나중에 나올 닫는 괄호와 매칭하기 위해 저장
            if (c == '(' || c == '{' || c == '[') {
                stack.push(c);
            }
            // 4. 닫는 괄호면 스택에서 꺼내서 매칭 확인
            // why: 가장 최근의 여는 괄호와 타입이 일치해야 함
            else {
                // 5. 스택이 비어있으면 매칭할 여는 괄호가 없음
                // why: 닫는 괄호에 대응하는 여는 괄호가 없는 경우
                if (stack.isEmpty()) {
                    return false;
                }
                
                char top = stack.pop();
                
                // 6. 괄호 타입이 일치하지 않으면 false
                // why: 같은 타입의 괄호끼리만 쌍을 이루어야 함
                if ((c == ')' && top != '(') ||
                    (c == '}' && top != '{') ||
                    (c == ']' && top != '[')) {
                    return false;
                }
            }
        }
        
        // 7. 모든 괄호를 처리한 후 스택이 비어있어야 함
        // why: 남은 여는 괄호가 있으면 짝이 맞지 않는 것
        return stack.isEmpty();
    }
}
```

**시간 복잡도**: O(n) - 문자열을 한 번 순회 (n은 문자열 길이)  
**공간 복잡도**: O(n) - 최악의 경우 모든 문자가 여는 괄호일 때 스택에 저장

## 면접 대본 (영어)

"Let me explain my approach to validating parentheses using a stack.

The key insight is that valid parentheses follow a Last-In-First-Out pattern. The most recently opened bracket must be closed first before any earlier brackets can be closed. This makes a stack the perfect data structure for this problem.

The algorithm is straightforward: I use a stack to keep track of opening brackets. As I traverse the string from left to right, whenever I encounter an opening bracket - '(', '{', or '[' - I push it onto the stack. When I encounter a closing bracket, I need to verify two things: first, that there's a matching opening bracket available, and second, that it's of the correct type.

Here's how it works in detail: For each character in the string, if it's an opening bracket, I simply push it onto the stack. If it's a closing bracket, I first check if the stack is empty. An empty stack means there's no opening bracket to match with, so the string is invalid and I return false immediately.

If the stack isn't empty, I pop the top element and check if it matches the current closing bracket. For example, if I see ')', the popped element must be '('. If I see '}', it must be '{'. If I see ']', it must be '['. If the types don't match, the string is invalid.

After processing all characters, there's one final check: the stack must be empty. If there are any opening brackets left in the stack, it means they were never closed, so the string is invalid.

Let me walk through an example: if we have s = '([)]', I push '(' onto the stack. Then I push '['. Next, I see ')', so I pop '[' from the stack. But ')' doesn't match '[', so I return false immediately.

Another example: s = '([])', I push '(', then '['. I see ']', pop '[' - they match. I see ')', pop '(' - they match. The stack is empty and all brackets matched correctly, so I return true.

A tricky case: s = '((', I push '(' twice. After processing all characters, the stack has two '(' remaining. Since the stack isn't empty, I return false - these opening brackets were never closed.

Another edge case: s = '))', the first ')' tries to pop from an empty stack, so I return false immediately - there's no opening bracket to match.

The time complexity is O(n) where n is the length of the string, since we process each character exactly once. The space complexity is O(n) in the worst case where the string contains only opening brackets, requiring us to store all of them in the stack. This solution elegantly handles all cases including nested and interleaved brackets by leveraging the LIFO property of stacks."
