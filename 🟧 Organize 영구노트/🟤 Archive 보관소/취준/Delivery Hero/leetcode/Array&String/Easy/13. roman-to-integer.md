#HashTable #Math #String

## 풀이 코드
```java
class Solution {
    public int romanToInt(String s) {
        // 1. 로마 숫자와 정수값을 매핑하는 HashMap 생성
        // why: 각 로마 문자의 값을 빠르게 조회하기 위함
        Map<Character, Integer> romanMap = new HashMap<>();
        romanMap.put('I', 1);
        romanMap.put('V', 5);
        romanMap.put('X', 10);
        romanMap.put('L', 50);
        romanMap.put('C', 100);
        romanMap.put('D', 500);
        romanMap.put('M', 1000);
        
        // 2. 결과값을 저장할 변수 초기화
        // why: 각 문자의 값을 누적하여 최종 정수값 계산
        int result = 0;
        
        // 3. 문자열을 순회하며 각 문자의 값을 처리
        // why: 왼쪽에서 오른쪽으로 읽으며 빼기 규칙 적용
        for (int i = 0; i < s.length(); i++) {
            int current = romanMap.get(s.charAt(i));
            
            // 4. 다음 문자가 존재하고 현재 값보다 크면 빼기, 아니면 더하기
            // why: IV, IX 같은 빼기 표현은 작은 값이 큰 값 앞에 옴
            if (i + 1 < s.length() && current < romanMap.get(s.charAt(i + 1))) {
                result -= current;
            } else {
                result += current;
            }
        }
        
        // 5. 최종 정수값 반환
        // why: 모든 문자를 처리한 결과가 로마 숫자의 정수 변환값
        return result;
    }
}
```

**시간 복잡도**: O(n) - 문자열을 한 번 순회 (n은 문자열 길이)  
**공간 복잡도**: O(1) - HashMap 크기가 고정(7개)이므로 상수 공간

## 면접 대본 (영어)

"Let me explain my approach to converting Roman numerals to integers.

The key insight here is understanding how Roman numerals work. Typically, larger symbols appear before smaller ones and we add them up. However, there's a special subtraction rule: when a smaller symbol appears before a larger one, we subtract instead of add. For example, IV is 4 (5-1) and IX is 9 (10-1).

I'll use a HashMap to store the mapping between Roman characters and their integer values. This gives me O(1) lookup time for each character.

Here's how the algorithm works: I iterate through the string from left to right. For each character, I get its value from the HashMap. Then I check if there's a next character and if the current value is less than the next value. If both conditions are true, it means we're in a subtraction case like IV or IX, so I subtract the current value from the result. Otherwise, I add the current value to the result.

Let me walk through an example: if we have s = 'MCMXCIV', initially result is 0. When i=0, current is M=1000, and the next character C=100 is smaller, so we add 1000, making result 1000. When i=1, current is C=100, next is M=1000 which is larger, so we subtract 100, making result 900. When i=2, current is M=1000, next is X=10 which is smaller, so we add 1000, making result 1900. When i=3, current is X=10, next is C=100 which is larger, so we subtract 10, making result 1890. When i=4, current is C=100, next is I=1 which is smaller, so we add 100, making result 1990. When i=5, current is I=1, next is V=5 which is larger, so we subtract 1, making result 1989. Finally, when i=6, current is V=5, there's no next character, so we add 5, making result 1994.

The time complexity is O(n) where n is the length of the string, since we traverse it once. The space complexity is O(1) because the HashMap size is fixed at 7 entries regardless of input size. This solution elegantly handles both the addition and subtraction rules in a single pass."
