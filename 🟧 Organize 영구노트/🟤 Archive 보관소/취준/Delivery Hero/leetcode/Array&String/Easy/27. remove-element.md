#Array #TwoPointers

## 풀이 코드
```java
class Solution {
    public int removeElement(int[] nums, int val) {
        // 1. val이 아닌 요소들을 앞쪽부터 채울 포인터 k 초기화
        // why: k는 결과 배열의 길이이자 다음에 채울 위치를 나타냄
        int k = 0;
        
        // 2. 배열을 순회하면서 val이 아닌 요소만 앞쪽에 저장
        // why: Two Pointer 기법으로 한 번의 순회만으로 제거 작업 완료
        for (int i = 0; i < nums.length; i++) {
            if (nums[i] != val) {
                nums[k] = nums[i];
                k++;
            }
        }
        
        // 3. val이 아닌 요소의 개수 k를 반환
        // why: 문제에서 요구하는 조건이 앞쪽 k개 요소만 유효하면 됨
        return k;
    }
}
```

**시간 복잡도**: O(n) - 배열을 한 번 순회  
**공간 복잡도**: O(1) - 추가 공간 사용 없이 in-place로 처리

## 면접 대본 (영어)

"Let me explain my approach to solving this in-place removal problem.

The key idea is to use a two-pointer technique. I'll maintain one pointer, k, that represents the position where I should place the next valid element. This pointer also serves as a counter for the number of elements that are not equal to val.

Here's how the algorithm works: I iterate through the entire array with index i. For each element, I check if it's equal to val. If it's not equal to val, I copy that element to position k and then increment k. If the element equals val, I simply skip it and move to the next element.

This approach is elegant because it handles everything in a single pass through the array. We don't need any extra space beyond the input array itself, making it a true in-place solution.

Let me walk through an example: if we have nums = [3,2,2,3] and val = 3, initially k is 0. When i=0, nums[0] is 3, which equals val, so we skip it. When i=1, nums[1] is 2, which doesn't equal val, so we place it at position k=0 and increment k to 1. Similarly for i=2, we place 2 at position k=1 and increment k to 2. When i=3, we skip 3 again. Finally, we return k=2.

The time complexity is O(n) where n is the length of the array, since we traverse it once. The space complexity is O(1) because we only use a constant amount of extra space. The order of elements in the result may differ from the original, but that's acceptable according to the problem statement."
