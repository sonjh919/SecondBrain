## 풀이 코드
```java
class Solution {
    public String longestCommonPrefix(String[] strs) {
        // 1. 배열이 비어있거나 null이면 빈 문자열 반환
        // why: 공통 접두사를 찾을 문자열이 없으면 조기 반환
        if (strs == null || strs.length == 0) return "";
        
        // 2. 첫 번째 문자열을 기준 접두사로 설정
        // why: 공통 접두사는 첫 번째 문자열보다 길 수 없음
        String prefix = strs[0];
        
        // 3. 나머지 문자열들과 비교하며 접두사 길이를 줄여나감
        // why: 모든 문자열과 공통된 부분만 남기기 위함
        for (int i = 1; i < strs.length; i++) {
            // 4. 현재 문자열이 prefix로 시작하지 않으면 prefix를 한 글자씩 줄임
            // why: indexOf가 0이 아니면 해당 prefix로 시작하지 않는다는 의미
            while (strs[i].indexOf(prefix) != 0) {
                prefix = prefix.substring(0, prefix.length() - 1);
                // 5. prefix가 빈 문자열이 되면 공통 접두사가 없으므로 반환
                // why: 더 이상 줄일 수 없으면 공통 접두사 없음
                if (prefix.isEmpty()) return "";
            }
        }
        
        // 6. 모든 문자열과 공통된 최장 접두사 반환
        // why: 모든 비교를 마친 후 남은 prefix가 최장 공통 접두사
        return prefix;
    }
}
```

```java
class Solution {

public String longestCommonPrefix(String[] strs) {

if (strs.length == 0) return "";

  

StringBuilder sb = new StringBuilder();

  

for (int i = 0; i < strs[0].length(); i++) {

char c = strs[0].charAt(i);

  

for (String str : strs) {

if (str.length() <= i || c != str.charAt(i)) {

return sb.toString();

}

}

  

sb.append(c);

}

  

return sb.toString();

}

}
```

**시간 복잡도**: O(S) - S는 모든 문자열의 총 문자 수 (최악의 경우 모든 문자를 비교)  
**공간 복잡도**: O(1) - prefix 문자열 외 추가 공간 사용 없음

## 면접 대본 (영어)

"Let me explain my approach to finding the longest common prefix among an array of strings.

The key insight is to start with the first string as our initial prefix candidate, then progressively shorten it by comparing with each subsequent string until we find a prefix that works for all strings.

First, I handle edge cases. If the array is null or empty, there's no common prefix, so I return an empty string. Then I set the first string as my initial prefix - this makes sense because the common prefix can't be longer than the shortest string, and we need a starting point for comparison.

Here's how the algorithm works: I iterate through the remaining strings starting from index 1. For each string, I check if it starts with the current prefix using the indexOf method. If indexOf returns 0, it means the string starts with this prefix, and I move to the next string. If it doesn't return 0, the current prefix is too long, so I shorten it by removing the last character and check again. I keep shortening the prefix until the current string starts with it, or until the prefix becomes empty.

If at any point the prefix becomes empty, it means there's no common prefix among all strings, so I return an empty string immediately.

Let me walk through an example: if we have strs = ['flower', 'flow', 'flight'], I start with prefix = 'flower'. Comparing with 'flow', 'flow' doesn't start with 'flower', so I shorten prefix to 'flowe', then 'flow'. Now 'flow' starts with 'flow', so I move on. Comparing with 'flight', 'flight' doesn't start with 'flow', so I shorten to 'flo', then 'fl'. Now 'flight' starts with 'fl', and I've checked all strings, so I return 'fl'.

Another example: if we have strs = ['dog', 'racecar', 'car'], prefix starts as 'dog'. When comparing with 'racecar', I keep shortening until prefix becomes empty, so I return an empty string.

The time complexity is O(S) where S is the sum of all characters in all strings, because in the worst case we might compare every character. The space complexity is O(1) if we don't count the prefix string itself, as we only use a constant amount of extra space. This solution is straightforward and handles all edge cases naturally."
