## 풀이 코드
```java
class Solution {
    public int majorityElement(int[] nums) {
        // 1. Boyer-Moore 투표 알고리즘을 위한 후보와 카운트 초기화
        // why: O(1) 공간으로 과반수 요소를 찾기 위한 최적 알고리즘
        int candidate = 0;
        int count = 0;
        
        // 2. 배열을 순회하며 후보를 선정하고 카운트 조정
        // why: 같은 값이면 카운트 증가, 다르면 감소하여 과반수 요소만 살아남음
        for (int num : nums) {
            if (count == 0) {
                candidate = num;
            }
            count += (num == candidate) ? 1 : -1;
        }
        
        // 3. 최종 후보를 과반수 요소로 반환
        // why: 문제 조건상 과반수 요소가 항상 존재하므로 검증 불필요
        return candidate;
    }
}
```

**시간 복잡도**: O(n) - 배열을 한 번 순회  
**공간 복잡도**: O(1) - 후보와 카운트 변수만 사용

## 면접 대본 (영어)

"Let me explain my approach to finding the majority element using the Boyer-Moore Voting Algorithm.

The key insight here is that the majority element appears more than n/2 times, which means it appears more frequently than all other elements combined. This property allows us to use a clever voting mechanism.

I'll maintain two variables: a candidate for the majority element and a count. The algorithm works in a single pass through the array.

Here's how it works: I iterate through each element in the array. If the count is zero, I set the current element as my new candidate. Then, if the current element equals the candidate, I increment the count by 1. If it's different, I decrement the count by 1. The idea is that we're essentially doing a pairing process - each occurrence of the candidate is paired with an occurrence of a different element, and they cancel each other out.

Let me walk through an example: if we have nums = [2,2,1,1,1,2,2], initially candidate is 0 and count is 0. When we see the first 2, count becomes 0 so we set candidate to 2 and count to 1. Second 2 makes count 2. Then we see 1, which decrements count to 1. Another 1 makes count 0. The next 1 sets candidate to 1 and count to 1. But then we see 2, which decrements count to 0. Finally, the last 2 sets candidate to 2 and count to 1. We return 2 as the majority element.

The beauty of this algorithm is that even if the candidate changes during the process, the true majority element will always end up as the final candidate. This is because the majority element appears more than half the time, so it will always survive the cancellation process.

The time complexity is O(n) since we make a single pass through the array. The space complexity is O(1) because we only use two variables regardless of the input size. This satisfies the follow-up requirement of linear time and constant space."
