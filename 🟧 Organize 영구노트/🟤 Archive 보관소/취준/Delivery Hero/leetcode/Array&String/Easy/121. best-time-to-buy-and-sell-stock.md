#Array #DynamicProgramming

## 풀이 코드
```java
class Solution {
    public int maxProfit(int[] prices) {
        // 1. 최소 가격과 최대 이익을 추적할 변수 초기화
        // why: 현재까지의 최소 가격과 최대 이익을 기록하며 한 번의 순회로 해결
        int minPrice = Integer.MAX_VALUE;
        int maxProfit = 0;
        
        // 2. 배열을 순회하며 각 날의 가격 확인
        // why: 각 시점에서 최소 매수가와 최대 이익을 갱신
        for (int price : prices) {
            // 3. 현재까지의 최소 가격 갱신
            // why: 가장 낮은 가격에 사야 이익을 극대화할 수 있음
            if (price < minPrice) {
                minPrice = price;
            } 
            // 4. 현재 가격에서 팔았을 때의 이익 계산 및 최대 이익 갱신
            // why: 현재 가격 - 최소 가격이 현재까지의 최대 이익보다 크면 갱신
            else if (price - minPrice > maxProfit) {
                maxProfit = price - minPrice;
            }
        }
        
        // 5. 최대 이익 반환
        // why: 이익을 낼 수 없으면 0, 있으면 최대 이익값 반환
        return maxProfit;
    }
}
```

**시간 복잡도**: O(n) - 배열을 한 번 순회  
**공간 복잡도**: O(1) - 두 개의 변수만 사용

## 면접 대본 (영어)

"Let me explain my approach to solving this stock trading problem.

The key insight is that to maximize profit, we need to buy at the lowest price and sell at the highest price that comes after it. We can find this optimal pair in a single pass through the array by keeping track of two things: the minimum price we've seen so far, and the maximum profit we can achieve.

I'll use two variables: minPrice to track the lowest price encountered so far, and maxProfit to track the best profit we've found. I initialize minPrice to the maximum integer value so any price will be lower, and maxProfit to 0 since we can't have negative profit - if no profit is possible, we simply don't trade.

Here's how the algorithm works: I iterate through each price in the array. At each step, I first check if the current price is lower than our minPrice. If it is, I update minPrice because I've found a better buying opportunity. If the current price is not a new minimum, I calculate the profit I would get by selling at this price - that's the current price minus minPrice. If this profit is greater than maxProfit, I update maxProfit.

Let me walk through an example: if we have prices = [7,1,5,3,6,4], initially minPrice is infinity and maxProfit is 0. On day 1, price is 7, which becomes our new minPrice. On day 2, price is 1, which is lower, so minPrice becomes 1. On day 3, price is 5, so profit would be 5-1=4, updating maxProfit to 4. On day 4, price is 3, profit would be 3-1=2, which doesn't beat our maxProfit. On day 5, price is 6, profit is 6-1=5, so maxProfit becomes 5. On day 6, price is 4, profit is 4-1=3, no update needed. We return 5.

The time complexity is O(n) where n is the number of days, since we make one pass through the prices array. The space complexity is O(1) because we only use two variables regardless of input size. This solution elegantly captures the optimal buy-low-sell-high strategy in a single traversal."
