#Array #TwoPointers

## 풀이 코드
```java
class Solution {
    public void merge(int[] nums1, int m, int[] nums2, int n) {
        // 1. 두 배열의 끝에서부터 비교하기 위한 포인터 설정
        // why: 뒤에서부터 채우면 nums1의 기존 요소를 덮어쓸 걱정 없이 병합 가능
        int p1 = m - 1;
        int p2 = n - 1;
        int p = m + n - 1;
        
        // 2. 두 배열을 뒤에서부터 비교하며 큰 값을 nums1의 뒤쪽부터 채움
        // why: 정렬된 배열이므로 뒤에서부터 비교하면 한 번의 순회로 병합 완료
        while (p1 >= 0 && p2 >= 0) {
            if (nums1[p1] > nums2[p2]) {
                nums1[p] = nums1[p1];
                p1--;
            } else {
                nums1[p] = nums2[p2];
                p2--;
            }
            p--;
        }
        
        // 3. nums2에 남은 요소가 있다면 nums1의 앞부분에 복사
        // why: nums1에 남은 요소는 이미 제자리에 있지만, nums2의 남은 요소는 복사 필요
        while (p2 >= 0) {
            nums1[p] = nums2[p2];
            p2--;
            p--;
        }
    }
}
```

**시간 복잡도**: O(m+n) - 두 배열을 각각 한 번씩 순회  
**공간 복잡도**: O(1) - 추가 공간 사용 없이 in-place로 병합

## 면접 대본 (영어)

"Let me walk you through my approach to solving this merge problem.

The key insight here is that we can merge these arrays efficiently by working backwards. Since both arrays are already sorted and nums1 has enough space at the end, we can start from the back and place the larger elements first.

I'll use three pointers: p1 pointing to the last valid element in nums1 at position m-1, p2 pointing to the last element in nums2 at position n-1, and p pointing to the very end of nums1 at position m+n-1.

The algorithm works like this: I compare the elements at p1 and p2, and whichever is larger gets placed at position p. Then I decrement both the pointer of the array I took from and the position pointer p. I continue this process until one of the arrays is exhausted.

There's an important edge case to handle. If nums2 still has elements left after nums1 is exhausted, I need to copy those remaining elements to the front of nums1. However, if nums1 has elements left, I don't need to do anything because they're already in the correct position.

The time complexity is O(m+n) since we traverse each array exactly once, and the space complexity is O(1) since we're merging in-place without using any extra data structures. This backward approach is elegant because it avoids the need to shift elements, which would happen if we merged from the front."
