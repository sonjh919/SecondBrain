#String #TwoPointers #StringMatching

## 풀이 코드
```java
class Solution {
    public int strStr(String haystack, String needle) {
        // 1. needle이 빈 문자열이면 0 반환
        // why: 빈 문자열은 모든 문자열의 시작 위치에 있다고 간주
        if (needle.isEmpty()) return 0;
        
        // 2. haystack이 needle보다 짧으면 -1 반환
        // why: needle을 포함할 수 없으므로 조기 반환
        if (haystack.length() < needle.length()) return -1;
        
        // 3. haystack을 순회하며 needle과 매칭 가능한 위치 탐색
        // why: 가능한 모든 시작 위치에서 needle과 비교
        for (int i = 0; i <= haystack.length() - needle.length(); i++) {
            // 4. 현재 위치에서 needle 길이만큼 부분 문자열 추출하여 비교
            // why: substring으로 needle과 동일한 길이의 문자열을 비교
            if (haystack.substring(i, i + needle.length()).equals(needle)) {
                return i;
            }
        }
        
        // 5. needle을 찾지 못하면 -1 반환
        // why: 모든 위치를 확인했지만 매칭되는 부분이 없음
        return -1;
    }
}
```

**시간 복잡도**: O((n-m+1) * m) = O(n*m) - n은 haystack 길이, m은 needle 길이 (최악의 경우 각 위치에서 substring 비교)  
**공간 복잡도**: O(m) - substring 생성 시 needle 길이만큼의 임시 문자열 생성

## 면접 대본 (영어)

"Let me explain my approach to finding the first occurrence of a substring in a string.

The key insight is to check every possible starting position in the haystack where the needle could fit, and compare the substring at that position with the needle.

First, I handle edge cases. If the needle is empty, I return 0 by convention - an empty string is found at the beginning of any string. If the haystack is shorter than the needle, it's impossible to contain the needle, so I return -1.

Here's how the algorithm works: I iterate through the haystack from index 0 to the last possible starting position where the needle could fit. The last possible position is at index haystack.length() minus needle.length(). At each position i, I extract a substring from the haystack starting at i with length equal to the needle's length, and compare it with the needle using the equals method. If they match, I've found the first occurrence, so I return the current index i immediately. If I complete the loop without finding a match, I return -1.

Let me walk through an example: if we have haystack = 'sadbutsad' and needle = 'sad', I start at i=0. I extract substring from 0 to 3: 'sad', which equals 'sad', so I return 0 immediately.

Another example: if we have haystack = 'leetcode' and needle = 'leeto', I check positions 0 through 3. At i=0, I extract 'leetc' which doesn't match 'leeto'. At i=1, I extract 'eetco' which doesn't match. At i=2, 'etcod' doesn't match. At i=3, 'tcode' doesn't match. I've checked all possible positions, so I return -1.

The time complexity is O(n*m) where n is the length of haystack and m is the length of needle. This is because we check up to n-m+1 positions, and at each position we perform a substring comparison that takes O(m) time. The space complexity is O(m) due to the substring creation. 

Note that Java's built-in indexOf method uses more optimized algorithms like KMP or Boyer-Moore for better performance, but this straightforward approach is easier to understand and implement in an interview setting."
