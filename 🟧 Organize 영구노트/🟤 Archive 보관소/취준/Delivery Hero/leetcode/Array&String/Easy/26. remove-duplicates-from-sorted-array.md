#Array #TwoPointers

## 풀이 코드
```java
class Solution {
    public int removeDuplicates(int[] nums) {
        // 1. 배열이 비어있거나 길이가 1이면 그대로 반환
        // why: 중복 제거가 필요 없는 경우 조기 반환으로 효율성 향상
        if (nums.length == 0) return 0;
        
        // 2. 고유한 요소를 저장할 위치를 가리키는 포인터 k 초기화
        // why: k는 다음 고유 요소가 들어갈 인덱스이자 고유 요소의 개수
        int k = 1;
        
        // 3. 배열을 순회하며 이전 요소와 다른 경우에만 k 위치에 저장
        // why: 정렬된 배열이므로 바로 이전 요소와만 비교하면 중복 판별 가능
        for (int i = 1; i < nums.length; i++) {
            if (nums[i] != nums[i - 1]) {
                nums[k] = nums[i];
                k++;
            }
        }
        
        // 4. 고유한 요소의 개수 k를 반환
        // why: 앞쪽 k개 요소가 중복 없는 정렬된 상태로 유지됨
        return k;
    }
}
```

**시간 복잡도**: O(n) - 배열을 한 번 순회  
**공간 복잡도**: O(1) - 추가 공간 사용 없이 in-place로 처리

## 면접 대본 (영어)

"Let me explain my approach to removing duplicates from this sorted array.

The key insight here is that since the array is already sorted, all duplicate values will be adjacent to each other. This means I only need to compare each element with its immediate predecessor to determine if it's a duplicate.

I'll use a two-pointer technique. The pointer k represents the position where I should place the next unique element. I initialize k to 1 because the first element is always unique and doesn't need to be checked.

Here's how the algorithm works: I iterate through the array starting from index 1. For each element at position i, I compare it with the element at position i-1. If they're different, it means I've found a new unique element, so I place it at position k and increment k. If they're the same, it's a duplicate, so I simply skip it and continue to the next element.

Let me walk through an example: if we have nums = [0,0,1,1,1,2,2,3,3,4], initially k is 1. When i=1, nums[1] equals nums[0] (both are 0), so we skip it. When i=2, nums[2] is 1 which differs from nums[1], so we place 1 at position k=1 and increment k to 2. We continue this process, and eventually k becomes 5, with the first five positions containing [0,1,2,3,4].

The time complexity is O(n) where n is the length of the array, since we make a single pass through it. The space complexity is O(1) because we only use a constant amount of extra space for the pointer k. This solution efficiently handles the in-place requirement while maintaining the sorted order of unique elements."

## 핵심 아이디어

정렬된 배열에서는 바로 이전 요소와만 비교하면 중복 판별이 가능하므로 Two Pointer로 O(n) 시간에 in-place 제거 가능
