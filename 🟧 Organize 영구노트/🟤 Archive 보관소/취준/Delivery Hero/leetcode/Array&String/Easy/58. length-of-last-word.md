#String

## 풀이 코드
```java
class Solution {
    public int lengthOfLastWord(String s) {
        // 1. 문자열의 끝에서부터 시작하는 포인터와 길이 변수 초기화
        // why: 마지막 단어를 찾기 위해 뒤에서부터 탐색
        int length = 0;
        int i = s.length() - 1;
        
        // 2. 끝에서부터 공백을 건너뜀
        // why: 문자열 끝에 trailing 공백이 있을 수 있음
        while (i >= 0 && s.charAt(i) == ' ') {
            i--;
        }
        
        // 3. 공백이 아닌 문자를 만나면 길이를 세기 시작
        // why: 마지막 단어의 문자들을 카운트
        while (i >= 0 && s.charAt(i) != ' ') {
            length++;
            i--;
        }
        
        // 4. 마지막 단어의 길이 반환
        // why: 공백을 만나거나 문자열의 시작에 도달하면 카운트 완료
        return length;
    }
}
```

**시간 복잡도**: O(n) - 최악의 경우 문자열 전체를 순회 (n은 문자열 길이)  
**공간 복잡도**: O(1) - 길이 변수와 인덱스만 사용

## 면접 대본 (영어)

"Let me explain my approach to finding the length of the last word in a string.

The key insight is that we should start from the end of the string and work backwards. This is more efficient than processing the entire string from the beginning, especially when there are trailing spaces.

The algorithm works in two phases. First, I skip any trailing spaces at the end of the string. This handles cases like 'Hello World  ' where there are extra spaces after the last word. I use a pointer i starting from the last index and decrement it while we encounter space characters.

Second, once I've skipped the trailing spaces, I start counting the characters of the last word. I continue moving backwards and incrementing a length counter until I either hit another space character or reach the beginning of the string. At that point, the length counter holds the length of the last word.

Let me walk through an example: if we have s = '   fly me   to   the moon  ', initially i is at the last index. First, we skip the two trailing spaces, moving i backward. Then we encounter 'n' from 'moon', and we start counting: n, o, o, m - that's 4 characters. When we hit the space before 'moon', we stop and return 4.

Another example: if we have s = 'Hello World', i starts at the end at 'd'. There are no trailing spaces to skip, so we immediately start counting: d, l, r, o, W - that's 5 characters. We hit the space and return 5.

The time complexity is O(n) in the worst case where the entire string is one word with no spaces, requiring us to traverse the full length. However, in typical cases, it's much faster since we only process from the end until we find the last word. The space complexity is O(1) because we only use two variables regardless of input size. This solution is efficient and handles edge cases like trailing spaces elegantly."
