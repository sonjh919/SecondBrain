#Array #PrefixSum

## 풀이 코드
```java
class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] answer = new int[n];
        
        // 1. 왼쪽에서 오른쪽으로 누적 곱 계산
        // why: answer[i]에 nums[0]부터 nums[i-1]까지의 곱을 저장
        answer[0] = 1;
        for (int i = 1; i < n; i++) {
            answer[i] = answer[i - 1] * nums[i - 1];
        }
        
        // 2. 오른쪽에서 왼쪽으로 누적 곱을 계산하며 최종 결과 생성
        // why: 오른쪽 곱을 누적하며 answer[i]에 곱하면 자신을 제외한 모든 곱
        int rightProduct = 1;
        for (int i = n - 1; i >= 0; i--) {
            answer[i] *= rightProduct;
            rightProduct *= nums[i];
        }
        
        // 3. 자신을 제외한 모든 원소의 곱이 담긴 배열 반환
        // why: 왼쪽 곱 × 오른쪽 곱 = 자신을 제외한 전체 곱
        return answer;
    }
}
```

**시간 복잡도**: O(n) - 배열을 두 번 순회 (n은 배열 길이)  
**공간 복잡도**: O(1) - 결과 배열 제외 시 추가 공간 사용 없음

## 면접 대본 (영어)

"Let me explain my approach to calculating the product of all elements except self without using division.

The key insight is to use prefix and suffix products. For each position i, the result is the product of all elements to its left multiplied by the product of all elements to its right. We can compute this in two passes: first calculate left products, then calculate right products while building the final answer.

I'll use the output array itself to store intermediate results, achieving O(1) extra space.

Here's how the algorithm works: In the first pass from left to right, I store in answer[i] the product of all elements to the left of i. For i=0, there are no elements to the left, so answer[0]=1. For i>0, answer[i] equals answer[i-1] times nums[i-1], which is the cumulative product of all elements before i.

In the second pass from right to left, I maintain a variable rightProduct that accumulates the product of all elements to the right. At each position i, I multiply answer[i] (which contains the left product) by rightProduct (the right product), giving us the product of all elements except nums[i]. Then I update rightProduct by multiplying it with nums[i] for the next iteration.

Let me walk through an example: nums = [1,2,3,4]. First pass (left products): answer[0]=1, answer[1]=1*1=1, answer[2]=1*2=2, answer[3]=2*3=6. So answer=[1,1,2,6]. Second pass (right products): Start with rightProduct=1. At i=3: answer[3]=6*1=6, rightProduct=1*4=4. At i=2: answer[2]=2*4=8, rightProduct=4*3=12. At i=1: answer[1]=1*12=12, rightProduct=12*2=24. At i=0: answer[0]=1*24=24, rightProduct=24*1=24. Final answer=[24,12,8,6].

Let's verify: For index 0, product except self = 2*3*4 = 24. For index 1, 1*3*4 = 12. For index 2, 1*2*4 = 8. For index 3, 1*2*3 = 6. Perfect!

Another example: nums = [-1,1,0,-3,3]. First pass: answer=[1,-1,-1,0,0]. Second pass: rightProduct=1. At i=4: answer[4]=0*1=0, right=3. At i=3: answer[3]=0*3=0, right=-9. At i=2: answer[2]=-1*-9=9, right=0. At i=1: answer[1]=-1*0=0, right=0. At i=0: answer[0]=1*0=0, right=0. Result=[0,0,9,0,0].

Why does this work? The product of all elements except nums[i] equals (product of elements before i) × (product of elements after i). By separating this into two passes, we avoid the need for division and handle zeros correctly. If there's a zero at any position, all positions except that zero's position will have 0 as the result.

An important edge case: multiple zeros. If nums = [0,0], first pass gives answer=[1,0]. Second pass: at i=1, answer[1]=0*1=0, right=0. At i=0, answer[0]=1*0=0, right=0. Result=[0,0], which is correct - product excluding each zero is still 0 because of the other zero.

The time complexity is O(n) where n is the array length, since we make two passes through the array. The space complexity is O(1) if we don't count the output array, as required by the follow-up. We only use one extra variable (rightProduct) regardless of input size.

This problem elegantly demonstrates prefix-suffix decomposition. Instead of computing the full product and dividing by each element (which doesn't work with zeros and is prohibited), we decompose the problem into left and right products. The two-pass approach with the output array serving double duty as storage for intermediate results is a clever space optimization."

## 핵심 아이디어

왼쪽에서 오른쪽으로 누적 곱을 계산한 후, 오른쪽에서 왼쪽으로 누적 곱을 곱하면 자신을 제외한 모든 곱을 O(n) 시간 O(1) 공간에 계산
