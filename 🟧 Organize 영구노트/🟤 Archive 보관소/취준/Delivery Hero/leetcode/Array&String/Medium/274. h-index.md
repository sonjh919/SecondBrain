#Array #Sorting #CountingSort

## 풀이 코드
```java
class Solution {
    public int hIndex(int[] citations) {
        // 1. 인용 수를 내림차순으로 정렬
        // why: 높은 인용 수부터 확인하여 h-index 조건 검사
        Arrays.sort(citations);
        
        int n = citations.length;
        
        // 2. 정렬된 배열을 뒤에서부터 순회
        // why: 가장 많이 인용된 논문부터 h-index 조건 확인
        for (int i = 0; i < n; i++) {
            // 3. 현재 위치에서 h-index 후보 계산
            // why: 뒤에서 i+1번째 논문이므로 i+1편의 논문이 있음
            int h = n - i;
            
            // 4. 현재 논문의 인용 수가 h 이상이면 h-index 찾음
            // why: h편의 논문이 각각 h번 이상 인용됨을 의미
            if (citations[i] >= h) {
                return h;
            }
        }
        
        // 5. 조건을 만족하는 h를 찾지 못하면 0 반환
        // why: 모든 논문의 인용 수가 0인 경우
        return 0;
    }
}
```

**시간 복잡도**: O(n log n) - 정렬에 소요되는 시간 (n은 논문 수)  
**공간 복잡도**: O(1) - 추가 공간 사용 없음 (정렬은 in-place)

## 면접 대본 (영어)

"Let me explain my approach to calculating the h-index.

The key insight is to sort the citations array and then find the largest h such that we have at least h papers with h or more citations. By sorting, we can efficiently check this condition from the highest citations downward.

The h-index definition says: a researcher has h-index h if they have h papers with at least h citations each, and the remaining papers have at most h citations each. The critical observation is that once we sort the papers by citation count, we can check this condition systematically.

I'll sort the array first, then iterate through it to find the maximum valid h.

Here's how the algorithm works: After sorting citations in ascending order, I iterate from the beginning. At each position i, I calculate h as n - i, which represents the number of papers from position i to the end. If citations[i] is greater than or equal to h, it means we have at least h papers (from i onward) with at least h citations each. This is our h-index, and I return it immediately.

If we complete the loop without returning, it means no valid h-index was found, so we return 0.

Let me walk through an example: citations = [3,0,6,1,5]. After sorting: [0,1,3,5,6]. Now I iterate: At i=0, h = 5-0 = 5, citations[0] = 0 < 5, continue. At i=1, h = 5-1 = 4, citations[1] = 1 < 4, continue. At i=2, h = 5-2 = 3, citations[2] = 3 >= 3, return 3.

Let's verify this is correct: We have papers with citations [0,1,3,5,6]. Taking the top 3 papers [3,5,6], each has at least 3 citations. The remaining papers [0,1] have at most 3 citations. So h-index is 3.

Another example: citations = [1,3,1]. After sorting: [1,1,3]. At i=0, h=3, citations[0]=1 < 3. At i=1, h=2, citations[1]=1 < 2. At i=2, h=1, citations[2]=3 >= 1, return 1. Indeed, we have 1 paper with at least 1 citation (actually we have 3, but h-index is the maximum h satisfying the condition).

An edge case: citations = [0]. After sorting: [0]. At i=0, h=1, citations[0]=0 < 1. We exit the loop and return 0, which is correct - no papers have any citations.

Another edge case: citations = [100]. After sorting: [100]. At i=0, h=1, citations[0]=100 >= 1, return 1. Correct - we have 1 paper with at least 1 citation, so h-index is 1, not 100. The h-index is capped by the number of papers.

Why does sorting make this work? After sorting, for any position i, all papers from i onward have citations greater than or equal to citations[i]. So if citations[i] >= h where h is the count of papers from i to the end, we've found a valid h-index. We check from smallest to largest citations because we want the maximum h.

Actually, let me reconsider the iteration direction. Since we want the maximum h, and larger h values are found when we have more papers meeting the threshold, we should check from the left (smallest citations) because that's where we have the most papers remaining. The first time citations[i] >= n-i is the maximum valid h.

The time complexity is O(n log n) due to sorting, where n is the number of papers. The iteration is O(n) but dominated by sorting. The space complexity is O(1) if we sort in-place, or O(n) if the sorting algorithm uses extra space.

An alternative approach would use counting sort since citations are bounded (0 to 1000). We could count how many papers have each citation count, then work backwards to find the h-index in O(n) time. But for the given constraints, the sorting approach is simpler and sufficiently efficient.

This problem elegantly demonstrates how sorting can transform a seemingly complex condition into a straightforward linear check. The h-index is a real metric used in academia to measure research impact, making this problem particularly interesting."
