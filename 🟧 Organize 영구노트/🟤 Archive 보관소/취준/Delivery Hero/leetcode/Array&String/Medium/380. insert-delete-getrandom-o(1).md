## 풀이 코드
```java
class RandomizedSet {
    // 1. 값과 인덱스를 매핑하는 HashMap과 값을 저장하는 ArrayList
    // why: HashMap으로 O(1) 검색, ArrayList로 O(1) 랜덤 접근
    private Map<Integer, Integer> map;
    private List<Integer> list;
    private Random random;

    public RandomizedSet() {
        // 2. 자료구조 초기화
        // why: 삽입/삭제/조회를 위한 준비
        map = new HashMap<>();
        list = new ArrayList<>();
        random = new Random();
    }
    
    public boolean insert(int val) {
        // 3. 이미 존재하면 false 반환
        // why: 중복 삽입 방지
        if (map.containsKey(val)) {
            return false;
        }
        
        // 4. list 끝에 값 추가하고 map에 인덱스 저장
        // why: ArrayList의 끝에 추가는 O(1), map에 위치 기록
        map.put(val, list.size());
        list.add(val);
        return true;
    }
    
    public boolean remove(int val) {
        // 5. 존재하지 않으면 false 반환
        // why: 없는 값은 삭제 불가
        if (!map.containsKey(val)) {
            return false;
        }
        
        // 6. 삭제할 값의 인덱스와 마지막 값 가져오기
        // why: 마지막 요소와 swap하여 O(1) 삭제 구현
        int index = map.get(val);
        int lastElement = list.get(list.size() - 1);
        
        // 7. 삭제할 위치에 마지막 값을 이동
        // why: 중간 삭제 대신 끝 삭제로 O(1) 달성
        list.set(index, lastElement);
        map.put(lastElement, index);
        
        // 8. 마지막 요소 제거 및 map에서 삭제
        // why: list와 map 모두에서 제거
        list.remove(list.size() - 1);
        map.remove(val);
        
        return true;
    }
    
    public int getRandom() {
        // 9. list에서 랜덤 인덱스 선택하여 반환
        // why: ArrayList는 인덱스 접근이 O(1)이고 균등 분포 보장
        return list.get(random.nextInt(list.size()));
    }
}
```

**시간 복잡도**: O(1) - 모든 연산이 평균 O(1) 시간  
**공간 복잡도**: O(n) - n개 원소를 HashMap과 ArrayList에 저장

## 면접 대본 (영어)

"Let me explain my approach to implementing a randomized set with O(1) operations.

The key insight is to combine two data structures: a HashMap for O(1) lookup and an ArrayList for O(1) random access. The HashMap stores value-to-index mappings, while the ArrayList stores the actual values. This dual structure allows us to achieve O(1) for all operations.

The challenging part is the remove operation. Removing from the middle of an ArrayList is O(n), but we can make it O(1) by swapping the element to remove with the last element, then removing the last element.

Let me explain each operation:

For insert: I first check if the value already exists in the map. If it does, I return false. If not, I add the value to the end of the list, record its index in the map, and return true. Adding to the end of an ArrayList is O(1) amortized.

For remove: I first check if the value exists. If not, return false. If it exists, I get its index from the map. Here's the clever part: I swap the element at that index with the last element in the list. To do this, I get the last element, place it at the index of the element to remove, and update the map to reflect this last element's new position. Then I remove the last element from the list (which is O(1)) and remove the value from the map. This way, we avoid the O(n) cost of removing from the middle.

For getRandom: I simply generate a random index between 0 and list.size()-1, and return the element at that index. Since ArrayList provides O(1) index access, this is efficient. Each element has equal probability of being selected since we're using uniform random distribution.

Let me walk through an example: Start with empty set. Insert 1: list=[1], map={1:0}. Insert 2: list=[1,2], map={1:0, 2:1}. Remove 1: We swap 1 with the last element (2), but they're at different positions. Actually, we swap 1 (at index 0) with 2 (at index 1). After swap: list=[2,1], map={1:1, 2:0}. Then remove last: list=[2], map={2:0}. Insert 3: list=[2,3], map={2:0, 3:1}.

Let me trace a more complex example: list=[5,2,8], map={5:0, 2:1, 8:2}. Remove 2: index=1, lastElement=8. Set list[1]=8: list=[5,8,8]. Update map: {5:0, 2:1, 8:1}. Remove last element: list=[5,8], remove from map: {5:0, 8:1}. Perfect!

An important edge case: what if we remove the last element itself? Say list=[5,8], remove 8. Index=1, lastElement=8. We set list[1]=8 (no change), update map for 8 to index 1 (no change), then remove last element. This works correctly because we're effectively swapping an element with itself.

Why does this maintain equal probability for getRandom? At any point, every element in the list has exactly one entry, and random.nextInt gives uniform distribution over valid indices. So each element has probability 1/n where n is the current size.

The time complexity is O(1) average for all operations. HashMap operations (get, put, containsKey, remove) are O(1) average. ArrayList operations we use - add to end, set at index, get at index, remove from end - are all O(1). The space complexity is O(n) where n is the number of elements, since we store each element twice (once in map, once in list).

This problem beautifully demonstrates how combining data structures can achieve properties that neither structure has alone. HashMap alone can't give us random access. ArrayList alone can't give us O(1) existence check or O(1) removal. Together, they complement each other perfectly."
