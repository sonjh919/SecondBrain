#Array #Math #TwoPointers

## 풀이 코드
```java
class Solution {
    public void rotate(int[] nums, int k) {
        // 1. k를 배열 길이로 나눈 나머지 계산
        // why: k가 배열 길이보다 크면 실제로는 k % n만큼만 회전하면 됨
        int n = nums.length;
        k = k % n;
        
        // 2. k가 0이면 회전할 필요 없음
        // why: 배열이 원래 상태 그대로
        if (k == 0) return;
        
        // 3. 전체 배열을 뒤집기
        // why: [1,2,3,4,5,6,7] -> [7,6,5,4,3,2,1]
        reverse(nums, 0, n - 1);
        
        // 4. 처음 k개 원소를 뒤집기
        // why: [7,6,5,4,3,2,1] -> [5,6,7,4,3,2,1]
        reverse(nums, 0, k - 1);
        
        // 5. 나머지 원소들을 뒤집기
        // why: [5,6,7,4,3,2,1] -> [5,6,7,1,2,3,4]
        reverse(nums, k, n - 1);
    }
    
    // 6. 배열의 특정 범위를 뒤집는 헬퍼 함수
    // why: 두 포인터를 이용한 in-place 뒤집기
    private void reverse(int[] nums, int start, int end) {
        while (start < end) {
            int temp = nums[start];
            nums[start] = nums[end];
            nums[end] = temp;
            start++;
            end--;
        }
    }
}
```

**시간 복잡도**: O(n) - 배열을 세 번 순회하지만 여전히 선형 시간  
**공간 복잡도**: O(1) - 추가 배열 없이 in-place로 처리

## 면접 대본 (영어)

"Let me explain my approach to rotating an array to the right by k positions.

The key insight is to use a three-reversal technique. Instead of actually moving elements k positions, we can achieve the same result by reversing different parts of the array. This elegant approach works in-place with O(1) extra space.

The algorithm has three steps: first, reverse the entire array. Second, reverse the first k elements. Third, reverse the remaining elements. This might seem counterintuitive, but let me show you why it works.

First, I handle an optimization: if k is larger than the array length, rotating by k is the same as rotating by k mod n, since rotating by n brings us back to the start. So I compute k = k % n to avoid unnecessary work.

Here's why the three reversals work: When we rotate right by k, the last k elements move to the front, and the first n-k elements shift right. By reversing the entire array, we bring the last k elements to the front, but in wrong order. Then by reversing the first k elements, we correct their order. Finally, by reversing the remaining n-k elements, we correct their order too.

Let me walk through an example: nums = [1,2,3,4,5,6,7], k = 3. First, k % 7 = 3, so k stays 3. Step 1: Reverse entire array: [7,6,5,4,3,2,1]. Step 2: Reverse first 3 elements: [5,6,7,4,3,2,1]. Step 3: Reverse remaining 4 elements: [5,6,7,1,2,3,4]. This is exactly the result we want!

Another example: nums = [-1,-100,3,99], k = 2. First k % 4 = 2. Step 1: [99,3,-100,-1]. Step 2: [3,99,-100,-1]. Step 3: [3,99,-1,-100]. Perfect!

Why does this mathematical trick work? Think about it: the last k elements need to become the first k elements. When we reverse the entire array, elements that were at positions n-k, n-k+1, ..., n-1 are now at positions k-1, k-2, ..., 0. But they're in reverse order. So we reverse them again to get the correct order. Similarly, the first n-k elements are now at positions n-1, n-2, ..., k, also in reverse order, so we reverse them to fix it.

An edge case: k = 0. We handle this by checking if k is 0 after the modulo operation and returning early if so.

Another edge case: k equals n. After k = k % n, k becomes 0, so we return early. The array stays unchanged, which is correct.

The reverse helper function uses two pointers starting from opposite ends, swapping elements and moving towards the center. This is a standard in-place reversal technique.

The time complexity is O(n) where n is the array length. We reverse the array three times, and each reversal is O(n), but 3n is still O(n). The space complexity is O(1) because we only use a constant amount of extra space for the swap operation and pointers.

For the follow-up about alternative solutions: Method 1 is using extra space - create a new array and place elements in their rotated positions. This is O(n) space. Method 2 is cyclic replacements - start from index 0 and move the element k positions right repeatedly until we return to the start, handling cycles correctly. This is O(1) space but more complex. Method 3 is our reversal approach - elegant, O(1) space, and easy to understand.

This problem beautifully demonstrates how a clever mathematical insight - that reversals can simulate rotation - can transform an apparently difficult in-place operation into a simple, elegant solution. The three-reversal technique is a classic algorithm worth remembering."
