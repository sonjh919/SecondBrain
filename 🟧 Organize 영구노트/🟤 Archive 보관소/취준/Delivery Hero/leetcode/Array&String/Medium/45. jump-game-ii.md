#Array #DynamicProgramming #Greedy

## 풀이 코드
```java
class Solution {
    public int jump(int[] nums) {
        // 1. 이미 마지막 위치에 있으면 점프 불필요
        // why: 길이가 1이면 점프할 필요 없음
        if (nums.length <= 1) {
            return 0;
        }
        
        // 2. 점프 횟수와 현재 점프로 도달 가능한 범위 초기화
        // why: 최소 점프 수를 세고 현재/다음 점프 범위를 추적
        int jumps = 0;
        int currentEnd = 0;
        int farthest = 0;
        
        // 3. 마지막 인덱스 직전까지 순회
        // why: 마지막 인덱스는 도달 목표이므로 -1까지만 확인
        for (int i = 0; i < nums.length - 1; i++) {
            // 4. 현재 위치에서 도달 가능한 최대 거리 업데이트
            // why: 다음 점프에서 갈 수 있는 가장 먼 위치 추적
            farthest = Math.max(farthest, i + nums[i]);
            
            // 5. 현재 점프 범위의 끝에 도달하면 점프 수행
            // why: 현재 점프로 갈 수 있는 범위를 다 확인했으므로 다음 점프
            if (i == currentEnd) {
                jumps++;
                currentEnd = farthest;
                
                // 6. 이미 마지막 인덱스에 도달 가능하면 조기 종료
                // why: 더 이상 점프할 필요 없음
                if (currentEnd >= nums.length - 1) {
                    break;
                }
            }
        }
        
        // 7. 최소 점프 횟수 반환
        // why: 모든 범위를 확인하며 센 점프 수
        return jumps;
    }
}
```

**시간 복잡도**: O(n) - 배열을 한 번 순회 (n은 배열 길이)  
**공간 복잡도**: O(1) - 세 개의 변수만 사용

## 면접 대본 (영어)

"Let me explain my approach to finding the minimum number of jumps to reach the last index.

The key insight is to use a greedy BFS-like approach. We think of the problem in terms of levels or layers - each jump creates a new layer of reachable positions. Within each layer, we explore all positions to find the farthest we can reach with the next jump. We only increment the jump count when we've exhausted the current layer.

I'll track three variables: jumps for counting jumps, currentEnd for the boundary of the current jump's reach, and farthest for the maximum distance achievable with the next jump.

Here's how the algorithm works: I iterate through the array up to the second-to-last index (we don't need to check the last index since it's our destination). At each position i, I update farthest to be the maximum of its current value and i + nums[i]. This tracks the farthest position we can reach with one more jump from anywhere in the current range.

The crucial part: when i reaches currentEnd, it means we've explored all positions reachable with the current number of jumps. At this point, we must make another jump to continue. So I increment jumps and update currentEnd to farthest. This establishes a new layer of reachable positions.

As an optimization, if currentEnd reaches or exceeds the last index after a jump, we can break early since we've found our answer.

Let me walk through an example: nums = [2,3,1,1,4]. Initially jumps=0, currentEnd=0, farthest=0. At i=0, farthest = max(0, 0+2) = 2. Now i equals currentEnd (both 0), so we jump: jumps=1, currentEnd=2. At i=1, farthest = max(2, 1+3) = 4. At i=2, farthest = max(4, 2+1) = 4. Now i equals currentEnd (both 2), so we jump: jumps=2, currentEnd=4. Since currentEnd (4) >= last index (4), we break. Return 2.

Let's trace why this works: The first jump from index 0 can reach indices 1 or 2 (since nums[0]=2). While exploring these positions, we find that from index 1, we can reach up to index 4 (1+3=4). So with 2 jumps total, we can reach the end.

Another example: nums = [2,3,0,1,4]. Farthest=0, currentEnd=0. At i=0, farthest=2, i==currentEnd, so jumps=1, currentEnd=2. At i=1, farthest=max(2,4)=4. At i=2, farthest=max(4,2)=4, i==currentEnd, so jumps=2, currentEnd=4. Since 4>=4, break and return 2.

Why is this greedy approach optimal? At each layer, we're exploring all positions reachable with the current number of jumps and finding the absolute maximum reach for the next jump. We never jump prematurely - we always wait until we've explored the entire current layer. This guarantees we find the minimum number of jumps because we're essentially doing a breadth-first search where each level represents one jump.

An edge case: nums = [1]. Length is 1, so we return 0 immediately - we're already at the destination.

Another consideration: what if nums = [1,1,1,1]? At i=0, farthest=1, jump to currentEnd=1. At i=1, farthest=2, jump to currentEnd=2. At i=2, farthest=3, jump to currentEnd=3. We make 3 jumps, which is correct.

The time complexity is O(n) where n is the array length, since we traverse the array once. The space complexity is O(1) since we only use three variables.

This problem is a beautiful example of implicit BFS. We're not explicitly maintaining a queue of positions to visit, but the currentEnd variable effectively marks the boundary of each BFS level. By tracking the farthest reachable position within each level and only incrementing the jump count when we finish a level, we naturally find the shortest path in terms of number of jumps."
