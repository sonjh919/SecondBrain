#Array #DynamicProgramming #Greedy

## 풀이 코드
```java
class Solution {
    public boolean canJump(int[] nums) {
        // 1. 도달 가능한 최대 인덱스를 추적할 변수 초기화
        // why: 현재까지 갈 수 있는 가장 먼 위치를 기록
        int maxReach = 0;
        
        // 2. 배열을 순회하며 도달 가능한 범위 업데이트
        // why: 각 위치에서 도달 가능한 최대 거리를 계산
        for (int i = 0; i < nums.length; i++) {
            // 3. 현재 위치가 도달 가능한 범위를 벗어나면 false
            // why: 이 위치에 도달할 수 없으므로 끝까지 갈 수 없음
            if (i > maxReach) {
                return false;
            }
            
            // 4. 현재 위치에서 갈 수 있는 최대 거리로 maxReach 업데이트
            // why: i + nums[i]가 현재 위치에서 도달 가능한 최대 인덱스
            maxReach = Math.max(maxReach, i + nums[i]);
            
            // 5. 이미 마지막 인덱스에 도달 가능하면 조기 종료
            // why: 더 이상 확인할 필요 없음
            if (maxReach >= nums.length - 1) {
                return true;
            }
        }
        
        // 6. 루프를 무사히 통과하면 마지막 인덱스 도달 가능
        // why: 모든 위치가 도달 가능한 범위 내에 있었음
        return true;
    }
}
```

**시간 복잡도**: O(n) - 배열을 한 번 순회 (n은 배열 길이)  
**공간 복잡도**: O(1) - maxReach 변수만 사용

## 면접 대본 (영어)

"Let me explain my approach to determining if we can reach the last index by jumping.

The key insight is to use a greedy approach by tracking the farthest position we can reach at any point. We don't need to try all possible jump combinations - we just need to know if the last index falls within our reachable range. As we traverse the array, we continuously update the maximum reachable index based on the current position and its jump length.

I'll maintain a variable maxReach that represents the farthest index we can reach from any position we've visited so far. Initially, it's 0 since we start at index 0.

Here's how the algorithm works: I iterate through each index i in the array. Before processing index i, I first check if i is greater than maxReach. If it is, it means we can't actually reach position i from the start, so we immediately return false - there's no way to reach the end.

If we can reach position i, I calculate the farthest we can jump from here: i + nums[i]. I update maxReach to be the maximum of its current value and this new potential reach. This ensures maxReach always represents the farthest position reachable so far.

As an optimization, if at any point maxReach is greater than or equal to the last index, I can return true immediately - we've proven we can reach the end.

If we successfully traverse the entire array without being blocked, we return true.

Let me walk through an example: nums = [2,3,1,1,4]. Initially maxReach = 0. At i=0, 0 <= maxReach (0), so we continue. MaxReach becomes max(0, 0+2) = 2. At i=1, 1 <= maxReach (2), so we continue. MaxReach becomes max(2, 1+3) = 4. Now maxReach (4) >= last index (4), so we return true immediately.

Another example: nums = [3,2,1,0,4]. MaxReach = 0. At i=0, maxReach = max(0, 0+3) = 3. At i=1, maxReach = max(3, 1+2) = 3. At i=2, maxReach = max(3, 2+1) = 3. At i=3, maxReach = max(3, 3+0) = 3. At i=4, we check: is 4 > maxReach (3)? Yes! We return false - we can't reach index 4.

An edge case: nums = [0]. The only element is the last index, so we return true immediately - we're already at the end.

Another edge case: nums = [1,0,1,0]. At i=0, maxReach = 1. At i=1, maxReach stays 1 (since 1+0=1). At i=2, we check: is 2 > 1? Yes, return false. We're stuck at index 1.

Why does this greedy approach work? The key observation is that if we can reach position i, and from i we can reach position j, then we can definitely reach all positions between i and j as well. So we don't need to explicitly track which positions we can reach - just tracking the maximum reachable position is sufficient. If at any point we encounter a position beyond our maximum reach, we know it's impossible to proceed.

The time complexity is O(n) where n is the length of the array, since we make at most one pass through the array. We might terminate early if we prove reachability or impossibility. The space complexity is O(1) because we only use one variable to track the maximum reach.

This problem beautifully demonstrates the power of greedy algorithms. While you could solve this with dynamic programming by checking reachability for each position, or with BFS by exploring all possible jumps, the greedy approach is more elegant and efficient. By always tracking the farthest we can go, we naturally capture all reachable positions without explicitly enumerating them."
