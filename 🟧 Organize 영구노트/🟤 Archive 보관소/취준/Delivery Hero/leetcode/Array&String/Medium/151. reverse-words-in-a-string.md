#TwoPointers #String

## 풀이 코드
```java
class Solution {
    public String reverseWords(String s) {
        // 1. 앞뒤 공백 제거 및 단어 분리
        // why: trim()으로 leading/trailing 공백 제거, split()으로 단어 추출
        String[] words = s.trim().split("\\s+");
        
        // 2. 결과를 저장할 StringBuilder 초기화
        // why: 문자열 연결을 효율적으로 처리
        StringBuilder result = new StringBuilder();
        
        // 3. 배열을 뒤에서부터 순회하며 단어 추가
        // why: 역순으로 단어를 연결하여 순서 반전
        for (int i = words.length - 1; i >= 0; i--) {
            result.append(words[i]);
            
            // 4. 마지막 단어가 아니면 공백 추가
            // why: 단어 사이에만 하나의 공백 삽입
            if (i > 0) {
                result.append(" ");
            }
        }
        
        // 5. 역순으로 연결된 단어 문자열 반환
        // why: 공백이 정리되고 순서가 반전된 최종 결과
        return result.toString();
    }
}
```

**시간 복잡도**: O(n) - 문자열 길이만큼 처리 (n은 문자열 길이)  
**공간 복잡도**: O(n) - 단어 배열과 결과 문자열 저장

## 면접 대본 (영어)

"Let me explain my approach to reversing the order of words in a string.

The key insight is to use Java's built-in string methods to handle whitespace normalization, then build the result by iterating through words in reverse order. The trim and split methods elegantly handle the various whitespace cases mentioned in the problem.

I'll first clean the input string and extract words, then build the result by appending words in reverse order.

Here's how the algorithm works: First, I use trim() to remove any leading or trailing spaces from the string. Then I use split with the regex pattern '\\s+', which splits on one or more whitespace characters. This automatically handles multiple consecutive spaces between words, giving us a clean array of words.

Next, I create a StringBuilder for efficient string construction. I iterate through the words array from the last index to the first, appending each word to the result. Between words (but not after the last word), I append a single space. Finally, I convert the StringBuilder to a string and return it.

Let me walk through an example: s = 'the sky is blue'. After trim: 'the sky is blue'. After split: ['the', 'sky', 'is', 'blue']. Now iterating backwards: append 'blue', append ' ', append 'is', append ' ', append 'sky', append ' ', append 'the'. Result: 'blue is sky the'.

Another example: s = '  hello world  '. After trim: 'hello world'. After split: ['hello', 'world']. Backward iteration: append 'world', append ' ', append 'hello'. Result: 'world hello'. Notice how trim removed the leading and trailing spaces, and split didn't create empty strings.

A complex example: s = 'a good   example'. After trim: 'a good   example'. After split with '\\s+': ['a', 'good', 'example']. The regex '\\s+' matches one or more spaces, so the three spaces between 'good' and 'example' are treated as a single delimiter. Backward iteration: append 'example', append ' ', append 'good', append ' ', append 'a'. Result: 'example good a'.

Why does split with '\\s+' work so well? The regex '\\s+' matches one or more consecutive whitespace characters, treating them as a single delimiter. This means multiple spaces are automatically collapsed, and we don't get empty strings in our array. Combined with trim(), we handle all the edge cases about extra spaces.

An important detail: I check 'if (i > 0)' before adding a space. This ensures we don't add a trailing space after the last word. We want spaces between words, not after the final word.

The time complexity is O(n) where n is the length of the string. Trim is O(n), split is O(n) as it scans through the string once, and building the result is O(n) as we process each word once. The space complexity is O(n) for storing the words array and the result StringBuilder.

For the follow-up about in-place with O(1) space: In Java, strings are immutable, so true in-place modification isn't possible. However, in languages with mutable strings (like C++), you could: first reverse the entire string, then reverse each word individually, and finally clean up extra spaces by shifting characters. This would be O(1) extra space beyond the string itself.

This problem demonstrates effective use of Java's string utilities. While you could manually parse the string character by character with two pointers, using trim and split with regex is more concise and handles edge cases naturally. The backward iteration is straightforward once we have a clean array of words."

## 핵심 아이디어
