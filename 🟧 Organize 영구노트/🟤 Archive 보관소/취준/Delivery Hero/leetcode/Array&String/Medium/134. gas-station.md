#Array #Greedy

## 풀이 코드
```java
class Solution {
    public int canCompleteCircuit(int[] gas, int[] cost) {
        // 1. 전체 가스와 현재 탱크, 시작 위치 초기화
        // why: 전체 순회 가능성과 현재 경로의 가스량 추적
        int totalGas = 0;
        int currentGas = 0;
        int startIndex = 0;
        
        // 2. 각 주유소를 순회하며 가스 수지 계산
        // why: 각 구간에서 얻는 가스와 소비하는 가스의 차이 누적
        for (int i = 0; i < gas.length; i++) {
            int diff = gas[i] - cost[i];
            totalGas += diff;
            currentGas += diff;
            
            // 3. 현재 가스가 음수가 되면 다음 위치를 시작점으로 설정
            // why: 현재 시작점부터 i까지 갈 수 없으므로 i+1부터 재시작
            if (currentGas < 0) {
                startIndex = i + 1;
                currentGas = 0;
            }
        }
        
        // 4. 전체 가스 수지가 음수면 순회 불가능
        // why: 총 가스가 총 비용보다 적으면 어디서 시작해도 불가능
        return totalGas >= 0 ? startIndex : -1;
    }
}
```

**시간 복잡도**: O(n) - 배열을 한 번 순회 (n은 주유소 개수)  
**공간 복잡도**: O(1) - 세 개의 변수만 사용

## 면접 대본 (영어)

"Let me explain my approach to finding the starting gas station for a complete circuit.

The key insights are: First, if the total gas is less than total cost, it's impossible to complete the circuit from any starting point. Second, if we can't reach station j from station i, then any station between i and j also can't reach j - so we should start from j+1. This greedy observation allows us to find the answer in a single pass.

I'll track three variables: totalGas for the overall gas balance, currentGas for the current journey's gas, and startIndex for the potential starting station.

Here's how the algorithm works: I iterate through each station, calculating the difference between gas gained and cost spent at each station. I add this difference to both totalGas and currentGas. The totalGas accumulates the overall balance to check if a solution exists. The currentGas tracks whether we can reach the current station from our current start point.

If currentGas becomes negative at any station i, it means we cannot reach station i from the current startIndex. Here's the crucial greedy insight: if we can't reach station i from startIndex, then we also can't reach station i from any station between startIndex and i-1. Why? Because starting from any intermediate station would give us even less gas accumulated. So we reset: set startIndex to i+1 and reset currentGas to 0.

After the loop, if totalGas is non-negative, a solution exists, and startIndex is the answer. If totalGas is negative, no solution exists, return -1.

Let me walk through an example: gas = [1,2,3,4,5], cost = [3,4,5,1,2]. Station 0: diff = 1-3 = -2, totalGas = -2, currentGas = -2. Since currentGas < 0, startIndex = 1, currentGas = 0. Station 1: diff = 2-4 = -2, totalGas = -4, currentGas = -2 < 0, startIndex = 2, currentGas = 0. Station 2: diff = 3-5 = -2, totalGas = -6, currentGas = -2 < 0, startIndex = 3, currentGas = 0. Station 3: diff = 4-1 = 3, totalGas = -3, currentGas = 3. Station 4: diff = 5-2 = 3, totalGas = 0, currentGas = 6. Final: totalGas = 0 >= 0, return startIndex = 3.

Let's verify: Starting at 3: tank = 0+4 = 4, travel to 4: 4-1+5 = 8, travel to 0: 8-2+1 = 7, travel to 1: 7-3+2 = 6, travel to 2: 6-4+3 = 5, travel to 3: 5-5 = 0. Success!

Another example: gas = [2,3,4], cost = [3,4,3]. Station 0: diff = -1, totalGas = -1, currentGas = -1 < 0, start = 1, current = 0. Station 1: diff = -1, totalGas = -2, currentGas = -1 < 0, start = 2, current = 0. Station 2: diff = 1, totalGas = -1, currentGas = 1. Final: totalGas = -1 < 0, return -1. No solution exists.

Why does this greedy approach work? The critical observation is that if the total gas equals or exceeds total cost, there must be a valid starting point. By eliminating invalid starting points as we go (when currentGas goes negative), we eventually land on the valid one. We don't need to explicitly check that startIndex can complete the circuit because: if totalGas >= 0 and startIndex is the only candidate remaining after our eliminations, it must work.

An important property: once we set startIndex to some position i, we know positions 0 through i-1 cannot be valid starting points. Combined with totalGas >= 0 guaranteeing a solution exists, startIndex must be the answer.

The time complexity is O(n) where n is the number of stations, since we make a single pass through the arrays. The space complexity is O(1) because we only use three variables regardless of input size.

This problem beautifully demonstrates greedy algorithm design. Instead of trying all n starting positions (which would be O(n²)), we use mathematical reasoning about gas balances to find the answer in one pass. The key insight that failing at station i means all previous stations also fail is what makes this optimization possible."
