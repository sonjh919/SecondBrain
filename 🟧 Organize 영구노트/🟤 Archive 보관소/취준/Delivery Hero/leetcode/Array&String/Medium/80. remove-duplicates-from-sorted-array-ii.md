#Array #TwoPointers

## 풀이 코드
```java
class Solution {
    public int removeDuplicates(int[] nums) {
        // 1. 배열 길이가 2 이하면 그대로 반환
        // why: 최대 2번까지 허용하므로 길이가 2 이하면 중복 제거 불필요
        if (nums.length <= 2) return nums.length;
        
        // 2. 유효한 요소를 저장할 위치 포인터 k를 2로 초기화
        // why: 처음 두 요소는 항상 유효하므로 k는 2부터 시작
        int k = 2;
        
        // 3. 인덱스 2부터 순회하며 k-2 위치의 값과 비교
        // why: 최대 2번 허용이므로 2칸 전 요소와 다르면 3번째 중복이 아님
        for (int i = 2; i < nums.length; i++) {
            if (nums[i] != nums[k - 2]) {
                nums[k] = nums[i];
                k++;
            }
        }
        
        // 4. 조건을 만족하는 요소의 개수 k를 반환
        // why: 앞쪽 k개 요소가 각 값이 최대 2번까지만 나타나는 상태로 유지됨
        return k;
    }
}
```

**시간 복잡도**: O(n) - 배열을 한 번 순회  
**공간 복잡도**: O(1) - 추가 공간 사용 없이 in-place로 처리

## 면접 대본 (영어)

"Let me explain my approach to solving this problem where we need to allow each element to appear at most twice.

The key insight is similar to the previous remove duplicates problem, but with a twist. Instead of comparing with just the previous element, I need to compare with the element that's two positions back in our result array. This is because we're allowing up to two occurrences of each value.

I'll use a two-pointer technique. The pointer k represents the position where I should place the next valid element. I initialize k to 2 because the first two elements are always valid - even if they're duplicates, we allow up to two occurrences.

Here's how the algorithm works: I iterate through the array starting from index 2. For each element at position i, I compare it with the element at position k-2. If they're different, it means adding this element won't create a third occurrence, so I place it at position k and increment k. If they're the same, it means this would be at least the third occurrence of this value, so I skip it.

Let me walk through an example: if we have nums = [1,1,1,2,2,3], initially k is 2. When i=2, nums[2] is 1, which equals nums[k-2]=nums[0]=1, so we skip it because it would be the third 1. When i=3, nums[3] is 2, which differs from nums[k-2]=nums[0]=1, so we place 2 at position k=2 and increment k to 3. When i=4, nums[4] is 2, which differs from nums[k-2]=nums[1]=1, so we place it at position k=3 and increment k to 4. When i=5, nums[5] is 3, which differs from nums[k-2]=nums[2]=2, so we place it at position k=4 and increment k to 5. Finally, we return k=5.

The time complexity is O(n) where n is the length of the array, since we make a single pass. The space complexity is O(1) because we only use a constant amount of extra space. The beauty of this solution is that by comparing with the element two positions back, we elegantly ensure no element appears more than twice."
