#String

## 풀이 코드
```java
class Solution {
    public String convert(String s, int numRows) {
        // 1. numRows가 1이거나 문자열 길이보다 크면 원본 반환
        // why: 지그재그 패턴이 의미 없는 경우
        if (numRows == 1 || numRows >= s.length()) {
            return s;
        }
        
        // 2. 각 행에 해당하는 문자를 저장할 StringBuilder 배열
        // why: 각 행별로 문자를 모아서 나중에 연결
        StringBuilder[] rows = new StringBuilder[numRows];
        for (int i = 0; i < numRows; i++) {
            rows[i] = new StringBuilder();
        }
        
        // 3. 현재 행과 방향을 추적하는 변수 초기화
        // why: 위아래로 이동하며 문자를 배치
        int currentRow = 0;
        boolean goingDown = false;
        
        // 4. 문자열의 각 문자를 순회하며 적절한 행에 배치
        // why: 지그재그 패턴에 따라 문자 분배
        for (char c : s.toCharArray()) {
            rows[currentRow].append(c);
            
            // 5. 첫 행이나 마지막 행에 도달하면 방향 전환
            // why: 지그재그의 꺾이는 지점
            if (currentRow == 0 || currentRow == numRows - 1) {
                goingDown = !goingDown;
            }
            
            // 6. 방향에 따라 다음 행으로 이동
            // why: 아래로 가면 +1, 위로 가면 -1
            currentRow += goingDown ? 1 : -1;
        }
        
        // 7. 모든 행을 순서대로 연결하여 결과 생성
        // why: 위에서 아래로 각 행을 읽어서 최종 문자열 생성
        StringBuilder result = new StringBuilder();
        for (StringBuilder row : rows) {
            result.append(row);
        }
        
        return result.toString();
    }
}
```

**시간 복잡도**: O(n) - 문자열을 한 번 순회 (n은 문자열 길이)  
**공간 복잡도**: O(n) - 각 행에 문자를 저장하는 StringBuilder 배열

## 면접 대본 (영어)

"Let me explain my approach to converting a string into a zigzag pattern.

The key insight is to simulate the zigzag movement by maintaining an array of StringBuilders, one for each row. As we traverse the input string, we append each character to the appropriate row based on our current position and direction. We alternate between going down and going up, changing direction when we hit the top or bottom row.

I'll use an array of StringBuilders to collect characters for each row, track the current row position, and track whether we're moving down or up.

Here's how the algorithm works: First, I handle edge cases - if numRows is 1 or greater than the string length, the zigzag pattern is meaningless, so I return the original string.

I create an array of StringBuilders, one for each row. Then I initialize currentRow to 0 (starting at the top) and goingDown to false (we'll flip it immediately to true when we start).

I iterate through each character in the string. For each character, I append it to the StringBuilder for the current row. Then I check if we're at a boundary - either row 0 (top) or row numRows-1 (bottom). If so, I flip the direction. Finally, I update currentRow by adding 1 if going down, or subtracting 1 if going up.

After processing all characters, I concatenate all rows in order from top to bottom to produce the final result.

Let me walk through an example: s = 'PAYPALISHIRING', numRows = 3. Initialize rows = [[], [], []]. Start with currentRow = 0, goingDown = false. 

Character 'P': append to row 0, at boundary (row 0), flip to goingDown = true, move to row 1. 
Character 'A': append to row 1, not at boundary, move to row 2. 
Character 'Y': append to row 2, at boundary (row 2), flip to goingDown = false, move to row 1. 
Character 'P': append to row 1, not at boundary, move to row 0. 
Character 'A': append to row 0, at boundary, flip to true, move to row 1.

Continuing this pattern: Row 0 gets: P, A, H, N. Row 1 gets: A, P, L, S, I, I, G. Row 2 gets: Y, I, R. Concatenating: 'PAHNAPLSIIGYIR'.

Another example: s = 'PAYPALISHIRING', numRows = 4. The zigzag goes: down 3 rows, up 2 rows (diagonal), down 3 rows, etc. Following the same logic: Row 0: P, I, N. Row 1: A, L, S, I, G. Row 2: Y, A, H, R. Row 3: P, I. Result: 'PINALSIGYAHRPI'.

Edge case: s = 'A', numRows = 1. Since numRows equals 1, we return 'A' immediately without processing.

Another edge case: s = 'AB', numRows = 3. Since numRows >= length, the string won't zigzag, so we return 'AB' directly.

Why does the direction flipping work? When we're at row 0, we must go down next. When we're at the bottom row, we must go up next. By checking these boundaries and flipping the direction, we naturally create the zigzag pattern. The currentRow update (+=1 or -=1) then moves us along that direction.

An important detail: I check the boundary condition before updating currentRow. This ensures that when we reach a boundary, we flip direction for the next character, not the current one.

The time complexity is O(n) where n is the length of the string, since we process each character exactly once. The space complexity is O(n) for storing the characters across all rows. In the worst case (numRows = 2), we split characters roughly evenly between two rows, still O(n).

This problem demonstrates how simulation can be an effective approach when the pattern is complex to express mathematically. While you could calculate the exact row for each character using a formula, simulating the zigzag movement is more intuitive and easier to implement correctly. The use of an array of StringBuilders is a classic technique for grouping elements by category before combining them."
