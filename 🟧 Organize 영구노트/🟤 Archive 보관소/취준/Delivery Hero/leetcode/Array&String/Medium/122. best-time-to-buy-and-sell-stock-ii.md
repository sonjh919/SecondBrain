#Array #DynamicProgramming #Greedy

## 풀이 코드
```java
class Solution {
    public int maxProfit(int[] prices) {
        // 1. 총 이익을 저장할 변수 초기화
        // why: 모든 수익성 있는 거래의 이익을 누적
        int maxProfit = 0;
        
        // 2. 배열을 순회하며 연속된 날의 가격 비교
        // why: 다음 날 가격이 오르면 거래로 이익 발생
        for (int i = 1; i < prices.length; i++) {
            // 3. 이전 날보다 가격이 오르면 그 차이만큼 이익 추가
            // why: 전날 사서 오늘 팔면 이익이므로 누적
            if (prices[i] > prices[i - 1]) {
                maxProfit += prices[i] - prices[i - 1];
            }
        }
        
        // 4. 누적된 최대 이익 반환
        // why: 모든 상승 구간의 이익을 합산한 결과
        return maxProfit;
    }
}
```

**시간 복잡도**: O(n) - 배열을 한 번 순회 (n은 배열 길이)  
**공간 복잡도**: O(1) - maxProfit 변수만 사용

## 면접 대본 (영어)

"Let me explain my approach to maximizing profit with multiple stock transactions.

The key insight is that we can capture all the profit by buying before every price increase and selling right after. Since we can buy and sell on the same day, we don't need to identify complete valleys and peaks - we can simply accumulate all the positive differences between consecutive days.

This is a greedy approach. The strategy is: whenever tomorrow's price is higher than today's, we buy today and sell tomorrow, capturing that price increase as profit. We repeat this for every consecutive day where the price goes up.

Here's why this works: imagine prices = [1,2,3,4,5]. The maximum profit is 5-1 = 4. But notice that buying at 1 and selling at 5 is equivalent to: buying at 1, selling at 2 (profit 1), buying at 2, selling at 3 (profit 1), buying at 3, selling at 4 (profit 1), buying at 4, selling at 5 (profit 1). Total profit is 1+1+1+1 = 4, the same! This is because (5-1) = (2-1) + (3-2) + (4-3) + (5-4).

The algorithm is simple: I initialize maxProfit to 0, then iterate through the array starting from index 1. At each position, I check if the current price is higher than the previous day's price. If it is, I add the difference to maxProfit. This difference represents the profit from buying yesterday and selling today.

Let me walk through an example: prices = [7,1,5,3,6,4]. Starting with maxProfit = 0. Day 1 to 2: 1 < 7, no profit. Day 2 to 3: 5 > 1, add 5-1=4, maxProfit = 4. Day 3 to 4: 3 < 5, no profit. Day 4 to 5: 6 > 3, add 6-3=3, maxProfit = 7. Day 5 to 6: 4 < 6, no profit. Return 7.

This correctly identifies two profitable transactions: buy at 1, sell at 5 (profit 4), and buy at 3, sell at 6 (profit 3).

Another example: prices = [1,2,3,4,5]. Day 1-2: add 1, Day 2-3: add 1, Day 3-4: add 1, Day 4-5: add 1. Total: 4. This is optimal - we capture the entire upward trend.

A failing case: prices = [7,6,4,3,1]. Every comparison shows the next price is lower, so we never add anything. Return 0, which is correct - we shouldn't buy at all in a declining market.

Why does this greedy approach work? The problem allows us to do as many transactions as we want, and we can even buy and sell on the same day. This means we're never punished for taking every small profit opportunity. In a continuous uptrend, it doesn't matter if we hold from bottom to top or if we buy and sell at every increment - the total profit is the same due to the additive property of differences.

The time complexity is O(n) where n is the length of the prices array, since we make a single pass through it. The space complexity is O(1) because we only use one variable to track the profit.

This problem is significantly different from 'Best Time to Buy and Sell Stock I' where we could only make one transaction. There, we needed to find the maximum difference between a buy and sell date. Here, with unlimited transactions, we can greedily take every profitable opportunity.

An interesting observation: this problem is essentially asking us to sum all positive differences in consecutive elements. It's a beautiful example of how a complex-sounding problem (multiple stock transactions) can be reduced to a simple greedy algorithm once you understand that you can decompose any large profit into a sum of smaller daily profits."
