## 풀이 코드
```java
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
    public List<Double> averageOfLevels(TreeNode root) {
        // 1. 결과를 저장할 리스트와 BFS를 위한 큐 생성
        // why: 레벨별 평균을 순서대로 저장하고 레벨 순회를 위해 큐 사용
        List<Double> result = new ArrayList<>();
        Queue<TreeNode> queue = new LinkedList<>();
        
        // 2. 루트 노드를 큐에 추가
        // why: BFS 시작점 설정
        queue.offer(root);
        
        // 3. 큐가 빌 때까지 레벨별로 순회
        // why: 모든 레벨을 처리할 때까지 반복
        while (!queue.isEmpty()) {
            // 4. 현재 레벨의 노드 개수 저장
            // why: 현재 레벨에 있는 노드들만 처리하기 위함
            int levelSize = queue.size();
            double sum = 0;
            
            // 5. 현재 레벨의 모든 노드를 처리
            // why: 같은 레벨의 노드들의 합을 계산
            for (int i = 0; i < levelSize; i++) {
                TreeNode node = queue.poll();
                sum += node.val;
                
                // 6. 다음 레벨의 노드들을 큐에 추가
                // why: 다음 레벨 처리를 위해 자식 노드들을 큐에 삽입
                if (node.left != null) {
                    queue.offer(node.left);
                }
                if (node.right != null) {
                    queue.offer(node.right);
                }
            }
            
            // 7. 현재 레벨의 평균을 계산하여 결과에 추가
            // why: 합을 노드 개수로 나누어 평균 계산
            result.add(sum / levelSize);
        }
        
        return result;
    }
}
```

**시간 복잡도**: O(n) - 모든 노드를 한 번씩 방문 (n은 노드 개수)  
**공간 복잡도**: O(w) - 큐에 저장되는 최대 노드 수 (w는 트리의 최대 너비)

## 면접 대본 (영어)

"Let me explain my approach to calculating the average value of nodes at each level in a binary tree.

The key insight is to use breadth-first search, also known as level-order traversal. BFS naturally processes the tree level by level, which is exactly what we need to calculate averages for each level separately.

I'll use a queue data structure to implement BFS. The queue follows FIFO order, which ensures we process nodes level by level from left to right.

Here's how the algorithm works: I start by initializing a result list to store the averages and a queue to facilitate the BFS. I add the root node to the queue to begin the traversal.

Then I enter a loop that continues as long as the queue isn't empty. At the start of each iteration, I capture the current size of the queue - this is crucial because it tells me exactly how many nodes are in the current level. I initialize a sum variable to accumulate the values of nodes at this level.

Next, I process exactly levelSize nodes - no more, no less. For each node, I poll it from the queue, add its value to the sum, and then add its children to the queue if they exist. This is important: by adding children after processing the current level, we ensure they'll be processed in the next iteration, maintaining the level-by-level order.

After processing all nodes in the current level, I calculate the average by dividing the sum by levelSize and add it to the result list. Then the loop continues to the next level.

Let me walk through an example: root = [3,9,20,null,null,15,7]. Initially, queue has [3]. Level 0: levelSize is 1, we process node 3, sum is 3, average is 3.0. We add 9 and 20 to the queue. Level 1: queue has [9,20], levelSize is 2. We process 9 (no children) and 20 (add 15,7), sum is 29, average is 14.5. Level 2: queue has [15,7], levelSize is 2, sum is 22, average is 11.0. Queue is empty, we return [3.0, 14.5, 11.0].

An important detail: I use double for the sum and division to ensure we get decimal precision. Even though node values are integers, the average might not be.

Another consideration: I need to be careful about integer overflow. The problem states node values can be as large as 2^31 - 1, and we might sum many of them. Using double for sum handles this naturally since double has much larger range than int.

The time complexity is O(n) where n is the number of nodes, because we visit each node exactly once. The space complexity is O(w) where w is the maximum width of the tree, which is the maximum number of nodes at any level. In the worst case for a complete binary tree, the last level could have n/2 nodes, so space complexity is O(n). However, for a balanced tree, it's typically much better.

This BFS approach is intuitive and natural for level-based problems. An alternative would be DFS with level tracking, but BFS is more straightforward when you explicitly need to process nodes level by level, as we do here for calculating averages."
