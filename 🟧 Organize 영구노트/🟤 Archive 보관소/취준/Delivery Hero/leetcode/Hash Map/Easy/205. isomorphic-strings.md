#HashTable #String

## 풀이 코드
```java
class Solution {
    public boolean isIsomorphic(String s, String t) {
        // 1. 두 개의 매핑 테이블 생성
        // why: s->t 매핑과 t->s 매핑을 모두 확인하여 일대일 대응 보장
        Map<Character, Character> mapS = new HashMap<>();
        Map<Character, Character> mapT = new HashMap<>();
        
        // 2. 두 문자열을 동시에 순회하며 매핑 관계 검증
        // why: 각 위치에서 문자 간 매핑이 일관성 있게 유지되는지 확인
        for (int i = 0; i < s.length(); i++) {
            char charS = s.charAt(i);
            char charT = t.charAt(i);
            
            // 3. s의 문자가 이미 매핑되어 있는지 확인
            // why: 같은 문자가 다른 문자로 매핑되면 안 됨
            if (mapS.containsKey(charS)) {
                if (mapS.get(charS) != charT) {
                    return false;
                }
            } else {
                mapS.put(charS, charT);
            }
            
            // 4. t의 문자가 이미 매핑되어 있는지 확인
            // why: 서로 다른 문자가 같은 문자로 매핑되면 안 됨 (일대일 대응)
            if (mapT.containsKey(charT)) {
                if (mapT.get(charT) != charS) {
                    return false;
                }
            } else {
                mapT.put(charT, charS);
            }
        }
        
        // 5. 모든 검증을 통과하면 isomorphic
        // why: 양방향 매핑이 모두 일관성 있게 유지됨
        return true;
    }
}
```

**시간 복잡도**: O(n) - 문자열을 한 번 순회 (n은 문자열 길이)  
**공간 복잡도**: O(k) - k는 고유 문자의 개수 (최악의 경우 O(n))

## 면접 대본 (영어)

"Let me explain my approach to checking if two strings are isomorphic.

The key insight is that isomorphic strings have a one-to-one character mapping. This means each character in s must consistently map to exactly one character in t, and vice versa. No two different characters in s can map to the same character in t.

I'll use two HashMaps to track the bidirectional mapping. One map tracks the mapping from characters in s to characters in t, and the other tracks the reverse mapping from t to s. This dual-map approach ensures we maintain a one-to-one correspondence.

Here's how the algorithm works: I iterate through both strings simultaneously, character by character. At each position, I have a character from s and a character from t. 

First, I check if the character from s has already been mapped. If it has, I verify that it maps to the current character from t. If it maps to a different character, that means one character in s is trying to map to two different characters, which violates the isomorphic property, so I return false.

Similarly, I check the reverse mapping. If the character from t has already been mapped to a character in s, I verify it's the current character from s. If not, it means two different characters from s are trying to map to the same character in t, which also violates the one-to-one property.

If neither character has been mapped yet, I add both mappings to their respective maps.

Let me walk through an example: if we have s = 'egg' and t = 'add', at i=0, 'e' maps to 'a' and 'a' maps to 'e'. At i=1, 'g' maps to 'd' and 'd' maps to 'g'. At i=2, we check 'g' - it's already mapped to 'd', which matches the current character, and 'd' is already mapped to 'g', which also matches. All checks pass, so we return true.

Another example: if we have s = 'foo' and t = 'bar', at i=0, 'f' maps to 'b'. At i=1, 'o' maps to 'a'. At i=2, we try to map 'o' to 'r', but 'o' is already mapped to 'a', not 'r'. This inconsistency means the strings are not isomorphic, so we return false.

A tricky case: s = 'badc' and t = 'baba'. At i=0, 'b' maps to 'b'. At i=1, 'a' maps to 'a'. At i=2, 'd' maps to 'b'. But wait - when we check the reverse mapping, 'b' is already mapped to 'b' from position 0, not to 'd'. This violates the one-to-one property, so we return false. This is why we need both maps.

The time complexity is O(n) where n is the length of the strings, since we make a single pass. The space complexity is O(k) where k is the number of unique characters, which in the worst case could be O(n) if all characters are unique. This solution correctly handles all edge cases including the bidirectional constraint."

## 핵심 아이디어
