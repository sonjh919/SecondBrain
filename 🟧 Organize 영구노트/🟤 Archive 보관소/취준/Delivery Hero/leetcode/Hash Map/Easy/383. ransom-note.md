#HashTable #String #Counting

## 풀이 코드
```java
class Solution {
    public boolean canConstruct(String ransomNote, String magazine) {
        // 1. ransomNote가 magazine보다 길면 false 반환
        // why: magazine의 문자로 더 긴 ransomNote를 만들 수 없음
        if (ransomNote.length() > magazine.length()) return false;
        
        // 2. 각 문자의 출현 횟수를 저장할 배열 생성
        // why: 소문자 알파벳만 사용하므로 크기 26의 배열로 충분
        int[] charCount = new int[26];
        
        // 3. magazine의 각 문자 출현 횟수를 카운트
        // why: magazine에 있는 문자들의 사용 가능한 개수 파악
        for (char c : magazine.toCharArray()) {
            charCount[c - 'a']++;
        }
        
        // 4. ransomNote의 각 문자를 사용하면서 카운트 감소
        // why: magazine에서 문자를 하나씩 사용하는 과정을 시뮬레이션
        for (char c : ransomNote.toCharArray()) {
            charCount[c - 'a']--;
            // 5. 카운트가 음수가 되면 해당 문자가 부족한 것
            // why: magazine에 없거나 이미 모두 사용한 문자를 요구하는 경우
            if (charCount[c - 'a'] < 0) {
                return false;
            }
        }
        
        // 6. 모든 문자를 성공적으로 사용했으면 true 반환
        // why: ransomNote의 모든 문자를 magazine에서 찾을 수 있음
        return true;
    }
}
```

**시간 복잡도**: O(m + n) - m은 magazine 길이, n은 ransomNote 길이 (각각 한 번씩 순회)  
**공간 복잡도**: O(1) - 크기 26의 고정 배열 사용 (알파벳 개수는 상수)

## 면접 대본 (영어)

"Let me explain my approach to checking if a ransom note can be constructed from a magazine.

The key insight is that we need to verify if the magazine contains enough of each character that appears in the ransom note. Each character in the magazine can only be used once, so we need to track the count of available characters.

First, I handle an optimization: if the ransom note is longer than the magazine, it's impossible to construct it, so I return false immediately.

I'll use a frequency counting approach with an array. Since we're only dealing with lowercase English letters, I can use an array of size 26 where each index represents a letter from 'a' to 'z'.

Here's how the algorithm works: First, I iterate through the magazine and count the frequency of each character by incrementing the corresponding index in my array. This gives me a inventory of available characters.

Then, I iterate through the ransom note. For each character, I decrement the count in my array, simulating the use of that character from the magazine. If at any point the count becomes negative, it means we don't have enough of that character available in the magazine, so I return false immediately.

If I successfully process all characters in the ransom note without any count going negative, it means the magazine has sufficient characters to construct the ransom note, so I return true.

Let me walk through an example: if we have ransomNote = 'aa' and magazine = 'aab', I first count the magazine: 'a' appears twice, 'b' appears once, so my array has [2, 1, 0, 0, ...]. Then I process the ransom note: first 'a' decrements the count to [1, 1, 0, ...], second 'a' decrements to [0, 1, 0, ...]. All counts remain non-negative, so I return true.

Another example: if we have ransomNote = 'aa' and magazine = 'ab', after counting magazine we have [1, 1, 0, ...]. Processing ransom note: first 'a' gives us [0, 1, 0, ...], second 'a' gives us [-1, 1, 0, ...]. The count is negative, so we return false immediately.

The time complexity is O(m + n) where m is the length of magazine and n is the length of ransom note, since we traverse each string once. The space complexity is O(1) because we use a fixed-size array of 26 elements regardless of input size. This solution is efficient and straightforward for the given constraints."

## 핵심 아이디어
