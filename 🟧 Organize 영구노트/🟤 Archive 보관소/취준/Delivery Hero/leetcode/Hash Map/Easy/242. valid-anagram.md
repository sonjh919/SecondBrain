## 풀이 코드
```java
class Solution {
    public boolean isAnagram(String s, String t) {
        // 1. 두 문자열의 길이가 다르면 false 반환
        // why: anagram은 같은 문자를 재배열한 것이므로 길이가 같아야 함
        if (s.length() != t.length()) return false;
        
        // 2. 각 문자의 출현 횟수를 저장할 배열 생성
        // why: 소문자 알파벳만 사용하므로 크기 26의 배열로 충분
        int[] charCount = new int[26];
        
        // 3. 두 문자열을 동시에 순회하며 카운트 조정
        // why: s의 문자는 증가, t의 문자는 감소시켜 한 번의 순회로 비교
        for (int i = 0; i < s.length(); i++) {
            charCount[s.charAt(i) - 'a']++;
            charCount[t.charAt(i) - 'a']--;
        }
        
        // 4. 모든 카운트가 0인지 확인
        // why: 모든 문자의 출현 횟수가 같으면 anagram
        for (int count : charCount) {
            if (count != 0) {
                return false;
            }
        }
        
        // 5. 모든 검증을 통과하면 anagram
        // why: 두 문자열이 같은 문자를 같은 횟수만큼 포함
        return true;
    }
}
```

**시간 복잡도**: O(n) - 문자열을 한 번 순회 (n은 문자열 길이)  
**공간 복잡도**: O(1) - 크기 26의 고정 배열 사용 (알파벳 개수는 상수)

## 면접 대본 (영어)

"Let me explain my approach to checking if two strings are anagrams.

The key insight is that anagrams contain exactly the same characters with the same frequencies, just in a different order. So if I count the frequency of each character in both strings, they should match perfectly.

First, I handle an optimization: if the two strings have different lengths, they can't possibly be anagrams, so I return false immediately.

I'll use a frequency counting approach with an array. Since we're only dealing with lowercase English letters, I can use an array of size 26 where each index represents a letter from 'a' to 'z'.

Here's how the algorithm works: I iterate through both strings simultaneously in a single loop. For each position, I increment the count for the character from string s and decrement the count for the character from string t. This is more efficient than doing two separate passes.

After processing both strings, if they're anagrams, all the increments from s should be perfectly canceled out by the decrements from t, resulting in all counts being zero. So I iterate through the count array and check if any count is non-zero. If I find any non-zero count, it means the character frequencies don't match, so the strings are not anagrams.

Let me walk through an example: if we have s = 'anagram' and t = 'nagaram', both have length 7. As I process: at i=0, 'a' from s increments to 1, 'n' from t decrements to -1. At i=1, 'n' from s goes to 0, 'a' from t goes to 0. I continue this process for all characters. After processing everything, when I check the final counts, all values are 0, so I return true.

Another example: if we have s = 'rat' and t = 'car', after processing, 'r' count is 1 (appears in s but not t at same frequency), 'c' count is -1 (appears in t but not s), and 'a' and 't' cancel out to 0. Since we have non-zero counts, we return false.

The time complexity is O(n) where n is the length of the strings, since we make one pass through the strings and one pass through the 26-element array. The space complexity is O(1) because we use a fixed-size array of 26 elements regardless of input size.

For the follow-up question about Unicode characters: instead of using a fixed array, I would use a HashMap to count character frequencies. The HashMap can handle any character set and would have space complexity of O(k) where k is the number of unique characters. The time complexity would still be O(n). The algorithm would remain the same - just replace the array with a HashMap."
