#HashTable #String

## 풀이 코드
```java
class Solution {
    public boolean wordPattern(String pattern, String s) {
        // 1. 문자열 s를 단어 배열로 분리
        // why: 각 단어를 pattern의 문자와 매핑하기 위함
        String[] words = s.split(" ");
        
        // 2. pattern 길이와 단어 개수가 다르면 false 반환
        // why: 일대일 매핑이 불가능하므로 조기 반환
        if (pattern.length() != words.length) return false;
        
        // 3. 양방향 매핑을 위한 두 개의 HashMap 생성
        // why: pattern->word와 word->pattern 매핑을 모두 확인하여 일대일 대응 보장
        Map<Character, String> charToWord = new HashMap<>();
        Map<String, Character> wordToChar = new HashMap<>();
        
        // 4. pattern과 words를 동시에 순회하며 매핑 관계 검증
        // why: 각 위치에서 문자와 단어 간 매핑이 일관성 있게 유지되는지 확인
        for (int i = 0; i < pattern.length(); i++) {
            char c = pattern.charAt(i);
            String word = words[i];
            
            // 5. pattern의 문자가 이미 매핑되어 있는지 확인
            // why: 같은 문자가 다른 단어로 매핑되면 안 됨
            if (charToWord.containsKey(c)) {
                if (!charToWord.get(c).equals(word)) {
                    return false;
                }
            } else {
                charToWord.put(c, word);
            }
            
            // 6. 단어가 이미 매핑되어 있는지 확인
            // why: 서로 다른 문자가 같은 단어로 매핑되면 안 됨 (일대일 대응)
            if (wordToChar.containsKey(word)) {
                if (wordToChar.get(word) != c) {
                    return false;
                }
            } else {
                wordToChar.put(word, c);
            }
        }
        
        // 7. 모든 검증을 통과하면 패턴 일치
        // why: 양방향 매핑이 모두 일관성 있게 유지됨
        return true;
    }
}
```

**시간 복잡도**: O(n + m) - n은 pattern 길이, m은 문자열 s의 길이 (split 연산 포함)  
**공간 복잡도**: O(w) - w는 고유 단어의 개수 (HashMap과 단어 배열 저장)

## 면접 대본 (영어)

"Let me explain my approach to checking if a string follows a given pattern.

The key insight is that this problem requires a bijection - a one-to-one and onto mapping - between characters in the pattern and words in the string. Each character must map to exactly one unique word, and each word must map to exactly one character.

First, I split the string s into an array of words using the space delimiter. Then I immediately check if the length of the pattern matches the number of words. If they don't match, it's impossible to establish a one-to-one mapping, so I return false.

I'll use two HashMaps to track the bidirectional mapping. One map goes from characters to words, and the other from words to characters. This dual-map approach ensures we maintain a bijection.

Here's how the algorithm works: I iterate through the pattern and words array simultaneously. At each position, I have a character from the pattern and a word from the string.

First, I check if the character has already been mapped to a word. If it has, I verify that it maps to the current word. If it maps to a different word, that means one character is trying to map to multiple words, which violates the pattern, so I return false. If the character hasn't been seen before, I add the mapping.

Similarly, I check the reverse mapping. If the word has already been mapped to a character, I verify it's the current character. If not, it means multiple characters are trying to map to the same word, which also violates the bijection property. If the word is new, I add its mapping.

Let me walk through an example: if we have pattern = 'abba' and s = 'dog cat cat dog', after splitting we get words = ['dog', 'cat', 'cat', 'dog']. At i=0, 'a' maps to 'dog' and 'dog' maps to 'a'. At i=1, 'b' maps to 'cat' and 'cat' maps to 'b'. At i=2, we check 'b' - it's already mapped to 'cat', which matches, and 'cat' is already mapped to 'b', which also matches. At i=3, 'a' is mapped to 'dog' and 'dog' is mapped to 'a', both match. All checks pass, so we return true.

Another example: if we have pattern = 'abba' and s = 'dog cat cat fish', at i=3, we try to map 'a' to 'fish', but 'a' is already mapped to 'dog'. This inconsistency means the pattern doesn't match, so we return false.

A tricky case: pattern = 'abba' and s = 'dog dog dog dog'. At i=1, 'b' maps to 'dog', but when we check the reverse, 'dog' is already mapped to 'a' from position 0. This violates the bijection, so we return false. This demonstrates why we need both maps.

The time complexity is O(n + m) where n is the pattern length and m is the string length - we need to split the string which takes O(m), then we iterate through n elements. The space complexity is O(w) where w is the number of unique words, for storing the mappings and the word array. This solution correctly handles the bijection constraint through dual mapping."
