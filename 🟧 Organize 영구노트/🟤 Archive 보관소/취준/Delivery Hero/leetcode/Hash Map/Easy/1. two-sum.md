#Array #HashTable

## 풀이 코드
```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        // 1. 값과 인덱스를 저장할 HashMap 생성
        // why: 이전에 본 숫자와 그 위치를 O(1) 시간에 조회하기 위함
        Map<Integer, Integer> map = new HashMap<>();
        
        // 2. 배열을 순회하며 complement 찾기
        // why: 현재 숫자와 더해서 target이 되는 숫자가 이미 나왔는지 확인
        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];
            
            // 3. complement가 map에 있으면 답을 찾은 것
            // why: 이전에 나온 숫자와 현재 숫자를 더하면 target이 됨
            if (map.containsKey(complement)) {
                return new int[] {map.get(complement), i};
            }
            
            // 4. 현재 숫자와 인덱스를 map에 저장
            // why: 나중에 나올 숫자들이 현재 숫자를 complement로 찾을 수 있도록
            map.put(nums[i], i);
        }
        
        // 5. 문제 조건상 항상 답이 존재하므로 도달하지 않음
        // why: 컴파일러를 위한 형식적 반환문
        return new int[] {};
    }
}
```

**시간 복잡도**: O(n) - 배열을 한 번 순회 (n은 배열 길이)  
**공간 복잡도**: O(n) - 최악의 경우 모든 원소를 HashMap에 저장

## 면접 대본 (영어)

"Let me explain my approach to solving the Two Sum problem.

The key insight is to use a HashMap to achieve a one-pass solution. Instead of checking every pair of numbers which would take O(n²) time, I can look up the complement of each number in constant time using a hash table.

The idea is simple: for each number in the array, I calculate what value I would need to add to it to reach the target - I call this the complement. If I've already seen this complement earlier in the array, then I've found my pair. If not, I store the current number and its index in the HashMap for future lookups.

Here's how the algorithm works: I iterate through the array once. At each position i, I calculate the complement as target minus the current number. Then I check if this complement exists in my HashMap. If it does, it means I've previously encountered a number that, when added to the current number, equals the target. So I return the indices of both numbers - the index stored in the HashMap and the current index i.

If the complement isn't in the HashMap, I add the current number and its index to the HashMap, then continue to the next element.

Let me walk through an example: if we have nums = [2, 7, 11, 15] and target = 9, at i=0, the number is 2 and complement is 7. The HashMap is empty, so I add 2->0 to the map. At i=1, the number is 7 and complement is 2. I check the HashMap and find that 2 exists at index 0. So I return [0, 1] immediately.

Another example: if we have nums = [3, 2, 4] and target = 6, at i=0, complement is 3, map is empty, so I add 3->0. At i=1, number is 2, complement is 4, not in map, so I add 2->1. At i=2, number is 4, complement is 2, which is in the map at index 1. I return [1, 2].

An important edge case: nums = [3, 3] and target = 6. At i=0, complement is 3, map is empty, so I add 3->0. At i=1, number is 3, complement is 3, which is in the map at index 0. I return [0, 1]. This works correctly because I check for the complement before adding the current number to the map, so I don't accidentally use the same element twice.

The time complexity is O(n) where n is the length of the array, since we traverse it once and each HashMap operation takes O(1) on average. The space complexity is O(n) in the worst case where we need to store almost all elements in the HashMap before finding the answer. This solution satisfies the follow-up requirement of being better than O(n²)."

## 핵심 아이디어

HashMap에 이전 숫자들을 저장하며 현재 숫자의 complement가 존재하는지 O(1)로 확인하여 전체 O(n) 시간에 해결
