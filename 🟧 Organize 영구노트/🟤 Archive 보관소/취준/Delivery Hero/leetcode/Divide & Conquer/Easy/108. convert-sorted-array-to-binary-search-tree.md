## 풀이 코드
```java
/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode() {}
 *     TreeNode(int val) { this.val = val; }
 *     TreeNode(int val, TreeNode left, TreeNode right) {
 *         this.val = val;
 *         this.left = left;
 *         this.right = right;
 *     }
 * }
 */
class Solution {
    public TreeNode sortedArrayToBST(int[] nums) {
        // 1. 배열 전체를 대상으로 재귀 시작
        // why: 전체 배열에서 균형 잡힌 BST를 구성
        return buildBST(nums, 0, nums.length - 1);
    }
    
    private TreeNode buildBST(int[] nums, int left, int right) {
        // 2. left가 right를 넘어가면 null 반환
        // why: 더 이상 처리할 원소가 없는 base case
        if (left > right) {
            return null;
        }
        
        // 3. 중간 인덱스를 루트로 선택
        // why: 중간값을 루트로 하면 좌우 서브트리의 크기가 거의 같아져 균형 유지
        int mid = left + (right - left) / 2;
        TreeNode root = new TreeNode(nums[mid]);
        
        // 4. 왼쪽 절반으로 왼쪽 서브트리 구성
        // why: 중간값보다 작은 값들로 왼쪽 서브트리 생성
        root.left = buildBST(nums, left, mid - 1);
        
        // 5. 오른쪽 절반으로 오른쪽 서브트리 구성
        // why: 중간값보다 큰 값들로 오른쪽 서브트리 생성
        root.right = buildBST(nums, mid + 1, right);
        
        // 6. 구성된 서브트리의 루트 반환
        // why: 현재 범위의 균형 잡힌 BST 반환
        return root;
    }
}
```

**시간 복잡도**: O(n) - 모든 원소를 한 번씩 노드로 생성 (n은 배열 길이)  
**공간 복잡도**: O(log n) - 재귀 호출 스택의 깊이 (균형 트리의 높이)

## 면접 대본 (영어)

"Let me explain my approach to converting a sorted array into a height-balanced Binary Search Tree.

The key insight is to use a divide-and-conquer strategy. Since the array is already sorted, if we choose the middle element as the root, all elements to its left will naturally go into the left subtree, and all elements to its right will go into the right subtree. This maintains the BST property. Moreover, by always choosing the middle element, we ensure that the left and right subtrees have roughly equal numbers of nodes, which guarantees a balanced tree.

I'll solve this recursively. The base case is when the left index exceeds the right index, meaning there are no elements to process, so I return null.

For the recursive case, I calculate the middle index of the current range. I'm careful to use 'left + (right - left) / 2' instead of '(left + right) / 2' to avoid potential integer overflow, though with the given constraints it wouldn't be an issue here.

I create a new TreeNode with the value at the middle index - this becomes the root of the current subtree. Then I recursively build the left subtree using the left half of the array (from left to mid-1), and the right subtree using the right half (from mid+1 to right). Finally, I return the root.

Let me walk through an example: nums = [-10,-3,0,5,9]. The array has 5 elements. Initially, left=0, right=4, mid=2, so we create root with value 0. For the left subtree, we call with left=0, right=1, mid=0, creating node -10. Then for -10's right child, we call with left=1, right=1, mid=1, creating node -3. For the right subtree of 0, we call with left=3, right=4, mid=3, creating node 5. Then for 5's right child, we call with left=4, right=4, mid=4, creating node 9. The resulting tree is [0,-10,5,null,-3,null,9], which is height-balanced.

Another example: nums = [1,3], mid is 0, root is 1, and 3 becomes the right child. We could also make 3 the root with 1 as left child - both are valid height-balanced BSTs. The algorithm chooses based on which element is at the middle index.

Why is this guaranteed to be balanced? At each step, we split the array roughly in half. The left subtree gets about n/2 elements, and so does the right subtree. This continues recursively, so the maximum height is log₂(n). Since both subtrees have similar heights at each level, the tree is height-balanced by definition.

An important property of this algorithm: it not only creates a BST (which is guaranteed by using a sorted array and maintaining left < root < right), but also ensures it's height-balanced. A height-balanced tree means that for every node, the heights of its left and right subtrees differ by at most one.

The time complexity is O(n) where n is the length of the array, because we create exactly one node for each array element, and each node is created in constant time. The space complexity is O(log n) for the recursion stack in a balanced tree. We don't count the output tree itself in space complexity.

This solution elegantly demonstrates how the sorted property of the input array allows us to construct a balanced BST in linear time, which is optimal since we need to process each element at least once."
