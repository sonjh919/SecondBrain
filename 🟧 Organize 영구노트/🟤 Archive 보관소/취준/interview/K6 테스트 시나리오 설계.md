### Question
K6로 수행한 15개 핵심 서비스 플로우에 대해 구체적으로 어떤 테스트 시나리오를 설계하셨나요?

### Answer
유저 테스트를 통해 유저가 일반적으로 앱을 키고 사용할 때의 플로우를 뽑아서 진행했습니다. 예를 들면 토론방 검색, 토론방 생성 및 댓글 생성, 좋아요 등이 있습니다.

### Feedback
**예상 베스트 답변**: 실제 사용자 행동 패턴을 분석하여 15개의 핵심 플로우를 선정했습니다. 유저 테스트와 Google Analytics/Firebase 등의 데이터를 기반으로 사용 빈도가 높은 기능을 우선순위화했습니다. 구체적인 플로우로는 (1) 회원가입/로그인, (2) 메인 페이지 진입 및 도서 목록 조회, (3) 토론방 검색 (Full-Text Index 활용), (4) 토론방 생성, (5) 토론방 상세 조회, (6) 댓글 작성/조회, (7) 좋아요/북마크 등록, (8) 알림 조회, (9) 마이페이지 조회 등이 포함됩니다. 각 플로우는 실제 사용자 시나리오를 반영하여 "앱 실행 → 로그인 → 토론방 검색 → 상세 조회 → 댓글 작성"처럼 연속된 액션으로 구성했습니다.

**현재 답변 평가**: 실제 유저 테스트를 기반으로 현실적인 시나리오를 선정하신 점이 매우 좋습니다. 토론방 검색, 생성, 댓글, 좋아요 같은 구체적인 예시도 제시하셨습니다. 다만 15개 플로우 전체 목록이나, 각 플로우의 우선순위를 어떻게 결정했는지(예: 사용 빈도, 비즈니스 임팩트 등), 그리고 K6 스크립트에서 이러한 플로우를 어떻게 시나리오로 구현했는지에 대한 설명이 추가되면 더욱 완성도 높은 답변이 됩니다.

**추천사항**: "유저 테스트"가 구체적으로 어떤 방식이었는지(베타 테스트, 사용자 인터뷰, 분석 도구 활용 등), 그리고 K6의 `scenario` 기능을 활용하여 여러 플로우를 동시에 테스트했는지 언급하시면 좋습니다. 또한 각 플로우별로 예상 TPS나 응답 시간 목표를 다르게 설정했는지도 설명하시면 더욱 체계적인 접근을 보여줄 수 있습니다.

### Recommendation
K6 부하 테스트 시나리오 설계는 실제 사용자 행동 패턴을 정확히 반영하는 것이 핵심입니다. TodokTodok 프로젝트에서는 데이터 기반 접근법으로 15개 핵심 플로우를 선정하고 테스트했습니다.

**플로우 선정 방법론**

**1. 데이터 수집**
- **유저 테스트**: 베타 테스터 20명의 실제 앱 사용 패턴 관찰
- **로그 분석**: 애플리케이션 로그에서 API 호출 빈도 분석
- **Firebase Analytics**: 화면별 체류 시간, 이벤트 발생 빈도 수집
- **팀 내 논의**: PM, 디자이너와 함께 비즈니스 크리티컬한 기능 식별

**2. 우선순위 기준**
- **사용 빈도**: 일일 호출 횟수가 많은 API
- **비즈니스 임팩트**: 서비스 핵심 가치와 직결된 기능
- **리소스 소비**: DB 쿼리가 복잡하거나 외부 API 호출이 있는 무거운 작업
- **사용자 경험**: 응답 지연 시 이탈률이 높은 기능

**15개 핵심 플로우 목록**

**인증 및 사용자 관리 (2개)**
1. 회원가입 및 이메일 인증
2. 로그인 및 토큰 발급

**도서 및 토론방 조회 (4개)**
3. 메인 페이지 진입 (인기 도서, 추천 토론방)
4. 도서 목록 조회 (페이징, 필터링)
5. 토론방 검색 (Full-Text Index 활용)
6. 토론방 상세 조회

**토론방 생성 및 참여 (3개)**
7. 토론방 생성 (도서 검색 + 생성)
8. 토론방 참여 신청
9. 댓글 작성 및 조회

**인터랙션 (3개)**
10. 좋아요 등록/취소
11. 북마크 추가/삭제
12. 대댓글 작성

**알림 및 마이페이지 (3개)**
13. 알림 목록 조회 및 읽음 처리
14. 마이페이지 조회 (참여 중인 토론방)
15. 프로필 수정

**K6 시나리오 설계**

**단일 플로우 테스트 (9개)**

각 플로우를 독립적으로 테스트하여 개별 API의 성능 한계 측정:

```javascript
// 예시: 토론방 검색 플로우
import http from 'k6/http';
import { check, sleep } from 'k6';

export let options = {
  stages: [
    { duration: '30s', target: 50 },  // Ramp-up
    { duration: '1m', target: 100 },  // Sustained load
    { duration: '30s', target: 0 },   // Ramp-down
  ],
  thresholds: {
    'http_req_duration': ['p(95)<500'], // 95%가 500ms 이하
    'http_req_failed': ['rate<0.01'],   // 실패율 1% 미만
  },
};

export default function() {
  const searchKeyword = '해리포터';
  
  // 1. 토론방 검색
  const searchRes = http.get(
    `${BASE_URL}/api/discussion-rooms/search?keyword=${searchKeyword}`,
    { headers: { 'Authorization': `Bearer ${token}` } }
  );
  
  check(searchRes, {
    'search status is 200': (r) => r.status === 200,
    'search response time < 500ms': (r) => r.timings.duration < 500,
  });
  
  sleep(1); // Think time
}
```

**복합 플로우 테스트 (6개)**

실제 사용자 행동을 모방한 연속적인 액션:

```javascript
// 예시: 토론방 참여 플로우
export default function() {
  // 1. 메인 페이지 진입
  http.get(`${BASE_URL}/api/main`);
  sleep(2);
  
  // 2. 토론방 검색
  const searchRes = http.get(`${BASE_URL}/api/discussion-rooms/search?keyword=추리소설`);
  const rooms = JSON.parse(searchRes.body).data;
  sleep(1);
  
  // 3. 첫 번째 토론방 상세 조회
  const roomId = rooms[0].id;
  http.get(`${BASE_URL}/api/discussion-rooms/${roomId}`);
  sleep(3);
  
  // 4. 댓글 작성
  http.post(`${BASE_URL}/api/discussion-rooms/${roomId}/comments`, 
    JSON.stringify({ content: '좋은 책이네요!' }),
    { headers: { 'Content-Type': 'application/json' } }
  );
  sleep(1);
  
  // 5. 좋아요
  http.post(`${BASE_URL}/api/discussion-rooms/${roomId}/likes`);
  sleep(1);
}
```

**성능 목표 설정**

각 플로우별 차별화된 목표:

| 플로우 카테고리 | 목표 TPS | P95 응답시간 | 실패율 |
|--------------|---------|------------|--------|
| 조회 (메인, 목록) | 100+ | < 300ms | < 0.1% |
| 검색 (Full-Text) | 80+ | < 500ms | < 0.1% |
| 생성 (토론방, 댓글) | 50+ | < 1000ms | < 0.5% |
| 인터랙션 (좋아요) | 150+ | < 200ms | < 0.1% |

**테스트 실행 전략**

1. **단일 플로우 개별 테스트**: 각 API의 한계 파악
2. **복합 플로우 테스트**: 실제 사용자 시나리오 검증
3. **가중치 혼합 테스트**: 프로덕션 트래픽 패턴 시뮬레이션
4. **스파이크 테스트**: 갑작스러운 트래픽 증가 대응력 검증
5. **스트레스 테스트**: 시스템 한계점 파악
