[](Index%20Fragmentation%20개념과%20발견.md)### Question
Full-Text Index에 대해서 자세하게 설명해줄 수 있나요?

### Answer
full-text index는 텍스트 데이터를 형태소 단위로 분리하여 토큰화하고, 각 토큰에 대해 역인덱스 구조를 생성하는 인덱스입니다.

### Feedback
핵심 개념인 토큰화와 역인덱스를 정확하게 언급하신 점이 좋습니다. 다만, Full-Text Index가 일반 인덱스와 어떻게 다른지, 그리고 실제 검색 시 어떤 방식으로 동작하는지에 대한 설명이 추가되면 더욱 완성도 높은 답변이 될 것입니다. 특히 LIKE 검색과의 차이점이나 토큰 매칭 방식을 언급하면 좋습니다.

### Recommendation
Full-Text Index는 텍스트 데이터를 효율적으로 검색하기 위한 특수한 인덱스 구조입니다. 일반적인 B-Tree 인덱스가 컬럼 값 전체를 기준으로 인덱싱하는 것과 달리, Full-Text Index는 텍스트 내용을 형태소 분석기를 통해 의미 있는 단위(토큰)로 분리합니다.

분리된 각 토큰에 대해 역인덱스(Inverted Index) 구조를 생성하는데, 이는 '토큰 → 해당 토큰이 포함된 문서(레코드) 목록'을 매핑하는 방식입니다. 예를 들어 "스프링 부트 학습"이라는 텍스트가 있다면, "스프링", "부트", "학습" 각각이 토큰이 되고, 각 토큰이 어떤 문서에 포함되어 있는지 인덱스에 저장됩니다.

검색 시에는 검색어도 동일하게 토큰화되어, 각 토큰과 일치하는 문서를 역인덱스에서 빠르게 찾아냅니다. 이는 LIKE '%keyword%' 방식의 검색이 전체 테이블을 스캔해야 하는 것과 달리, 인덱스를 통해 O(1)에 가까운 속도로 해당 문서를 찾을 수 있어 대용량 텍스트 검색에 매우 효율적입니다. MySQL에서는 MATCH() AGAINST() 구문을 사용하여 Full-Text 검색을 수행할 수 있습니다.
