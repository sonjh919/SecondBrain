### Question
단일 테스트와 가중치 테스트의 차이는 무엇인가요?

### Answer
단일 테스트는 하나의 상황에 대해서만 테스트했습니다. 메인 페이지 진입 시 호출되는 api들을 하나의 그룹으로 묶고, 진입 상황 하나에 대해서만 테스트를 진행했습니다.
가중치 테스트는 사용자가 실제로 앱을 사용하면서 일어나는 상황들에 대해 각각의 그룹으로 묶고, 더 중요하고 자주 일어나는 상황에 높은 가중치를 주어 한꺼번에 테스트를 진행했습니다.

### Feedback
실제 사용 시나리오를 기반으로 단일/가중치 테스트를 설계하신 접근이 매우 실무적입니다. 다만, 구체적으로 어떤 기준으로 가중치를 부여했는지(예: 사용자 행동 패턴 분석 데이터, 로그 분석 결과 등), 그리고 가중치 테스트를 통해 발견한 추가적인 병목이나 인사이트가 있었는지 언급하면 더욱 깊이 있는 답변이 될 것입니다.

### Recommendation
단일 테스트는 특정 사용자 시나리오 하나에 집중하여 개별 API 그룹의 성능을 측정하는 테스트입니다. 예를 들어 메인 페이지 진입 시 호출되는 여러 API들(토론방 목록 조회, 추천 도서 조회 등)을 하나의 시나리오 그룹으로 묶어, 해당 상황만 집중적으로 테스트했습니다. 이를 통해 각 시나리오별 최대 처리량과 병목 지점을 명확히 파악할 수 있었습니다.

반면 가중치 테스트는 실제 운영 환경을 시뮬레이션하기 위해 사용자가 앱을 사용하면서 발생하는 다양한 상황들을 동시에 테스트하는 방식입니다. 메인 페이지 진입, 토론방 생성, 게시글 작성, 댓글 조회 등 핵심 플로우 15개를 각각 그룹화하고, 실제 사용자 행동 패턴과 예상 빈도를 분석하여 가중치를 부여했습니다. 예를 들어 조회 API는 높은 가중치(70%), 생성/수정 API는 낮은 가중치(30%)를 적용하는 식입니다.

이렇게 가중치 테스트를 진행한 결과, 단일 테스트에서는 문제없던 설정값이 실제로는 여러 API가 동시에 리소스를 경쟁하면서 성능이 떨어지는 것을 발견했습니다. 특히 DB 커넥션 풀이 여러 종류의 쿼리에 의해 동시에 점유되면서 대기 시간이 증가하는 현상을 확인할 수 있었고, 이는 Scale Out 결정의 핵심 근거가 되었습니다. 가중치 테스트를 통해 실제 운영 환경에 더 가까운 성능 데이터를 얻을 수 있었습니다.
