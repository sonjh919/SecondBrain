#정렬 #Onlogn

+ 기준값(pivot)을 선정해 해당 값보다 작은 데이터와 큰 데이터로 분류하는 것을 반복해 정렬하는 분할 정복(Divide And Conquer) 알고리즘
+ pivot 선정 기준이 시간 복잡도에 많은 영향을 끼치지만, 평균적으로는 O(nlogn)
+ 추가 메모리 공간을 필요로 하지 않는다.

### 퀵 정렬 핵심 이론
+ pivot을 중심으로 계속 **데이터를 2개의 집합으로 나누면서 정렬하는 것**

1. 분할(Divide) : 입력 배열을 피벗을 기준으로 비균등하게 2개의 부분 배열(피벗을 중심으로 왼쪽 = 피벗보다 작은 요소들, 오른쪽 = 피벗보다 큰 요소들)로 분할한다.
2. 정복(Conquer) : 부분 배열을 정렬한다. 부분 배열의 크기가 충분히 작지 않으면 순환 호출을 이용하여 다시 분할 정복 방법을 적용한다.
3. 결합(Combine) : 정렬된 부분 배열들을 하나의 배열에 합병한다.
