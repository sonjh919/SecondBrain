#자료구조 #배열

같은 자료형의 변수들을 하나의 묶음으로 다루는 것

### 선언
```java
int[] arr = {1,2,3,4,5};
int[] arr = new int[] {1,2,3,4,5};
int[] arr = new int[N]; // 0으로 초기화
```

### 쓸만한 메서드
```java
int[] arr1 = new int[]{1,2,3,4,5};  
int[] arr2 = new int[]{1,2,4,5,6};  
  
Arrays.binarySearch(arr1, 0); // O(log n)  
Arrays.asList(arr1); // 자바 5, O(1)  
Arrays.compare(arr1, arr2); // 자바 9, O(N)
Arrays.copyOf(arr1); // O(N) 얕은복사  
Arrays.copyOfRange(arr1, 1, 3); // O(N) 얕은복사  
Arrays.fill(arr1, 1); // O(N)  
Arrays.mismatch(arr1, arr2); // 자바 9, O(N)
Arrays.sort(arr1); // O(n log n) 최악은 O(n^2)이나, 나올일 거의 없음.
```

### 배열 특징
#### 1. 인덱스와 값의 상관관계
배열로 얻을 수 있는 정보는 인덱스랑 값이다.
[[값에 접근할 때에는 인덱스를 이용]]한다.

#### 2. 연속적인 인덱스
배열은 연속적인 메모리로 할당되기 때문에, [[인덱스가 연속적]]이다.

#### 3. 고정 크기
[[배열은 고정 크기]]이다.
배열은 생성 시 크기가 정해지면 변경할 수 없다.
이 말은 인덱스가 연속적이라는 뜻!
즉, 순서를 동적으로 바꾸고 싶으면 배열을 쓰면 안된다!
