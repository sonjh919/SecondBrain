#자료구조 #리스트

**크기를 동적으로 조정**할수 있는 대표적인 배열 기반의 리스트
내부적으로 배열을 사용하여 요소를 저장하며, 요소를 추가하거나 삭제할 때 크기가 동적으로 조정된다.

### 선언
```java
List<Integer> list = new ArrayList<>();
```

### 쓸만한 메서드
```java
List<Integer> immutable_list = List.of(6,7,8,9,10); // 수정불가  
List<Integer> list = new ArrayList<>(Arrays.asList(2,4,5,1,3)); // 수정가능  
  
Collections.sort(list); // O(nlogn)  
Collections.reverse(list); // O(n)  
  
int max = Collections.max(list); // O(n)  
int min = Collections.min(list); // O(n)  
  
Collections.swap(list, 0, 1); // O(1), 탐색 포함하면 O(n)
int res = Collections.binarySearch(list, 3); // O(logn)  
boolean isSame = Collections.disjoint(immutable_list, list); // O(n*m) // 일치값이 하나도 없으면 true, 하나라도 있으면 false  
int count = Collections.frequency(list,1); // O(n) // list 안에 1이 몇번 등장했는지  
  
List<Integer> list3 = new ArrayList<>(Arrays.asList(1,2,3,4,5));  
List<Integer> list4 = new ArrayList<>(Arrays.asList(3,4));  
  
int index = Collections.indexOfSubList(list3, list4); // O(n*m) 뒤에 리스트가 앞에 리스트의 몇번째 인덱스부터 나오는지 // 2
Collections.rotate(list3,2); // O(n) // 마지막 요소를 꺼내서 맨앞으로 넣는 것. 리스트,횟수 // [1,2,3,4,5] -> [4,5,1,2,3]
Collections.rotate(list3,-2);// O(n) // 맨앞 요소를 꺼내서 맨뒤로 넣는 것. 리스트,횟수 // [1,2,3,4,5] -> [3,4,5,1,2]  
list3.lastIndexOf(2); // O(n) 뒤에서부터 위치 찾기  
list3.indexOf(2); // O(n) 앞에서부터 위치 찾기  
  
boolean contains = new HashSet<>(list3).containsAll(list4); // O(n*m) // 주어진 컬렉션과 모두 일치해야 true 
(set)boolean contain = list3.contains(3); // O(n) // 일치하는지
```

### 리스트 특징
#### 1. 중간에 값을 넣을 수 있음 == 고정 인덱스가 아님!
중간에 값을 넣으면, 그 이후의 인덱스는 한 칸씩 밀린다!!

```java
List<Integer> list = new ArrayList<>();  
list.add(1);  
list.add(2);  
list.add(3); // 1, 2, 3
list.add(1,4); // 1, 4, 2, 3
```
