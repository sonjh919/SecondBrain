#자료구조 

+ 노드(**Node**)(또는 정점, Vertex)과 간선(Edge)로 이루어진 자료구조
+ Node간의 관계를 표현하는 조직도

### 그래프 관련 용어
| 용어     | 내용                |
| ------ | ----------------- |
| Node   | Node에는 데이터가 저장된다. |
| Edge   | Node를 연결하는 선      |
| Weight | Edge에 할당되는 가중치    |
### 그래프 표현 3가지
### 에지 리스트
+ **에지를 중심**으로 그래프를 표현하는 방법
+ 배열에 출발 노드, 도착 노드를 저장하여 에지를 표현함.
+ 또는 출발 노드, 도착 노드, 가중치를 저장하여 가중치가 있는 에지를 표현함.

+ 특정 노드와 관련되어있는 에지를 탐색하기는 쉽지 않다.
+ **벨만 포드**, **크루스칼** 알고리즘에 사용하며, 노드 중심 알고리즘에는 잘 사용하지 않는다.

#### 가중치 없는 그래프 표현하기
+ 1에서 2로 뻗어나가는 에지는 `[1,2]`로 표현한다.

![[edge_list_graph.jpg]]

#### 가중치 있는 그래프 표현하기
+ 1에서 2로 향하는 가중치가 8인 에지는 `[1,2,8]`로 표현한다.

![[edge_list_weight_graph.jpg]]

### 인접 행렬
+ 2차원 배열을 자료구조로 이용하여 그래프를 표현한다.
+ **노드 중심**으로 그래프를 표현한다.

+ 구현 난이도가 쉽다.
+ 두 노드를 연결하는 에지의 여부와 가중치는 배열에 직접 접근하면 바로 확인 가능하다.
+ 노드와 관련되어 있는 에지를 탐색하려면 N번 접근해야 하므로 노드 개수에 비해 에지가 적을 때에는 공간 효율성이 떨어진다.
+ 노드 개수가 많은 경우 아예 2차원 배열 선언 자체를 할 수 없다.
	+ 노드가 3만개를 넘으면 java heap space 에러가 발생한다.

#### 가중치 없는 그래프 표현하기
+ 1에서 2를 향하는 에지를 1행 2열에 1을 저장하는 방식으로 표현
+ 1을 저장하는 이유는 가중치가 없기 때문
![[array_graph.jpg]]

#### 가중치 있는그래프 표현하기
+ 2에서 5를 향하는 에지의 가중치를 2행 5열에 저장하는 방식으로 표현
![[array_weight_graph.jpg]]

### 인접 리스트
+ ArrayList로 그래프를 표현
+ 노드 개수만큼 ArrayList를 선언

+ 다른 방법에 비해 구현이 복잡한 편
+ 노드와 연결되어 있는 에지를 탐색하는 시간은 매우 빠름
+ 노드 개수가 커도 공간 효율이 좋아 메모리 초과 에러도 발생하지 않는다.
+ **실제 코테에서 선호하는 방식!!**

#### 가중치 없는 그래프 표현하기
+ N번 노드와 연결되어 있는 노드를 배열의 위치 N에 연결된 노드 개수반큼 배열을 연결하는 방식으로 표현
	+ 노드 1과 연결된 2, 3 노드는 `ArrayList[1]`에 `[2,3]`을 연결하는 방식으로 표현
![[arrayList_graph.jpg]]

#### 가중치 있는 그래프 표현하기
+ 가중치가 있는 경우 자료형을 클래스로 사용한다.
	+ 노드 1과 2가 가중치 8 에지로, 노드 1과 3이 가중치 3 에지로 연결되어있다는 뜻!
![[arrayList_weight_graph.jpg]]
