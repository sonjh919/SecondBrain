### Question
이력서에 Mock 데이터를 운영과 유사한 분포로 수정했다고 하셨는데, 어떻게 운영 데이터의 분포를 파악하셨나요?

### Answer
운영 데이터가 그렇게 크지 않았기 때문에 충분히 쿼리로도 파악이 가능해서 데이터 분포를 파악하기도 어렵지 않았었습니다. 이런 데이터를 토대로 저희가 조금씩 점검을 해가면서 충분히 데이터를 파악할 수 있었고, 생성형 AI를 활용해서 데이터셋을 확실하게 뽑아내려고 노력했던 것 같습니다.

### Feedback
운영 데이터 규모가 작아서 직접 쿼리로 파악할 수 있었다는 현실적인 답변입니다. 다만, 어떤 쿼리를 사용해서 분포를 파악했는지, 그리고 생성형 AI를 어떻게 활용했는지 구체적으로 설명하면 더 설득력 있는 답변이 됩니다. 또한 데이터 규모가 컸다면 어떻게 했을지에 대한 대안도 언급하면 좋습니다.

### Recommendation
"운영 데이터가 아직 초기 단계여서 규모가 크지 않았기 때문에 직접 쿼리로 분포를 파악할 수 있었습니다.

구체적으로는 SELECT 컬럼명, COUNT(*) FROM 테이블 GROUP BY 컬럼명 형태의 쿼리로 각 컬럼의 카디널리티와 값 분포를 확인했습니다. 특히 문제가 됐던 복합 인덱스의 선두 컬럼은 운영에서 실제로 수백 개의 고유값을 가지고 있었는데, Mock 데이터에서는 2~3개 값만 사용하고 있어서 카디널리티 차이가 컸습니다.

분포를 파악한 후에는 생성형 AI를 활용해서 운영과 유사한 패턴의 테스트 데이터셋을 생성했습니다. 예를 들어 '선두 컬럼은 500개의 고유값, 두 번째 컬럼은 날짜 범위 내 균등 분포'와 같은 조건을 주고 INSERT문을 생성하도록 했습니다.

만약 운영 데이터가 대규모였다면, 전체 데이터를 스캔하는 대신 INFORMATION_SCHEMA의 통계 정보나 ANALYZE TABLE 결과를 활용하거나, 샘플링 기법(TABLESAMPLE 또는 WHERE RAND() < 0.01)을 사용해서 분포를 추정했을 것입니다."
